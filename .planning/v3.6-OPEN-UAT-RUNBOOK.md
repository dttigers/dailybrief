## Phase 114 — Safari Extension Quick-Capture Parity (~3 min)

**Prerequisite (already met today):**
- `xcodebuild clean build` exited 0 — rebuild_sha `1076fa7`, time `2026-04-26T16:24:51Z`
- `codesign --verify --deep --strict` passed on `.app` and `.appex`
- `.app` opened once via Finder/`open` to refresh extension binding

**Run on the iMac (current machine):**

1. **⌘Q Safari** (full quit, not just close window — Pitfall 1).
2. Reopen Safari → **Settings → Extensions** → confirm **Vigil Capture** toggle is ON.
3. Click the **Vigil Capture** toolbar icon → popup opens.
4. Observe popup:
   - [ ] Textarea is **empty** (no auto-prefilled URL/title) — SC#1
   - [ ] Cursor focused inside textarea — SC#1
   - [ ] **"Include page URL"** checkbox visible, **unchecked** by default — SC#2 / D-07
   - [ ] **"Cmd+Enter to capture"** hint visible below the button — SC#3 visual
5. Type `phase 114 uat smoke test` → press **⌘+Enter** (NO mouse click).
   - [ ] Button briefly shows "Capturing..." → success area shows "Analyzing..." → within 5s shows **"✓ Captured!"** + a category-badge pill (e.g. Idea / Task / Reflection) — SC#4 / D-09
   - [ ] If category never arrives in 5s: success area shows plain "✓ Captured!" no badge, then closes — SC#4 timeout / D-10 (acceptable, note in observed)
6. Open `https://news.ycombinator.com` (or any real page).
7. Click toolbar icon → type `url append test` → **check the "Include page URL" box** → click Capture.
8. In PWA `/thoughts` (or `curl -s -H "Authorization: Bearer $VK" https://api.vigilhub.io/v1/thoughts | jq '.thoughts[0].content'`):
   - [ ] Most recent thought contains both `url append test` AND a trailing `\n\n${page-title}: https://news.ycombinator.com/...` — SC#2 / D-06 verbatim format
9. **No regressions:** [ ] Phase 107 status pill renders, container app still wired (smoke check on container app launch).

**Sign-off:** Update `114-HUMAN-UAT.md` frontmatter `status: ship-with-uat-pending` → `verified` and tick remaining checkboxes. Commit.

### Test 1 — SC#4 — Lockout audit FIRST (no signup needed)

Run **before** SC#1 because if the migration didn't run, every existing user is locked out and downstream tests don't matter:

```bash
psql "$RAILWAY_DB_URL" -c "
  SELECT
    COUNT(*) AS total_users,
    COUNT(email_verified_at) AS verified_users,
    COUNT(*) FILTER (WHERE email_verified_at IS NULL) AS unverified
  FROM users;
"
```
- [ ] `unverified = 0` AND `verified_users = total_users` — load-bearing assertion (no user locked out)

```bash
psql "$RAILWAY_DB_URL" -c "
  SELECT id, email, (email_verified_at = created_at) AS backfilled
  FROM users
  WHERE id != 1
  ORDER BY id
  LIMIT 5;
"
```
- [ ] All 5 sampled non-seed users return `backfilled = t` — proves D-02 backfill ran

- [ ] Log in as `jamesonmorrill1@gmail.com` → `/settings` → **no verify banner** appears (seed user grandfathered).
