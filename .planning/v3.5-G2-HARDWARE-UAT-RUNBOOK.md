## Section 3 — Phase 45 UX Retest (~8 min)

The Phase 45 fixes (commits b8d5aa2 + 942ee4e) for tap-expand + swipe-out were verified in **simulator only**. This section confirms the same behavior on physical glasses. If anything diverges, it's a SDK/hardware-specific bug — file new, do NOT regress Phase 45.

### 3a. Tap-to-expand work orders

1. Glasses on, plugin canvas open, work-orders list rendered with 3 DEMO_BRIEF tasks.
2. Tap a task in the list (single tap on glasses gesture, NOT iPhone app).
   - [ ] Routes to `task-detail` sub-screen
   - [ ] Full task content visible (no 45-char truncation)
   - [ ] Body renders via TextContainerProperty (scrollable if long)
3. Tap second task → second detail view → confirm same.

### 3b. Swipe out of list

1. Back to work-orders list.
2. Swipe up (SCROLL_TOP gesture on glasses).
   - [ ] Carousel navigates HOME ← WORK_ORDERS (does NOT stay stuck in list)
3. Swipe down (SCROLL_BOTTOM gesture).
   - [ ] Carousel navigates WORK_ORDERS → AFFIRMATION
4. Confirm `isEventCapture: 1` is doing its job: list-internal swipes propagate to nav handler.

### 3c. Double-click home (regression check)

- [ ] Double-click on home still navigates home (legacy path, not removed by Phase 45).

### 3d. Sub-screen exit

1. Inside task-detail, swipe back (any direction).
   - [ ] Returns to work-orders list
   - [ ] task-detail body uses TextContainer (not List), so swipe propagates correctly per Phase 45 architectural pattern.

**If anything diverges:** capture exact gesture + observed behavior in `.planning/phases/106-g2-store-resubmit-atomic/HARDWARE-DIVERGENCE.md` (new file). Do NOT block 106-05 ship — divergence is a follow-up bug, not a Phase 45 regression.
