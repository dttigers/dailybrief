# Milestone v1.0: MVP

**Status:** SHIPPED 2026-04-02
**Phases:** 1-7
**Total Plans:** 17

## Overview

Transform the existing DailyBrief CLI and menu bar monitor into a full personal AI life assistant. Starting with a shared core library and local data layer, we build frictionless text capture, AI-powered triage, a central dashboard, voice/image capture, an evolved daily brief, and Google Calendar integration — delivering a system where every thought is captured with zero friction and organized automatically.

## Phases

### Phase 1: Foundation

**Goal**: Extract shared JarvisCore SPM library with data models, GRDB/SQLite storage with FTS5 search, and config management
**Depends on**: Nothing (first phase)
**Plans**: 3 plans

Plans:
- [x] 01-01: SPM multi-target setup + JarvisCore library scaffold
- [x] 01-02: GRDB schema, models, ThoughtStore actor with FTS5 search
- [x] 01-03: Migrate existing CLI services into JarvisCore

**Details:**
- Established shared library target with GRDB 7.10.0 for SQLite persistence
- Built Thought model with GRDB conformances, DatabaseManager actor with v1 migrations
- ThoughtStore actor with complete CRUD + FTS5 full-text search
- Migrated 7 data models and AppConfig to JarvisCore with public access

### Phase 2: Text Capture

**Goal**: Instant text capture via menu bar popover (< 1s) with global keyboard shortcut and automatic timestamps
**Depends on**: Phase 1
**Plans**: 2 plans

Plans:
- [x] 02-01: Menu bar popover UI + CaptureService actor
- [x] 02-02: Global keyboard shortcut + timestamp automation

**Details:**
- CaptureService actor in JarvisCore with input validation
- Floating CapturePanel (NSPanel) with text input, character count, Cmd+Enter submit
- Carbon API GlobalHotKey (Cmd+Shift+J) — no Accessibility permissions required
- AppDelegate initializes full JarvisCore data stack on launch

### Phase 3: AI Triage

**Goal**: Every captured thought auto-categorized by Claude into task/therapy/idea/reflection/project with confidence scores
**Depends on**: Phase 2
**Plans**: 2 plans

Plans:
- [x] 03-01: TriageService actor + Claude API integration with SwiftAnthropic
- [x] 03-02: Confidence score UX + category override flow

**Details:**
- TriageService actor calling Claude API for 5-category thought classification
- Fire-and-forget triage after capture for instant UX
- Category pill display with color coding and confidence percentage
- User override sets confidence to 1.0; auto-dismiss pauses during picker

### Phase 4: Dashboard

**Goal**: Central SwiftUI dashboard displaying all captured entries with search, and settings UI
**Depends on**: Phase 3
**Plans**: 2 plans

Plans:
- [x] 04-01: Dashboard window with entry list, category views, and FTS5 search
- [x] 04-02: Settings UI for data source configuration

### Phase 5: Voice & Image Capture

**Goal**: Voice recording with on-device transcription and photo/image capture with AI-generated descriptions
**Depends on**: Phase 3
**Plans**: 3 plans

Plans:

### Phase 6: Evolved Daily Brief

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**

- Used `.target` for JarvisCore (library, not executable)
- GRDB 7.10.0 (latest stable 7.x)
- Carbon RegisterEventHotKey over NSEvent.addGlobalMonitorForEvents (no Accessibility permissions)
- ThoughtStore.update() without inout for Swift 6 cross-actor compliance
- Fire-and-forget triage from capture for instant UX
- SFSpeechRecognizer over WhisperKit (Intel Mac CoreML crash avoidance)
- Audio/image import via dashboard toolbar (floating NSPanel + NSOpenPanel incompatible)
- GoogleCalendarService in JarvisCore for cross-target access

**Issues Resolved:**

- 8+ Swift 6 Sendable/actor isolation blocking issues resolved across all phases
- CategoryFilter enum fix for SwiftUI List nil tag selection
- NSApp.setActivationPolicy(.regular) fix for keyboard focus in MenuBarExtra apps
- WhisperKit → SFSpeechRecognizer technology swap for Intel Mac compatibility
- Settings window sizing iterations (500x400 → 600x420 → 700x460)

**Issues Deferred:** None

**Technical Debt Incurred:** None noted
