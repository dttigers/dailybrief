# Milestone v2.1: Server Deployment

**Status:** ✅ SHIPPED 2026-04-05
**Phases:** 37-44
**Total Plans:** 13

## Overview

Deploy Vigil Core API to a production server with PostgreSQL, authentication, and HTTPS — enabling the G2 glasses and future mobile clients to connect from anywhere.

## Phases

### Phase 37: PostgreSQL Migration

**Goal**: Replace better-sqlite3 with PostgreSQL via Drizzle ORM; define schema, migrations, and update all route queries
**Depends on**: v2.0 complete
**Plans**: 4 plans

Plans:
- [x] 37-01: Drizzle Schema & Database Setup (Wave 1)
- [x] 37-02: Thoughts CRUD + FTS Route Migration (Wave 2, parallel)
- [x] 37-03: Tags, Links & Bulk Route Migration (Wave 2, parallel)

## Milestone Summary

**Key Decisions:**
- Decision: Drizzle ORM over raw SQL (Rationale: type-safe queries, migration tooling)
- Decision: Railway over self-hosted VPS (Rationale: managed Postgres, zero DevOps)
- Decision: SHA-256 hashed bearer tokens with vk_ prefix (Rationale: simple auth for API-to-API)
- Decision: Keep Railway default domain (Rationale: HTTPS included, custom domain unnecessary)
- Decision: In-memory rate limiting over Redis (Rationale: single-instance, no external dependency)
- Decision: Programmatic migrations on deploy (Rationale: idempotent, no manual step)

**Issues Resolved:**
- CORS preflight failing (middleware ordering — CORS before auth)
- Mac app API decoding errors (response shape mismatch after PostgreSQL migration)
- G2 task count wrong (API response format change)
- Smoke test tag assertion failing (JSONB array vs string comparison)
- Triage endpoint Anthropic auth errors in smoke test (treated as non-blocking warnings)

**Issues Deferred:**
- Mac app full E2E verification deferred (needs physical Mac running production config)
- G2 hardware testing deferred (awaiting physical Even G2 glasses)
- Custom domain setup deferred (Railway default domain sufficient for now)

**Technical Debt Incurred:**
- In-memory rate limiting won't survive restarts or scale to multiple instances
- Local SQLite fallback mode still exists but untested since PostgreSQL migration
- Smoke test uses hardcoded API key rather than environment variable
