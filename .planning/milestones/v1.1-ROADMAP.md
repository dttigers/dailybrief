# Milestone v1.1: Always On

**Status:** ✅ SHIPPED 2026-04-03
**Phases:** 8-13
**Total Plans:** 16

## Overview

Transform Jarvis from a terminal-launched tool into an always-running background assistant with smarter AI features and multi-Mac sync. Added LaunchAgent auto-start, passive folder watching, sports UX improvements, AI-powered insights, CloudKit sync, and integration polish.

## Phases

### Phase 8: Launch Agent

**Goal**: Convert from terminal-dependent CLI to macOS Launch Agent. Auto-start at login, run silently in background, menu bar icon always present.
**Depends on**: v1.0 MVP complete
**Plans**: 3 plans

Plans:
- [x] 08-01: Install script & binary discovery
- [x] 08-02: Built-in brief scheduler
- [x] 08-03: Consolidate & verify

**Details:**
Created install.sh for binary deployment to ~/.local/bin, implemented BriefScheduler with daily 6 AM timer, consolidated to single LaunchAgent architecture with monitor app handling both presence and scheduling. Removed CLI LaunchAgent in favor of app-managed scheduling.

### Phase 9: Folder Watching

**Goal**: Monitor dedicated folders for new audio and photo files. Auto-transcribe audio and process images daily.
**Depends on**: Phase 8
**Plans**: 2 plans

Plans:
- [x] 09-01: FolderWatcherService + Config
- [x] 09-02: App lifecycle wiring + Settings UI

**Details:**
Built FolderWatcherService with DispatchSource-based directory monitoring for passive audio/image capture. Added FolderWatchingConfig to AppConfig and a Folders settings tab for enabling/configuring watched paths. Integrated into AppDelegate lifecycle.

### Phase 10: Sports UI + Daily Brief

**Goal**: Replace raw numeric team/sport IDs in settings with team name picker. Improve daily brief content, formatting, and add new sections.
**Depends on**: Phase 9
**Plans**: 2 plans

Plans:
- [x] 10-01: MLBTeamData model + team name picker in Settings
- [x] 10-02: Dynamic PDF sports section with config-driven names

**Details:**
Developed MLBTeamData model with all 30 MLB teams. Replaced numeric ID steppers with user-friendly team name picker dropdown. Updated PDF renderer to dynamically display configured team and division names instead of hardcoded values.

### Phase 11: Smart Suggestions

**Goal**: AI proactively surfaces insights — pattern recognition, action prompts, and thought connections.
**Depends on**: Phase 10
**Plans**: 1 plan

Plans:
- [x] 11-01: InsightService actor + Insight model + InsightsConfig

**Details:**
Built Insight model and InsightService actor for Claude-powered thought analysis with four insight types (pattern, connection, actionPrompt, trend). Added backward-compatible InsightsConfig to AppConfig.

### Phase 12: Cloud Sync

**Goal**: Implement CloudKit-based database sync across multiple Macs.
**Depends on**: Phase 11
**Plans**: 4 plans

Plans:
- [x] 12-01: Sync metadata fields + database migration
- [x] 12-02: CloudKit infrastructure + record mapping
- [x] 12-03: SyncService actor (push/pull/conflict resolution)
- [x] 12-04: App lifecycle wiring + Settings UI

**Details:**
Complete CloudKit sync architecture: added sync metadata fields (syncStatus, cloudKitRecordID, lastSyncedAt, needsSync) to Thought model with v2 database migration. Created CloudKitManager actor with custom zone setup and CKRecord mapping. Built SyncService actor with bidirectional push/pull and last-write-wins conflict resolution. Integrated periodic sync with UI controls in Settings.

**Goal**: Tie everything together — edge cases, refinement, cross-feature integration testing, UX polish.
**Depends on**: Phase 12
**Plans**: 4 plans

Plans:

## Milestone Summary

**Decimal Phases:**
None — all phases were sequential (8-13).

**Key Decisions:**

- Decision: LaunchAgent over Login Items (Rationale: More control over lifecycle, single architecture)
- Decision: DispatchSource for folder watching (Rationale: Low overhead, immediate file detection)
- Decision: MLBTeamData model with all 30 teams (Rationale: Better UX than raw numeric IDs)
- Decision: CloudKit over Supabase for sync (Rationale: Native Apple integration, no server costs, privacy)
- Decision: Last-write-wins conflict resolution (Rationale: Simple, predictable for single-user sync)
- Decision: CoreGraphics for image conversion (Rationale: No external dependencies, handles HEIC/TIFF/BMP)

**Issues Resolved:**

- Sports settings UX — replaced confusing numeric steppers with team name dropdowns
- CLI/app LaunchAgent conflict — consolidated to single app-managed architecture
- NSOpenPanel incompatibility with NSPanel — moved import to dashboard toolbar
- Settings window too narrow for new tabs — widened to 850px

**Issues Deferred:**

- Pattern recognition for therapy prep (deferred to future milestone)
- Undo/redo for thought editing
- Thought tagging beyond AI categories
- Brief history browsing

**Technical Debt Incurred:**

- SyncService uses simple periodic polling; could upgrade to CKSubscription for push notifications
- InsightService lookback is time-based only; could add category/tag filtering
- Folder watching doesn't handle nested subdirectories
