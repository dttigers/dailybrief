# Milestone v1.2: Daily Driver

**Status:** ✅ SHIPPED 2026-04-03
**Phases:** 14-18
**Total Plans:** 14

## Overview

Make Jarvis reliable and practical for daily use — batch workflows, task tracking, multi-sport coverage, and stability. Fixed LaunchAgent reliability, added multi-file upload, full task status workflow, multi-sport support (MLB/NFL/NBA/NHL), configurable IMAP email, and polish.

## Phases

### Phase 14: LaunchAgent Fix & Folder Cleanup

**Goal**: Debug and fix LaunchAgent exit code -4 for reliable auto-start. Add auto-delete of watched folder files after successful processing.
**Depends on**: v1.1 Always On complete
**Plans**: 2 plans

Plans:
- [x] 14-01: Fix LaunchAgent plist + install script + diagnostic logging
- [x] 14-02: Auto-delete watched folder files after processing

**Details:**
Hardened LaunchAgent plist with Aqua session constraints and KeepAlive dict form to prevent restart loops. Added NSLog startup breadcrumbs for crash diagnostics with stale "in progress" detection. Implemented opt-in auto-delete toggle (default-off) for watched folder files after successful processing.

### Phase 15: Multi-File Upload

**Goal**: Batch import multiple photos and audio files from the dashboard toolbar via file picker or drag & drop.
**Depends on**: Phase 14
**Plans**: 1 plan

Plans:
- [x] 15-01: Multi-file upload from dashboard toolbar

**Details:**
Multi-select file picker with batch import pipeline and drag & drop support. Sequential processing to avoid API rate limits. Per-file progress tracking and error collection. Non-native image formats auto-convert to JPEG before processing.

### Phase 16: Task Status Workflow

**Goal**: Add status tracking to tasks and work orders. Support open → in progress → done workflow with UI controls.
**Depends on**: Phase 15
**Plans**: 4 plans

Plans:
- [x] 16-01: Thought TaskStatus model + DB migration + ThoughtStore + CloudKit sync
- [x] 16-02: WorkOrder 3-state CompletionStore + CLI commands
- [x] 16-03: Dashboard UI status controls + filtering
- [x] 16-04: PDF status rendering for tasks and work orders

**Details:**
Added TaskStatus enum (open/inProgress/done) with database migration, ThoughtStore query methods, and CloudKit sync mapping. Extended CompletionStore with 3-state WorkOrderStatus and --status flag for CLI. Built interactive dashboard UI with click-to-cycle status icons, strikethrough done tasks, and sidebar sub-filters with counts. Status-aware PDF rendering with checkboxes sorting inProgress > open > done for both tasks and work orders.

### Phase 17: Multi-Sport Support

**Goal**: Extend sports beyond MLB to include NFL, NBA, and NHL with team pickers, ESPN API integration, and adaptive PDF layout.
**Depends on**: Phase 16
**Plans**: 4 plans

Plans:
- [x] 17-01: Team data models (NFL, NBA, NHL) + multi-sport AppConfig
- [x] 17-02: ESPNSportsService + DailyBriefData wiring
- [x] 17-03: Multi-sport Settings UI with per-league toggles/pickers
- [x] 17-04: Multi-sport PDF rendering on Page 2

**Details:**
Created NFL/NBA/NHL team data models with ESPN-verified IDs (plan data was significantly wrong for NHL). Refactored SportsConfig for multi-sport support with backward compatibility. Built ESPNSportsService actor with concurrent TaskGroup-based data fetching. Settings UI with per-sport enable toggles and division-grouped team pickers. PageTwoRenderer with adaptive layout — full mode for 1 sport, compact for 2+, very compact for 3-4 sports.

### Phase 18: Polish & Integration

**Goal**: Tie everything together — edge cases, refinement, cross-feature integration testing, UX polish.
**Depends on**: Phase 17
**Plans**: 3 plans

Plans:
- [x] 18-01: Email/IMAP config rename + backward-compatible migration
- [x] 18-02: Shared ImageConversion utility + cross-feature audit
- [x] 18-03: Final build verification + v1.2 milestone closure

**Details:**
Renamed GmailConfig to EmailConfig with configurable IMAP host/port/TLS fields using dual CodingKeys for backward-compatible migration from old "gmail" key. Extracted shared ImageConversion utility from duplicated JPEG conversion code. Final build verification with zero errors/warnings across all three targets. Removed dead code cleanly.

## Milestone Summary

**Decimal Phases:**
None — all phases were sequential (14-18).

**Key Decisions:**

- Decision: Sequential batch processing for multi-file upload (Rationale: Avoid API rate limits)
- Decision: TaskStatus optional for non-task thoughts (Rationale: Not all thoughts are actionable)
- Decision: All work orders shown in PDF with visual de-emphasis (Rationale: Better than filtering done orders)
- Decision: ESPN team IDs verified against live API (Rationale: Plan data was significantly wrong for NHL)
- Decision: Division-based team presence matching for standings (Rationale: More reliable than divisionId)
- Decision: Adaptive PDF layout for multi-sport (Rationale: 1 sport = full, 2+ = compact, 3-4 = very compact)
- Decision: Dual CodingKeys for config rename (Rationale: Backward-compatible migration from "gmail" to "email")

**Issues Resolved:**

- LaunchAgent exit code -4 — fixed with Aqua session constraints and KeepAlive dict form
- Duplicate image conversion code — extracted shared ImageConversion utility
- Hardcoded Gmail config — replaced with configurable IMAP email settings
- Raw numeric sport IDs for NFL/NBA/NHL — team name pickers for all leagues
- No task status tracking — full open/in-progress/done workflow

**Issues Deferred:**

- Pattern recognition for therapy prep (deferred to future milestone)
- Undo/redo for thought editing
- Thought tagging beyond AI categories
- Brief history browsing
- CKSubscription push notifications (upgrade from polling)

**Technical Debt Incurred:**

- SyncService still uses periodic polling; CKSubscription upgrade deferred
- Folder watching doesn't handle nested subdirectories
- InsightService lookback is time-based only
