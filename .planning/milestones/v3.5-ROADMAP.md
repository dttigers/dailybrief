## Phase Details

### Phase 103: Capture Repair & Server Observability Foundations
**Goal**: The photo capture pipeline works correctly for all clients, and server exceptions are observable in PostHog
**Depends on**: Phase 102 (multi-user foundation in production)
**Requirements**: CAP-01, CAP-02, ANLY-01 (server half), AUTH-08
**Success Criteria** (what must be TRUE):
  1. A HEIC file dropped into the Mac watched folder (iCloud path) produces a thought with non-empty content — no silent failure
  2. A photo uploaded via `POST /v1/process-photo` returns a thought with a non-null category field (verified via curl before any code is written)
  3. Unhandled exceptions thrown in vigil-core routes appear in PostHog Cloud with stack traces (verified by intentionally triggering a test error)
  4. `GET /v1/me` returns `{ userId, email }` for a valid JWT — prerequisite for Phase 104
  5. Local `npm run dev` events do NOT appear in PostHog Cloud; Railway production events DO appear
**Plans**: 5 plans (00 wave-0 RED, 01 PostHog singleton, 02 HEIC convert + sync triage, 03 GET /v1/me, 04 wire-up + verification)

### Phase 104: PWA Auth UI & Browser Observability
**Goal**: PWA visitors can sign up and log in with email/password, and browser-side errors are tracked in PostHog
**Depends on**: Phase 103 (needs /v1/me for posthog.identify, PostHog server singleton for env consistency)
**Requirements**: AUTH-06, AUTH-07, ANLY-01 (browser half)
**Success Criteria** (what must be TRUE):
  1. New user signup form (email + password) → dashboard authenticated in single flow
  2. Existing user login → dashboard with email visible in header
  3. Wrong email and wrong password both display identical generic error (no enumeration)
  4. React render errors caught by error boundary appear in PostHog (verified by test throw)
  5. JWT stored in sessionStorage, not localStorage (DevTools Application tab confirmed)
**Plans**: 3 plans (01 RED scaffolds, 02 storage migration + AuthPage + posthog + ErrorBoundary, 03 identify wiring + Settings section)

### Phase 105: Product Events, API Metrics & User Identity
**Goal**: The capture funnel, API error rates, and user identity are visible in the PostHog dashboard
**Depends on**: Phase 103 + Phase 104 (both PostHog SDKs initialized first)
**Requirements**: ANLY-02, ANLY-03, ANLY-04
**Success Criteria** (what must be TRUE):
  1. After capture + triage, two server-side events with same userId: `thought_created` + `triage_completed`
  2. Photo upload produces `photo_uploaded` event filtered by userId
  3. Per-route API metrics (status, latency, route) populated for authenticated requests
  4. No event property contains user-generated string content (allowlist enums/booleans/numbers only)
**Plans**: 3 plans (01 BLOCKED_PROPERTY_NAMES guard + identifyUser, 02 metrics middleware + 5 capture-funnel events, 03 server-side identify in /v1/me)

### Phase 106: G2 Store Resubmit (Atomic)
**Goal**: All three Even Hub store rejection items are resolved and verified before a single resubmission is made
**Depends on**: Nothing (independent of all server/PWA work)
**Requirements**: G2-01, G2-02, G2-03
**Success Criteria** (what must be TRUE):
  1. Fresh simulator screenshots exist at native 576×288 with deterministic demo data
  2. Double-tap on G2 home shows host-rendered exit confirmation; non-home double-tap returns to home (no regression)
  3. G2 plugin WebView renders brand-compliant UI on 4-bit greyscale (unified VIGIL header, 1px greyscale body borders, Vigil-voice fallbacks); amended from original "Vigil brand colors and Inter font" wording per CONTEXT D-06 (teal/Inter cannot physically render on 4-bit greyscale)
  4. All three items verified in single session before `.ehpk` is uploaded — no partial resubmission
**Plans**: 5 plans (01 atomic-gate scaffold, 02 G2-02 host exit-confirm, 03 G2-03 unified header + greyscale + fallbacks, 04 G2-01 VITE_SCREENSHOT_MODE + DEMO constants, 05 hardware UAT + screenshots + ehpk pack)

**Hardware UAT outcome (2026-05-05):** All 3 gates verified on physical G2 hardware (firmware 2.2.0.28). 6 simulator/hardware divergences captured in `HARDWARE-DIVERGENCE.md` (containerName 16-char limit, list-container SCROLL swallowing, iPhone canvas absence as launcher-only, home body overflow, glyph fallout, device-status flutter). None blocking ship.
