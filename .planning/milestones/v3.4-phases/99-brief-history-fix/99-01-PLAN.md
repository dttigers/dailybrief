   // ── brief_pdfs table (Phase 99 — BYTEA storage for daily brief PDFs) ──────
   export const briefPdfs = pgTable(
     "brief_pdfs",
     {
       briefId: integer("brief_id")
         .primaryKey()
         .references(() => briefs.id, { onDelete: "cascade" }),

       byteLength: integer("byte_length").notNull(),
       createdAt: timestamp("created_at", { withTimezone: true })

     },
   );

Per D-01: PDF bytes live in Postgres BYTEA. Per D-02: sibling table (not column on briefs) so `SELECT * FROM briefs` never accidentally pulls MB of binary.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Drizzle schema → generated SQL | drizzle-kit translates TypeScript schema to raw SQL; any typo in schema propagates to prod DDL |
| Migration file → Railway boot | Dockerfile `node dist/db/migrate.js` runs migrations unconditionally on every boot; a malformed migration can crash the pod before the API starts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-99-01 | Tampering | schema.ts → generated SQL | mitigate | Task 2 verifies generated SQL contains the exact column names and FK before commit; acceptance_criteria greps enforce this |
| T-99-02 | Denial of Service | Railway boot on bad migration | mitigate | Task 2 applies the migration locally against dev DB BEFORE commit, catching SQL errors outside the prod boot path |
| T-99-03 | Information Disclosure | brief_pdfs bytes column | accept | Bytes are PDFs the user generated for themselves; no PII beyond what briefs already expose. FK cascade delete ensures orphan rows cannot linger after brief deletion |
| T-99-04 | Repudiation | missing audit trail on brief_pdfs writes | accept | `created_at` default now() provides a write timestamp; explicit audit log is out-of-scope for BRIEF-01 |
</threat_model>
