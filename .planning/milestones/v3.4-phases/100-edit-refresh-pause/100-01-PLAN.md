```typescript
  // Auto-refresh with edit-aware pause gate (Phase 100 / EDIT-01, D-01 D-02 D-06 D-08)
  useEffect(() => {
    // D-02: Set of thought ids currently being edited; gate refresh on size > 0
    const activeEdits = new Set<number>()
    let pollId: ReturnType<typeof setInterval> | null = null

    const startPoll = () => {
      if (pollId !== null) return
      pollId = setInterval(() => {
        if (activeEdits.size === 0) refetch()
      }, 30_000)
    }
    const stopPoll = () => {
      if (pollId !== null) {
        clearInterval(pollId)
        pollId = null
      }
    }

    const handleVisibility = () => {
      // D-06: visibilitychange is ALSO gated on active edits
      if (document.visibilityState === 'visible' && activeEdits.size === 0) {
        refetch()
      }
    }
    const handleCreated = () => {
      // D-06: vigil:thought-created is ALSO gated on active edits
      if (activeEdits.size === 0) refetch()
    }
    const handleEditStarted = (e: Event) => {
      const id = (e as CustomEvent<{ id: number }>).detail?.id
      if (typeof id !== 'number') return
      activeEdits.add(id)
      // D-09: clearInterval during pause; restart on resume so resumed interval
      // is a full 30s from the resume moment (not a leftover partial tick)
      stopPoll()
    }
    const handleEditEnded = (e: Event) => {
      const id = (e as CustomEvent<{ id: number }>).detail?.id
      if (typeof id !== 'number') return
      const hadEntry = activeEdits.delete(id)
      // D-08: only the N→0 transition (last edit ending) triggers catch-up + restart.
      // Stray end without matching start (hadEntry=false) is a no-op per Test 5.
      if (hadEntry && activeEdits.size === 0) {
        refetch()
        startPoll()
      }
    }

    document.addEventListener('visibilitychange', handleVisibility)
    window.addEventListener('vigil:thought-created', handleCreated)
    window.addEventListener('vigil:edit-started', handleEditStarted)
    window.addEventListener('vigil:edit-ended', handleEditEnded)
    startPoll()

    return () => {
      document.removeEventListener('visibilitychange', handleVisibility)
      window.removeEventListener('vigil:thought-created', handleCreated)
      window.removeEventListener('vigil:edit-started', handleEditStarted)
      window.removeEventListener('vigil:edit-ended', handleEditEnded)
      stopPoll()
    }
  }, [refetch])
```

Notes on the exact shape:

1. Per D-02, use `Set<number>`. Closure-scoped inside the effect is fine (no external read access needed). A `useRef` is NOT required because the Set lifecycle matches the effect lifecycle — if `refetch` identity changes the effect re-runs and we build a fresh Set (refetch is `useCallback` with `[]` deps so its identity is stable, the effect re-runs once on mount).

2. Per D-08, the catch-up + interval reset fires ONLY on the N→0 transition — not on every edit-ended. The `hadEntry && activeEdits.size === 0` guard enforces this. If `hadEntry` is false (stray end), the Set was unchanged and we do nothing (Test 5 contract).

3. Per D-09, the implementation is the `clearInterval` on start / `setInterval` on end strategy. The alternative (single always-on interval checking the flag) is rejected because it doesn't reset the 30s cadence on resume — after resume you'd get a poll at whatever fraction of 30s remained from the pre-pause interval.

4. Do NOT lift `activeEdits` into module scope or a singleton. Each useThoughts mount owns its own Set. For the PWA there's only ever one useThoughts mount active at a time (the Thoughts page), so the Set scope is fine.

5. `CustomEvent<{id: number}>` cast: the dispatched events carry `{id: number}` per D-02 / Task 3. The defensive `typeof id !== 'number'` guard protects against future callers accidentally dispatching without detail.
