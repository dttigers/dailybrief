## New UI Elements — Exact Specifications

### Chat Icon Button on ThoughtRow

Position: Inline in the action button group at top-right of each row, after the re-triage `↻` button, before the relative timestamp.

```
[Category badge] [Task status badge?]    [♡] [↻] [💬] [2h ago]
```

Styles (exact Tailwind classes):
```
text-gray-400/50 hover:text-teal-400 transition-colors cursor-pointer
```

Icon: `💬` (Unicode U+1F4AC, Speech Balloon emoji). Matches the pattern of `↻` and `♥/♡` — Unicode text characters, no SVG required.

`title` attribute: `"Chat about this thought"` (matches the tooltip pattern of existing buttons).

Size: inherits `text-base leading-none` from surrounding `text-xs` context — visually comparable to the other action icons.

Disabled state: none required. Chat is always available on any thought.

### Loading State — First AI Response

No new component. The existing `isLoading` render in ChatPage (`animate-pulse` "..." bubble) handles this automatically. The executor must verify this renders immediately when ChatPage mounts from a thought (since `sendMessage` is called on mount).

### Auto-Send Transition

The thought text appears as a user bubble (teal-600 bg) immediately — `sendMessage` adds it to `messages` synchronously before the API call. The AI "..." loading bubble follows. No skeleton or intermediate state needed.

## Interaction Contract

### Chat Button Tap (ThoughtRow)

1. User taps `💬` on a ThoughtRow.
2. `onChat()` closure fires (no args — thought data captured in ThoughtsPage closure).
3. `navigate('/chat', { state: { thoughtText, thoughtId } })` executes immediately.
4. No confirmation, no loading indicator on the button itself. Navigation is synchronous.

### ChatPage Mount from Thought

1. ChatPage mounts. `useLocation().state` contains `{ thoughtText, thoughtId }`.
2. `clearChat()` fires synchronously — resets `activeSessionId` to null, `messages` to `[]`.
3. `sendMessage(thoughtText)` fires — adds user message to `messages`, calls API, creates new session.
4. `window.history.replaceState({}, '')` fires — clears location state to prevent replay on tab switch.
5. AI response arrives, loading state clears. User sees thought as first user message, AI response below.
6. User can continue multi-turn conversation with the standard input form.

### Chat Button Visibility

Always visible on every ThoughtRow (not conditional on thought category, task status, or any other property). The `onChat` prop is optional — rows without `onChat` prop simply do not render the button. In Phase 98, all ThoughtsPage rows receive `onChat`.

## Accessibility Notes

- Chat button: `title="Chat about this thought"` provides tooltip for mouse users. For screen readers, add `aria-label="Chat about this thought"` in implementation (title alone is insufficient for SR).
- `💬` emoji has implicit Unicode name "SPEECH BALLOON" — screen readers will announce it. Combined with `aria-label`, this is acceptable.
- All existing ThoughtRow buttons use `title` only — match the existing pattern for consistency. Stretch goal: add `aria-label` to all action buttons while touching the file.
- ChatPage auto-send: no focus management change needed. The input field already receives focus naturally after mount.
