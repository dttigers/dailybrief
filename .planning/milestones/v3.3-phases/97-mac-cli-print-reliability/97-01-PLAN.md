2. Add a private static method `checkPrinterReachable` inside `enum PrintService` (per D-06):
```swift
private static func checkPrinterReachable(_ printerName: String) throws {
    let task = Process()
    task.executableURL = URL(fileURLWithPath: "/usr/sbin/lpstat")
    task.arguments = ["-p", printerName]
    task.standardOutput = Pipe()
    task.standardError = Pipe()
    try task.run()
    task.waitUntilExit()
    if task.terminationStatus != 0 {
        Logger.error("Printer not reachable: \(printerName)")
        throw PrintError.printerNotReachable(printerName)
    }
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI -> vigil-core API | CLI fetches PDF and triggers generation over HTTPS with Bearer auth |
| CLI -> CUPS/lpr | CLI invokes system print command with user-configured printer name |
| CLI -> lpstat | CLI queries printer status via system command |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-97-01 | Injection | PrintService lpr args | mitigate | printer_name comes from user's own config.json (not untrusted input); Process API uses argument array (not shell string), preventing injection |
| T-97-02 | Injection | Doctor lpstat args | mitigate | Same as T-97-01 — printerName from config.json, passed as Process argument array element |
| T-97-03 | Denial of Service | POST /v1/brief/generate on 404 | accept | CLI only calls regenerate once on 404 (no retry loop); API already rate-limited by Railway; this is the user's own API key hitting their own server |
| T-97-04 | Information Disclosure | lpr error output | accept | Error messages logged locally only (~/Library/Logs/DailyBrief/); no PII in printer error codes |
</threat_model>
