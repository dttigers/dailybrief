  **1. Extend PDFConfig in AppConfig.swift** (inside the existing `AppConfig.PDFConfig` struct):

  Add these fields with defaults matching current hardcoded values:
  ```
  paperSize: String = "notebook"  // "notebook" (current 3.75x7.5), "a5", "half-letter", "letter", "custom"
  customWidthInches: Double = 3.75
  customHeightInches: Double = 7.5
  marginPoints: Double = 12.0
  fontScale: Double = 1.0  // 0.75 to 1.5 multiplier
  enabledSections: [String] = ["workOrders", "todo", "calendar", "sports", "affirmation", "thoughts", "insights", "therapyPrep"]
  ```

  Use `decodeIfPresent` with defaults for backward compatibility — existing configs without these fields should work unchanged, producing the exact same PDF as before.

  **Paper size presets (all in points, 72 dpi):**
  - "notebook": page=612x792 (letter), content=270x540 (3.75"x7.5" centered) — CURRENT DEFAULT
  - "a5": page=420x595, content=380x555 (full page minus margins)
  - "half-letter": page=396x612, content=356x572 (full page minus margins)
  - "letter": page=612x792, content=572x752 (full page minus margins)
  - "custom": page computed from customWidthInches/customHeightInches * 72, content = page minus 2*margin

  **2. Create PDFLayout struct in PDFStyles.swift:**

  Add a `struct PDFLayout: Sendable` above the existing `PDFStyles` enum. This struct holds all computed layout dimensions:
  ```swift
  struct PDFLayout: Sendable {
      let pageWidth: CGFloat
      let pageHeight: CGFloat
      let contentWidth: CGFloat
      let contentHeight: CGFloat
      let contentX: CGFloat
      let contentY: CGFloat
      let margin: CGFloat
      let innerPadding: CGFloat
      let titleSize: CGFloat
      let headerSize: CGFloat
      let bodySize: CGFloat
      let smallSize: CGFloat
      let tinySize: CGFloat
      let tableRowHeight: CGFloat
      let tableHeaderHeight: CGFloat
      let checkboxSize: CGFloat
      let noteLineSpacing: CGFloat
      let enabledSections: Set<String>
  }
  ```

  Add `static func layout(from config: AppConfig.PDFConfig) -> PDFLayout` that computes all values based on paper size preset and font scale. For "notebook" preset, values must exactly match current PDFStyles hardcoded values (regression-safe). Font sizes are base values * fontScale. Row heights scale proportionally with font scale.

  Keep existing PDFStyles enum intact with its color constants, font creation methods, dash pattern, and border line width — these don't need to be configurable. Update font methods to accept a size parameter: `titleFont(size: CGFloat)` instead of using the static constant, so renderers can pass `layout.titleSize`.
  </action>
