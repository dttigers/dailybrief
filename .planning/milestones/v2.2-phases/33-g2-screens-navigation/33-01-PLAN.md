Create `vigil-g2-plugin/src/navigation.ts` — screen navigation state machine.

**Screen enum and state:**
Define screens as a const object (same pattern as ContainerId — avoid const enum due to erasableSyntaxOnly):
```
export const Screen = {
  HOME: 'home',
  WORK_ORDERS: 'work-orders',
  AFFIRMATION: 'affirmation',
} as const
export type ScreenName = typeof Screen[keyof typeof Screen]
```

Navigation order: HOME → WORK_ORDERS → AFFIRMATION → HOME (circular).
- Swipe down (SCROLL_BOTTOM_EVENT) = next screen
- Swipe up (SCROLL_TOP_EVENT) = previous screen
- Double tap (DOUBLE_CLICK_EVENT) = return to HOME

**Exports:**
- `currentScreen: ScreenName` (module-level state)
- `getNextScreen(current: ScreenName): ScreenName`
- `getPrevScreen(current: ScreenName): ScreenName`
- `navigateTo(screen: ScreenName, bridge: EvenAppBridge): Promise<void>` — calls the appropriate screen builder function and `bridge.rebuildPageContainer()`. For now, HOME uses the existing `buildHomeScreen()`, WORK_ORDERS and AFFIRMATION are placeholders that render simple text containers saying "Work Orders - Coming Soon" / "Affirmation - Coming Soon" (Plan 02 replaces these).
- `handleNavEvent(eventType: OsEventTypeList, bridge: EvenAppBridge): Promise<void>` — maps SCROLL_TOP → prev, SCROLL_BOTTOM → next, DOUBLE_CLICK → home. Updates currentScreen and calls navigateTo.

**Import from SDK:** `RebuildPageContainer`, `TextContainerProperty`, `OsEventTypeList` and the bridge type.

**Update constants.ts:** Add container IDs for new screens (WORK_ORDERS_HEADER: 4, WORK_ORDERS_LIST: 5, WORK_ORDERS_FOOTER: 6, AFFIRMATION_HEADER: 7, AFFIRMATION_BODY: 8, AFFIRMATION_FOOTER: 9). Keep existing HOME_* IDs.

**Update main.ts:**
- Import `handleNavEvent` from navigation.ts
- In the `onEvenHubEvent` callback, add handling for `listEvent` and `sysEvent`:
  - If `event.listEvent` has eventType SCROLL_TOP, SCROLL_BOTTOM, or DOUBLE_CLICK → call `handleNavEvent(eventType, bridge)`
  - If `event.sysEvent` has eventType SCROLL_TOP, SCROLL_BOTTOM, or DOUBLE_CLICK → call `handleNavEvent(eventType, bridge)`
  - Keep existing FOREGROUND_ENTER/EXIT handling
- The initial `createStartUpPageContainer` call stays (home screen on startup)

Do NOT change the home screen builder or its file — that stays as-is for Plan 02.
