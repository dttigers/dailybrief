<task type="auto">
  <name>Task 2: Add 60-second auto-refresh and foreground refresh</name>
  <files>vigil-g2-plugin/src/main.ts, vigil-g2-plugin/src/navigation.ts</files>
  <action>
**Add refresh function to navigation.ts:**

Export a `refreshCurrentScreen(bridge: EvenAppBridge): Promise<void>` function that:
1. Reads `currentScreen`
2. Calls `navigateTo(currentScreen, bridge)` — this already fetches fresh API data and rebuilds the page

**Update main.ts — 60s refresh timer:**

After initial setup in `init()`:
- Start a `setInterval` at 60_000ms (60 seconds) that calls `refreshCurrentScreen(bridge)`
- Store the interval ID in a module-level variable

**Update main.ts — foreground refresh:**

In the existing `FOREGROUND_ENTER_EVENT` handler:
- Call `refreshCurrentScreen(bridge)` immediately when app enters foreground
- This gives instant-fresh data when user switches back to Vigil

**Update main.ts — foreground exit cleanup:**

In `FOREGROUND_EXIT_EVENT`:
- Optionally clear the interval timer on background (save battery/network)
- Restart it on next FOREGROUND_ENTER

Pattern:
```
let refreshTimer: ReturnType<typeof setInterval> | null = null

function startRefreshTimer(bridge) {
  stopRefreshTimer()
  refreshTimer = setInterval(() => refreshCurrentScreen(bridge), 60_000)
}

function stopRefreshTimer() {
  if (refreshTimer) { clearInterval(refreshTimer); refreshTimer = null }
}
```

Start timer in init() after first render. Stop on FOREGROUND_EXIT. Start + immediate refresh on FOREGROUND_ENTER.

Do NOT add retry logic or exponential backoff — the API client already returns fallback data on error. Keep it simple.
