<tasks>

<task type="auto">
  <name>Task 1: Create LaunchAgent for Vigil Core API auto-start</name>
  <files>LaunchAgent/com.jamesonmorrill.vigilcore.plist</files>
  <action>
    Create a LaunchAgent plist for the Vigil Core Node.js API server, following the same pattern as the existing DailyBriefMonitor plist.

    Key configuration:
    - Label: com.jamesonmorrill.vigilcore
    - ProgramArguments: /usr/local/bin/node, with the built dist/index.js path (use absolute path to vigil-core/dist/index.js)
    - WorkingDirectory: absolute path to vigil-core/ (needed for better-sqlite3 to find the DB)
    - RunAtLoad: true (start on login)
    - KeepAlive with SuccessfulExit: false (restart on crash, not on clean exit)
    - LimitLoadToSessionType: Aqua (GUI sessions only)
    - ProcessType: Background (not Interactive — it's a headless server)
    - StandardOutPath: ~/Library/Logs/DailyBrief/vigilcore-stdout.log
    - StandardErrorPath: ~/Library/Logs/DailyBrief/vigilcore-stderr.log
    - EnvironmentVariables: PATH must include /usr/local/bin (for node), and ANTHROPIC_API_KEY — read the current key from ~/.config/dailybrief/config.json's ai.claude_api_key field and include it. Also set PORT=3001 explicitly.

    Before creating the plist, run `npm run build` in vigil-core/ to ensure dist/index.js exists.

    After creating the plist, copy it to ~/Library/LaunchAgents/ and load it with `launchctl load ~/Library/LaunchAgents/com.jamesonmorrill.vigilcore.plist`.

    Do NOT use launchctl bootstrap (older load syntax is correct for per-user agents on current macOS).
  </action>
  <verify>
    - launchctl list | grep vigilcore shows the agent loaded
    - curl http://localhost:3001/v1/health returns 200 with ok status
    - Log files created at ~/Library/Logs/DailyBrief/vigilcore-stdout.log
  </verify>
  <done>Vigil Core API auto-starts on login, restarts on crash, and is currently running on port 3001</done>
</task>
