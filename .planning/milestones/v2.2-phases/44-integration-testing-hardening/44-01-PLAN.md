<task type="auto">
  <name>Task 1: Add rate limiting, timeout, and security headers middleware</name>
  <files>vigil-core/src/middleware/rate-limit.ts, vigil-core/src/index.ts</files>
  <action>
1. Create `vigil-core/src/middleware/rate-limit.ts` with a simple in-memory sliding-window rate limiter:
   - Track requests per IP using a Map<string, {count, windowStart}>
   - Default: 100 requests per 60-second window (configurable via RATE_LIMIT_MAX and RATE_LIMIT_WINDOW_MS env vars)
   - Return 429 Too Many Requests with JSON `{ error: "Rate limit exceeded", retryAfter: N }` when exceeded
   - Include `Retry-After` header in 429 responses
   - Clean up stale entries periodically (every 60s) to prevent memory leak
   - Export as Hono MiddlewareHandler

2. In `vigil-core/src/index.ts`, add three middleware layers AFTER cors but BEFORE auth:
   - `secureHeaders()` from `hono/secure-headers` — adds X-Content-Type-Options, X-Frame-Options, etc.
   - `timeout(30_000)` from `hono/timeout` — 30-second request timeout with 408 response
   - The custom rate limiter from step 1

   Middleware order: cors → secureHeaders → timeout → rateLimiter → health (no auth) → bearerAuth → protected routes

   Do NOT add `bodyLimit` — the existing routes already handle body parsing and the API accepts images via describe-image which can be large.
  </action>
  <verify>
    cd vigil-core && npm run build — TypeScript compiles without errors
  </verify>
  <done>Rate limiter, timeout, and secure headers middleware applied. Build passes.</done>
</task>
