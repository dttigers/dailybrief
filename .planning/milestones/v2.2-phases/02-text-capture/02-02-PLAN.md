<task type="auto">
  <name>Task 1: Create GlobalHotKey utility using Carbon API</name>
  <files>Sources/DailyBriefMonitor/GlobalHotKey.swift</files>
  <action>
Create a GlobalHotKey class that wraps Carbon's `RegisterEventHotKey` / `UnregisterEventHotKey` for system-wide keyboard shortcut registration. This approach does NOT require Accessibility permissions (unlike NSEvent.addGlobalMonitorForEvents).

```
import Carbon
import AppKit

final class GlobalHotKey: @unchecked Sendable {
    private var hotKeyRef: EventHotKeyRef?
    private var eventHandler: EventHandlerRef?
    private let callback: () -> Void

    init(keyCode: UInt32, modifiers: UInt32, callback: @escaping () -> Void)
    deinit — calls unregister()

    func register() — registers the hotkey with Carbon
    func unregister() — unregisters if registered
}
```

Implementation details:

1. Store the callback and key configuration
2. In `register()`:
   - Create an `EventHotKeyID` with a unique signature (use `OSType` from "JRVS" — four chars) and id 1
   - Install a Carbon event handler for `kEventClassKeyboard` / `kEventHotKeyPressed` using `InstallEventHandler(GetApplicationEventTarget(), handler, 1, &eventSpec, pointer, &eventHandler)`
   - The handler callback needs access to the GlobalHotKey instance. Use `Unmanaged<GlobalHotKey>.passUnretained(self).toOpaque()` as the userData pointer. In the C handler function, recover with `Unmanaged<GlobalHotKey>.fromOpaque(userData).takeUnretainedValue()` and call its callback on `DispatchQueue.main.async`.
   - Call `RegisterEventHotKey(keyCode, modifiers, hotKeyID, GetApplicationEventTarget(), 0, &hotKeyRef)`

3. In `unregister()`:
   - If hotKeyRef is non-nil, call `UnregisterEventHotKey(hotKeyRef!)`
   - If eventHandler is non-nil, call `RemoveEventHandler(eventHandler!)`
   - Set both to nil

4. Provide a static convenience method or constants for common modifier combos:
   - `static let cmdShiftModifiers: UInt32 = UInt32(cmdKey | shiftKey)` (Carbon modifier flags)
   - Note: Carbon uses `cmdKey`, `shiftKey`, `optionKey`, `controlKey` from Carbon.HIToolbox

5. The C function handler must be a top-level function (or static method) since it's used as a C function pointer. Define it outside the class:
```swift
private func hotKeyHandler(
    nextHandler: EventHandlerCallRef?,
    event: EventRef?,
    userData: UnsafeMutableRawPointer?
) -> OSStatus {
    guard let userData else { return OSStatus(eventNotHandledErr) }
    let hotKey = Unmanaged<GlobalHotKey>.fromOpaque(userData).takeUnretainedValue()
    DispatchQueue.main.async {
        hotKey.callback()
    }
    return noErr
}
```

Key code for "J" is `0x26` (kVK_ANSI_J from Carbon.HIToolbox.Events).

Do NOT use NSEvent.addGlobalMonitorForEvents — it requires Accessibility permissions granted in System Settings, which is fragile and requires user action. Carbon RegisterEventHotKey works without extra permissions.
