  <name>Task 1: Add edit state management and undo support to DashboardViewModel</name>
  <files>Sources/DailyBriefMonitor/Dashboard/DashboardViewModel.swift</files>
  <action>
Add expand/collapse state to DashboardViewModel:
- `expandedThoughtIds: Set<Int64>` — tracks which thoughts are expanded to show full content
- `toggleExpanded(_ thought: Thought)` — toggles a thought's id in/out of the set

Add editing state properties to DashboardViewModel:
- `editingThoughtId: Int64?` — tracks which thought is being edited (nil = none)
- `editedContent: String` — the in-progress edited text

Add methods:
- `startEditing(_ thought: Thought)` — sets editingThoughtId and editedContent to thought's current content, also adds thought to expandedThoughtIds so the full text is visible while editing
- `saveEdit(undoManager: UndoManager?)` — validates editedContent is non-empty (trimmed), fetches thought from store, saves original content for undo, updates thought.content and thought.modifiedAt via `store.update()`, registers undo action with UndoManager that restores the original content (calling saveEdit again with the old content swapped in), clears editing state, refreshes thoughts list
- `cancelEdit()` — clears editingThoughtId and editedContent without saving

For the undo registration pattern:
```
undoManager?.registerUndo(withTarget: self) { [oldContent, thoughtId] vm in
    Task { @MainActor in
        vm.editedContent = oldContent
        vm.editingThoughtId = thoughtId
        // Perform the reverse save
        ...
    }
}
undoManager?.setActionName("Edit Thought")
```

The undo action should restore the previous content to the database and register a redo. The simplest correct approach: register undo that calls a private `applyEdit(id:content:undoManager:)` method which both saves and registers the reverse undo. Both saveEdit and undo use this same method.

Do NOT add a separate UndoManager — use the one passed from the SwiftUI environment.

**ThoughtRowView changes:**

Add new properties:
- `isExpanded: Bool` — whether this row shows full content
- `isEditing: Bool` — whether this row is in edit mode
- `editedContent: Binding<String>` — bound to ViewModel's editedContent
- `onToggleExpand: (() -> Void)?` — callback to toggle expand/collapse
- `onStartEdit: (() -> Void)?` — callback to enter edit mode
- `onSaveEdit: (() -> Void)?` — callback to save
- `onCancelEdit: (() -> Void)?` — callback to cancel

In the content HStack, conditionally show:
- When NOT editing: existing `Text(thought.content)` with:
  - `.lineLimit(isExpanded ? nil : 2)` — remove line limit when expanded (currently hardcoded to 2)
  - `.truncationMode(.tail)` (keep existing)
  - Single click (`.onTapGesture(count: 1)`) to toggle expand/collapse
  - Double click (`.onTapGesture(count: 2)`) to trigger `onStartEdit`
  - Important: place double-click gesture BEFORE single-click so SwiftUI distinguishes them
- When editing: `TextEditor(text: editedContent)` with:
  - `.font(.body)` to match existing text style
  - `.frame(minHeight: 40, maxHeight: 120)` for reasonable size
  - `.focused()` via @FocusState to auto-focus on edit start
  - `.onKeyPress(.escape) { onCancelEdit?(); return .handled }` for cancel
  - `.onKeyPress(.return, modifiers: .command) { onSaveEdit?(); return .handled }` for save (Cmd+Enter since plain Enter is newline in TextEditor)

When editing, show small save/cancel buttons below the editor:
```
HStack {
    Button("Save") { onSaveEdit?() }
        .keyboardShortcut(.return, modifiers: .command)
    Button("Cancel", role: .cancel) { onCancelEdit?() }
        .keyboardShortcut(.escape, modifiers: [])
}
.font(.caption)
```

Add "Edit" and expand/collapse toggle to the context menu (before the Divider):
```
Button {
    onToggleExpand?()
} label: {
    Label(isExpanded ? "Collapse" : "Expand", systemImage: isExpanded ? "chevron.up" : "chevron.down")
}
Button {
    onStartEdit?()
} label: {
    Label("Edit", systemImage: "pencil")
}
```
