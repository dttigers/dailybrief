<task type="auto">
  <name>Task 2: Create POST /v1/chat endpoint with thought context</name>
  <files>vigil-core/src/routes/chat.ts, vigil-core/src/index.ts</files>
  <action>
  Create `vigil-core/src/routes/chat.ts` with a POST route handler:

  **Request body:**
  ```json
  {
    "messages": [
      {"role": "user", "content": "What should I focus on today?"},
      {"role": "assistant", "content": "Based on your tasks..."},
      {"role": "user", "content": "Tell me more about the first one"}
    ],
    "includeContext": true,
    "contextLimit": 20
  }
  ```

  - `messages`: Array of {role, content} — full conversation history (required, at least 1 message, last must be role "user")
  - `includeContext`: boolean (optional, default true) — whether to inject recent thoughts into system prompt
  - `contextLimit`: number (optional, default 20, max 50) — how many recent thoughts to include

  **Implementation:**
  1. Validate: messages array non-empty, last message role is "user", contextLimit between 1 and 50
  2. If includeContext is true: query recent non-deleted thoughts from DB (ordered by createdAt desc, limit contextLimit). Format as numbered list with category, date, and content. Include task status for task-category thoughts.
  3. Build system prompt:
     ```
     You are Vigil, an ambient AI life assistant built for ADHD brains. You help the user understand their thoughts, break down tasks, generate ideas, and find patterns in their captured data.

     Be concise, warm, and actionable. The user has ADHD — avoid walls of text, use bullet points, and lead with the most important thing.

     [If context included:]
     Here are the user's recent captured thoughts for context:
     1. [task/open] (2026-04-05) Fix the login bug
     2. [therapy] (2026-04-05) Feeling overwhelmed with deadlines
     ...
     ```
  4. Call `callClaudeConversation` with system prompt, messages array, maxTokens 1024
  5. Return: `{ response: string, contextUsed: number }`

  **Error handling:** Follow existing patterns — 400 for validation errors, 503 if AI client unavailable, 500 for unexpected errors. Return `{ error: string }`.

  **Register route:** In `vigil-core/src/index.ts`, import and mount the chat route at `/v1/chat` (same pattern as other routes — after auth middleware).

  **Rate limiting note:** The existing 100 req/60s rate limit applies. Chat is more expensive per-call (multi-turn + context), but the global rate limit is sufficient for a single-user tool.
  </action>
