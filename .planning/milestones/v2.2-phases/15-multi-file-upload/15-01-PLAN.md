  <name>Task 1: Multi-select FilePicker + batch import logic in ViewModel</name>
  <files>Sources/DailyBriefMonitor/ImagePicker.swift, Sources/DailyBriefMonitor/Dashboard/DashboardViewModel.swift</files>
  <action>
**FilePicker changes (ImagePicker.swift):**

1. Add a new private helper `pickFiles(types:message:)` that sets `allowsMultipleSelection = true` and returns `[URL]` (using `panel.urls`). Keep the existing single-file `pickFile` helper intact (other code may use it).

2. Add new public methods:
   - `pickAudioFiles() -> [URL]` — same UTTypes as `pickAudioFile()`, multi-select, message "Select audio files to transcribe"
   - `pickImageFiles() -> [URL]` — same UTTypes as `pickImage()`, multi-select, message "Select images to capture"
   - `pickFiles() -> [URL]` — combines both audio AND image UTTypes into one picker, message "Select files to import". This is the primary batch picker.

All methods are `@MainActor static`.

**DashboardViewModel changes:**

1. Replace `importStatus: String?` with richer batch state:
   ```swift
   var importProgress: ImportProgress?

   struct ImportProgress {
       var current: Int
       var total: Int
       var currentFile: String
       var phase: String  // "Transcribing", "Analyzing", "Categorizing", "Saving"
   }
   ```
   Keep `importError` but change it to collect batch errors:
   ```swift
   var importErrors: [String] = []  // replaces importError: String?
   ```

2. Add `importFiles()` method that:
   - Calls `FilePicker.pickFiles()` to get `[URL]`
   - If empty (user cancelled), returns immediately
   - Sets `isImporting = true`, clears `importErrors`
   - Classifies each URL by extension into audio vs image (use same extensions as FolderWatcherService: audio = wav/mp3/m4a/aiff, image = jpg/jpeg/png/gif/webp/heic/heif/tiff/tif/bmp)
   - Processes files sequentially (not parallel — avoids Claude API rate limits and keeps progress predictable)
   - For each file, updates `importProgress` with current count, total, filename, and phase
   - Audio files: transcribe → capture → triage (same flow as existing `importAudio`)
   - Image files: describe → capture → triage (same flow as existing `importImage`)
   - For image files with non-native formats (heic/heif/tiff/tif/bmp), convert to JPEG data first using NSImage → NSBitmapImageRep → jpegData (same pattern as FolderWatcherService). Then call `imageDescriptionService.describe(imageData:mediaType:)` with `.jpeg`.
   - On per-file error: append "\(filename): \(error.localizedDescription)" to `importErrors`, continue to next file (don't stop batch)
   - After all files: set `isImporting = false`, clear `importProgress`, call `refresh()`

3. Update existing `importAudio()` and `importImage()` to use the new multi-select pickers (`pickAudioFiles()` / `pickImageFiles()`) and route through the same batch processing logic. This avoids code duplication — extract a shared `processFiles(urls:)` method that both `importFiles()` and the type-specific methods call.

4. Remove old `importStatus: String?` and `importError: String?` properties since they're replaced by `ImportProgress` and `importErrors`.
  </action>

**Status bar update:**

2. Replace the import status bar section (currently `if let status = viewModel.importStatus`) with batch-aware progress:
   ```swift
   if let progress = viewModel.importProgress {
       HStack(spacing: 8) {
           ProgressView(value: Double(progress.current), total: Double(progress.total))
               .frame(width: 100)
           Text("\(progress.phase) \(progress.currentFile) (\(progress.current)/\(progress.total))")
               .font(.subheadline)
               .foregroundStyle(.secondary)
               .lineLimit(1)
       }
       .frame(maxWidth: .infinity)
       .padding(.vertical, 8)
       .background(Color(nsColor: .controlBackgroundColor))
   ```

3. Replace the single error bar (`if let error = viewModel.importError`) with batch error summary:
   ```swift
   if !viewModel.importErrors.isEmpty {
       VStack(alignment: .leading, spacing: 4) {
           HStack(spacing: 8) {
               Image(systemName: "exclamationmark.triangle.fill")
                   .foregroundStyle(.yellow)
               Text("\(viewModel.importErrors.count) file(s) failed to import")
                   .font(.subheadline)
                   .fontWeight(.medium)
               Spacer()
               Button("Dismiss") {
                   viewModel.importErrors.removeAll()
               }
               .controlSize(.small)
           }
           ForEach(viewModel.importErrors, id: \.self) { error in
               Text("• \(error)")
                   .font(.caption)
                   .foregroundStyle(.red)
           }
       }
       .padding(.horizontal, 12)
       .padding(.vertical, 8)
       .background(Color(nsColor: .controlBackgroundColor))
   }
   ```
