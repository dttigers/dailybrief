<task type="auto">
  <name>Task 1: Add ConfigLoader.save() and create SettingsViewModel + SettingsView</name>
  <files>Sources/JarvisCore/Config/ConfigLoader.swift, Sources/DailyBriefMonitor/Settings/SettingsViewModel.swift, Sources/DailyBriefMonitor/Settings/SettingsView.swift</files>
  <action>
  **ConfigLoader.swift (in JarvisCore):**
  - Add a `public static func save(_ config: AppConfig) throws` method
  - Serialize config to JSON (JSONEncoder with .prettyPrinted + .sortedKeys)
  - Write to `~/.config/dailybrief/config.json` (same path as load reads from)
  - Create directory if it doesn't exist (FileManager.createDirectory with intermediateDirectories)
  - This is the inverse of the existing `load()` method

  **Create `Settings/` subdirectory under `Sources/DailyBriefMonitor/`.**

  **SettingsViewModel.swift:**
  - `@Observable class SettingsViewModel` with `@MainActor`
  - Properties mirror AppConfig fields as editable values:
    - Gmail: email, appPassword, searchSubjectPattern, lookbackDays
    - Sports: teamId, divisionId, leagueId
    - AI: claudeApiKey, claudeModel
    - PDF: outputDirectory, keepDays
    - Printing: enabled, printerName, copies
    - Reminders: listName
  - `isSaving: Bool`, `saveError: String?`, `showSaveSuccess: Bool`
  - `init()` — loads current config via ConfigLoader.load(), populates fields. If load fails, use defaults.
  - `save()` method — constructs AppConfig from current field values, calls ConfigLoader.save(). Shows success/error state.

  **SettingsView.swift:**
  - TabView with tabs for each section: AI, Gmail, Sports, PDF, Printing, Reminders
  - Use `Form` within each tab for clean layout
  - **AI tab**: SecureField for API key, TextField for model name
  - **Gmail tab**: TextField for email, SecureField for app password, TextField for search pattern, Stepper for lookback days
  - **Sports tab**: TextField/Stepper for team ID, division ID, league ID (these are MLB API IDs)
  - **PDF tab**: TextField for output directory, Stepper for keep days
  - **Printing tab**: Toggle for enabled, TextField for printer name, Stepper for copies
  - **Reminders tab**: TextField for list name
  - Bottom bar: "Save" button (calls viewModel.save()), save status indicator
  - Window-appropriate size: ~500x400, not resizable
  </action>
  <verify>swift build compiles without errors</verify>
  <done>ConfigLoader can save config, SettingsView renders all config sections with editable fields, save persists to disk</done>
</task>
