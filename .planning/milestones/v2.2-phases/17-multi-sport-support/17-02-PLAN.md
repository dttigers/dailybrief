  <files>Sources/DailyBrief/Services/ESPNSportsService.swift</files>
  <action>
Create an `actor ESPNSportsService` that fetches data from the ESPN public API for any sport. It should be initialized with a sport type string and a SportLeagueConfig (from 17-01).

ESPN API endpoints (free, no auth, JSON):
- Scoreboard: `https://site.api.espn.com/apis/site/v2/sports/{sport}/{league}/scoreboard?dates=YYYYMMDD`
- Standings: `https://site.api.espn.com/apis/v2/sports/{sport}/{league}/standings`

Sport/league path segments:
- NFL: football/nfl
- NBA: basketball/nba
- NHL: hockey/nhl

The service should have the same three methods as SportsService:
1. `fetchYesterdayGame() async throws -> GameScore?` — Hit scoreboard with yesterday's date, find the game matching config.teamId, map to existing GameScore model
2. `fetchUpcomingGame() async throws -> UpcomingGame?` — Hit scoreboard for today through 7 days ahead (one request per day or use date range), find first future game for team
3. `fetchStandings() async throws -> [StandingsEntry]` — Hit standings endpoint, find the division matching config.divisionId or the team's division, map to existing StandingsEntry model

ESPN scoreboard response structure (key fields):
```json
{
  "events": [{
    "competitions": [{
      "competitors": [
        {"id": "8", "homeAway": "home", "team": {"displayName": "Detroit Lions"}, "score": "24"},
        {"id": "9", "homeAway": "away", "team": {"displayName": "Green Bay Packers"}, "score": "17"}
      ],
      "venue": {"fullName": "Ford Field"},
      "status": {"type": {"completed": true}}
    }],
    "date": "2026-01-05T18:00Z",
    "season": {"type": 2}
  }]
}
```

ESPN standings response structure (key fields):
```json
{
  "children": [{
    "name": "American Football Conference",
    "children": [{
      "name": "AFC East",
      "standings": {
        "entries": [{
          "team": {"displayName": "Buffalo Bills"},
          "stats": [
            {"name": "wins", "value": 13},
            {"name": "losses", "value": 4},
            {"name": "gamesBehind", "value": 0},
            {"name": "streak", "displayValue": "W3"}
          ]
        }]
      }
    }]
  }]
}
```

Create Codable response models (ESPNScoreboardResponse, ESPNStandingsResponse, etc.) inside the file, similar to how SportsService.swift defines MLB API models.

For `fetchUpcomingGame()`, since ESPN scoreboard only returns one day at a time, check today first, then iterate day-by-day up to 7 days until a future game is found. Use a helper to avoid 7 sequential requests — check today and tomorrow in parallel, expand if needed.

Reuse the existing `GameScore`, `UpcomingGame`, and `StandingsEntry` models — they're generic enough (team names, scores, W/L, GB, streak).

For NHL standings: note that NHL uses points/wins/losses/OTL instead of W/L/GB. Map `wins` and `losses` normally, use "PTS" or points as the streak column value, and games back may not apply — use "—" if not available. Actually, adapt: for NHL standings the `StandingsEntry.gamesBack` can show points instead (relabel in PDF later), or just use the ESPN `gamesBehind` stat if available.

**DailyBrief.swift changes:**
After the existing MLB fetch block, add a loop for other enabled sports:

```swift
// Fetch additional sports data
var additionalSports: [SportData] = []
let enabledOtherSports = [
    ("nfl", "NFL", "football/nfl", config.sports.nfl),
    ("nba", "NBA", "basketball/nba", config.sports.nba),
    ("nhl", "NHL", "hockey/nhl", config.sports.nhl),
].filter { $0.3.enabled }

for (key, displayName, sportPath, sportConfig) in enabledOtherSports {
    let service = ESPNSportsService(sport: sportPath, config: sportConfig)
    let game = await tryFetch("\(displayName) score") { try await service.fetchYesterdayGame() }
    let upcoming = await tryFetch("\(displayName) upcoming") { try await service.fetchUpcomingGame() }
    let standings = await tryFetch("\(displayName) standings") { try await service.fetchStandings() }
    additionalSports.append(SportData(
        sport: key,
        sportDisplayName: displayName,
        teamName: sportConfig.teamName,
        divisionName: sportConfig.divisionName,
        gameScore: game ?? nil,
        upcomingGame: upcoming ?? nil,
        standings: standings ?? []
    ))
}
```

Pass `additionalSports` into the DailyBriefData init. Use `async let` to fetch all additional sports concurrently where possible — create all ESPNSportsService instances and fetch in parallel using a TaskGroup or multiple async let bindings.
