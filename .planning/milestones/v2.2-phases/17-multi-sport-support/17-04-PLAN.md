Refactor PageTwoRenderer to render sports sections for MLB (existing fields) plus all entries in `data.additionalSports`.

Extract the current sport rendering logic into a reusable private static method:

```swift
private static func drawSportSection(
    context: CGContext,
    teamName: String,
    divisionName: String,
    gameScore: GameScore?,
    upcomingGame: UpcomingGame?,
    standings: [StandingsEntry],
    sportDisplayName: String,  // "MLB", "NFL", etc. — used as section header prefix
    leftX: CGFloat,
    rightEdge: CGFloat,
    usableWidth: CGFloat,
    startY: CGFloat,
    isCompact: Bool  // true when multiple sports — use smaller fonts/spacing
) -> CGFloat  // returns the Y position after rendering
```

In the main `draw()` method:
1. Render MLB section first (using existing data.gameScore, data.standings, etc.) — always shown if there's MLB data
2. Loop through `data.additionalSports` and render each enabled sport's section
3. When multiple sports are active (1+ additionalSports), use compact mode:
   - Smaller title (use headerFont instead of titleFont for sport name)
   - Reduce spacing between sections
   - Add a sport label prefix like "NFL" before the team name
   - If space is tight, skip the venue line and upcoming game details
4. After all sports sections, render the affirmation and notes section (keep at bottom)

For the `shortenTeamName` helper: make it sport-aware. Currently it uses `MLBTeamData.allTeams`. When rendering NFL/NBA/NHL standings, it should use the appropriate team data. Pass a closure or team lookup function, or check all four team data sources.

Spacing strategy:
- 1 sport: Current full layout (unchanged)
- 2 sports: Each gets ~40% of page height, compact mode
- 3-4 sports: Each gets ~25-30%, very compact (score + standings only, skip upcoming game)

Keep affirmation section at bottom regardless of sport count.
