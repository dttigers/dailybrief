<task type="auto">
  <name>Task 1: Create SQLite-to-PostgreSQL migration script</name>
  <files>vigil-core/scripts/migrate-sqlite.ts, vigil-core/package.json</files>
  <action>
  1. Install `better-sqlite3` and `@types/better-sqlite3` as devDependencies in vigil-core.

  2. Create `vigil-core/scripts/migrate-sqlite.ts` following the pattern in `scripts/generate-key.ts` (uses `tsx`, imports `db` from connection, uses Drizzle for inserts).

  3. Script behavior:
     - Accept `--sqlite` argument for SQLite path (default: `~/Library/Application Support/Jarvis/jarvis.sqlite`)
     - Accept optional `--dry-run` flag that logs counts without inserting
     - Open SQLite database read-only with better-sqlite3
     - Query all rows from `thoughts` table (SELECT *)
     - Query all rows from `thought_links` table (SELECT *)
     - Log counts: "Found N thoughts and M links in SQLite"

  4. Transform each SQLite thought row for PostgreSQL insertion:
     - `isFavorited`: SQLite integer (0/1) → boolean (false/true)
     - `tags`: SQLite TEXT (JSON string like '["tag1","tag2"]') → parsed JS array (or null if empty/null)
     - `createdAt`, `modifiedAt`: SQLite datetime strings → JavaScript Date objects
     - `lastSyncedAt`: same conversion, or null
     - All other text fields pass through directly (category, source, syncStatus, taskStatus, therapyClassification, cloudKitRecordID, content, confidence)
     - Do NOT include `id` — let PostgreSQL auto-assign serial IDs
     - Do NOT include `search_vector` — it's a generated column

  5. Insert thoughts in batches of 100 using `db.insert(thoughts).values(batch).returning({ id, cloudKitRecordID })` to capture the new PostgreSQL IDs.

  6. Build an ID mapping: `Map<sqliteId, postgresId>` using cloudKitRecordID as the join key (read SQLite id + cloudKitRecordID before insert, match to returned postgres id + cloudKitRecordID after insert).

  7. Transform and insert thought_links using the ID mapping:
     - Map `sourceThoughtId` and `targetThoughtId` from SQLite IDs to PostgreSQL IDs
     - Skip any links where either ID is unmapped (log warning)
     - Insert in batches of 100

  8. Print summary: "Migrated N thoughts and M links. Skipped K links (unmapped IDs)."

  9. Add npm script to package.json: `"db:migrate-sqlite": "tsx scripts/migrate-sqlite.ts"`

  Important: The script must call `process.exit(0)` after completion (the postgres connection pool keeps the process alive otherwise, same pattern as generate-key.ts).

  Avoid: Do NOT use the API endpoints for insertion — they auto-generate cloudKitRecordID and set createdAt to now, losing original data. Direct Drizzle inserts preserve all original values.
  </action>
  <verify>
  - `cd vigil-core && npx tsx scripts/migrate-sqlite.ts --dry-run` succeeds, showing thought/link counts from SQLite without inserting
  - TypeScript compiles: `npx tsc --noEmit` passes
  </verify>
  <done>Migration script exists, runs in dry-run mode showing SQLite data counts, TypeScript clean</done>
</task>
