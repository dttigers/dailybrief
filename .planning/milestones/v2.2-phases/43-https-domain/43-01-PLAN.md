<task type="auto">
  <name>Task 1: Add CORS middleware to Vigil Core API</name>
  <files>vigil-core/src/index.ts</files>
  <action>
    Import `cors` from `hono/cors` and add CORS middleware to the Hono app BEFORE all routes.

    Configuration:
    - `origin`: Read from `process.env.CORS_ORIGINS` (comma-separated), fall back to `['*']` for now
    - `allowMethods`: `['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS']`
    - `allowHeaders`: `['Content-Type', 'Authorization']`

    Place `app.use('*', cors(...))` before the health route and auth middleware. This ensures preflight OPTIONS requests are handled before auth rejects them.

    Do NOT add any new npm dependencies — `hono/cors` is built into Hono.
  </action>
  <verify>
    Build: `cd vigil-core && npm run build` succeeds.
    Local test: `curl -I -X OPTIONS http://localhost:3001/v1/health -H "Origin: https://example.com"` returns Access-Control-Allow-Origin header.
  </verify>
  <done>CORS middleware active, preflight requests return correct headers, build passes.</done>
</task>

  <context>
    Railway currently serves at https://vigil-core-production.up.railway.app. A custom domain provides a stable URL decoupled from the hosting platform. You need to own the domain and be able to add a CNAME record.

  <options>
    <option id="custom">
      <name>Use a custom domain you own</name>
      <pros>Branded URL (e.g., api.vigil.dev), portable across hosting platforms, professional</pros>

    </option>
    <option id="skip">
      <name>Keep Railway default domain for now</name>
      <pros>Zero config, already working, can add custom domain later</pros>
      <cons>URL tied to Railway, looks less professional, harder to migrate later if clients hardcode it</cons>
    </option>
  </options>
