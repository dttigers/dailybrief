  <name>Task 1: Create TriageService actor in JarvisCore</name>
  <files>Sources/JarvisCore/Services/TriageService.swift</files>
  <action>
Create a `TriageService` actor in JarvisCore that categorizes thoughts via the Claude API.

**TriageResult struct:**
- `category: ThoughtCategory` — one of the 5 types (task, therapy, idea, reflection, project)
- `confidence: Double` — 0.0 to 1.0 confidence score
- Make it public, Codable, Sendable

**TriageService actor:**
- Init takes `apiKey: String` and `model: String` (same pattern as ClaudeAIProvider in AIService.swift)
- `func triage(_ content: String) async throws -> TriageResult` — core method
- Calls Claude API at `https://api.anthropic.com/v1/messages` with:
  - System prompt instructing Claude to categorize the thought into exactly one of: task, therapy, idea, reflection, project
  - System prompt must request JSON response: `{"category": "task", "confidence": 0.85}`
  - Include brief descriptions of each category in the prompt so Claude understands the taxonomy:
    - task: actionable to-do item, something to do or buy
    - therapy: feelings, emotions, therapy questions, mental health reflections
    - idea: creative ideas, feature concepts, business ideas, "what if" thoughts
    - reflection: observations, journal entries, life reflections, gratitude
    - project: project notes, technical decisions, work-related context
  - max_tokens: 100 (response is small JSON)
- Parse the JSON response to extract category string and confidence double
- Map category string to ThoughtCategory enum (case-insensitive)
- Clamp confidence to 0.0–1.0 range
- If parsing fails, throw a TriageError

**TriageError enum:**
- `apiError(String)` — HTTP/network errors
- `parseError(String)` — response parsing failures
- Conform to LocalizedError with errorDescription (follow existing ConfigError/AIError pattern)

**Key patterns to follow:**
- URLSession.shared for HTTP (same as ClaudeAIProvider.callClaude)
- `x-api-key` header, `anthropic-version: 2023-06-01` (same as AIService.swift)
- Actor isolation (same as CaptureService pattern)
- Public access on all types for cross-module use

<task type="auto">
  <name>Task 2: Wire auto-triage into capture flow</name>
  <files>Sources/DailyBriefMonitor/AppDelegate.swift</files>
  <action>
Wire TriageService into the DailyBriefMonitor capture flow so every captured thought is auto-triaged.

**AppDelegate changes:**
1. Load AppConfig via `ConfigLoader.load()` in applicationDidFinishLaunching to get AI config
2. Create TriageService with `config.ai.claudeApiKey` and `config.ai.claudeModel`
3. Store triageService as a property alongside captureService
4. Modify the onCapture closure to:
   a. Call `captureService.captureText(text)` to save the thought (existing behavior)
   b. Fire-and-forget triage in a background Task — do NOT block the capture flow
   c. In the background Task: call `triageService.triage(thought.content)`, then update the thought with category + confidence via `thoughtStore.save()`
   d. ThoughtStore needs to be accessible — store it as a property on AppDelegate

**Important design decisions:**
- Triage is fire-and-forget from the capture perspective — the user gets immediate "Captured!" feedback regardless of triage speed. Plan 03-02 will add UI to show the triage result.
- If triage fails (API error, no internet), log the error but don't surface it to the user — the thought is still saved. It can be re-triaged later.
- If ConfigLoader.load() fails (no config file), skip TriageService creation — capture still works without triage (graceful degradation).
- Use NSLog for triage errors (consistent with existing DB init error logging in AppDelegate).

**onCapture closure update:**
```
onCapture: { text in
    let thought = try await service.captureText(text)
    // Fire-and-forget triage
    if let triageService = self.triageService, let thoughtStore = self.thoughtStore {
        Task {
            do {
                let result = try await triageService.triage(thought.content)
                var updated = thought
                updated.category = result.category
                updated.confidence = result.confidence
                try thoughtStore.save(&updated)
            } catch {
                NSLog("Triage failed: \(error.localizedDescription)")
            }
        }
    }
}
```

**Swift 6 considerations:**
- capturePanel, captureService, triageService, thoughtStore are all accessed in closures — use same patterns from Phase 2 (capture references before closure, Task { @MainActor } where needed)
- TriageService and ThoughtStore are both actors — safe to call from background Task
