<task type="auto">
  <name>Task 1: Create Vite + TypeScript project with Even Hub SDK</name>
  <files>vigil-g2-plugin/package.json, vigil-g2-plugin/tsconfig.json, vigil-g2-plugin/vite.config.ts, vigil-g2-plugin/index.html, vigil-g2-plugin/app.json</files>
  <action>
Create the `vigil-g2-plugin/` directory at the repo root (alongside `vigil-core/`).

1. Initialize a Vite vanilla-ts project:
   - `npm create vite@latest vigil-g2-plugin -- --template vanilla-ts`
   - Then cd into it and install the SDK: `npm install @evenrealities/even_hub_sdk`
   - Install CLI tools globally: `npm install -g @evenrealities/evenhub-cli @evenrealities/evenhub-simulator`

2. Clean up the Vite template:
   - Remove default `src/counter.ts`, `src/style.css`, `src/typescript.svg`, `public/vite.svg`
   - Simplify `index.html` to bare minimum — just a script tag pointing to `/src/main.ts`. The Even Hub SDK runs as a web app inside the glasses bridge, so no visible HTML is needed. Keep the `<div id="app"></div>` as a minimal mount point.
   - Update `tsconfig.json`: set `"strict": true`, `"target": "ES2020"`, `"module": "ESNext"`, `"moduleResolution": "bundler"`

3. Create `app.json` manifest (Even Hub plugin descriptor):
   ```json
   {
     "name": "Vigil",
     "version": "0.1.0",
     "description": "Ambient AI life assistant for Even G2 smart glasses",
     "entrypoint": "dist/index.html",
     "permissions": []
   }
   ```

4. Configure `vite.config.ts`:
   - Set base to `"./"` (relative paths for packaged plugin)
   - Set build outDir to `"dist"`
  </action>
  <verify>
   - `cd vigil-g2-plugin && npm run build` succeeds
   - `ls vigil-g2-plugin/app.json` exists
   - `ls vigil-g2-plugin/node_modules/@evenrealities/even_hub_sdk` exists
  </verify>
  <done>Vite+TS project builds clean, SDK installed, app.json manifest present</done>
</task>

<task type="auto">
  <name>Task 2: Initialize SDK bridge and render minimal startup container</name>
  <files>vigil-g2-plugin/src/main.ts</files>
  <action>
Create `src/main.ts` as the plugin entry point:

1. Import `waitForEvenAppBridge` from `@evenrealities/even_hub_sdk`
2. Create an async `init()` function that:
   a. Calls `const bridge = await waitForEvenAppBridge()` to get the bridge instance
   b. Calls `bridge.createStartUpPageContainer()` with a single text container:
      - Full width (576x288), containerID: 1, containerName: "home"
      - Content: "VIGIL\n━━━━━━━━━━━━━━━━━━━━\nConnecting..."
      - isEventCapture: 1
   c. Logs "Vigil G2 plugin initialized" to console
3. Call `init()` at module level

This proves the SDK bridge initializes and can render to the display. The home screen content will be built in Plan 02.

Do NOT add event listeners yet — Phase 33 handles navigation. Just get the bridge up and a container rendered.
  </action>
  <verify>
   - `cd vigil-g2-plugin && npm run build` succeeds without TypeScript errors
   - `src/main.ts` imports from `@evenrealities/even_hub_sdk`
   - `src/main.ts` calls `createStartUpPageContainer` with valid container structure
  </verify>
  <done>Bridge initializes, startup container renders with Vigil branding, build passes</done>
</task>
