<task type="auto">
  <name>Task 2: Create FolderWatcherService</name>
  <files>Sources/JarvisCore/Services/FolderWatcherService.swift</files>
  <action>
    Create a new `FolderWatcherService` actor in JarvisCore/Services/ that:

    **1. Initialization:**
    - Takes dependencies: `TranscriptionService`, `ImageDescriptionService?`, `CaptureService`, `TriageService?`
    - Takes config: `FolderWatchingConfig` (paths + enabled flag)
    - Maintains a `Set<String>` of processed file names loaded from a JSON manifest

    **2. Processed file tracking:**
    - Manifest file at `~/Library/Application Support/Jarvis/processed-files.json` (array of filenames)
    - On init, load existing manifest. On each successful processing, append filename and save.
    - Track by filename only (not full path) — simple and sufficient for dedicated watched folders.

    **3. Folder monitoring with DispatchSource:**
    - `start()` method: For each configured folder path (audio, image), expand tilde, create directory if missing, open file descriptor with `open(path, O_EVTONLY)`, create `DispatchSource.makeFileSystemObjectSource(fileDescriptor:eventMask:queue:)` with `.write` event mask.
    - On event: call `scanFolder(path:type:)` after a short debounce delay (0.5s via DispatchWorkItem) to handle files still being written.
    - `stop()` method: Cancel all dispatch sources, close file descriptors.

    **4. Scan and process pipeline:**
    - `scanFolder(path:type:)`: List directory contents, filter by known extensions, skip already-processed files.
    - Audio extensions: `.wav`, `.mp3`, `.m4a`, `.aiff`
    - Image extensions: `.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`
    - For each new file:
      - Audio: `transcriptionService.transcribe(audioURL:)` → captured text → `captureService.captureText(text)` with source `.voice`
      - Image: `imageDescriptionService.describe(imageURL:)` → description → `captureService.captureText(text)` with source `.image`
      - If triage service available: triage the captured thought (same pattern as AppDelegate capture panel)
      - On success: add filename to processed set, save manifest
      - On failure: log error via NSLog, skip file (will retry on next scan)

    **5. Error handling:**
    - Wrap individual file processing in do/catch — one file failing doesn't stop others.
    - Log errors with NSLog (consistent with existing codebase pattern).
    - If folder doesn't exist and can't be created, log and skip that folder.

    **Note on CaptureService:** Currently `captureText` may not accept a `source` parameter. If CaptureService.captureText only takes a String, check its signature. If it saves with `.text` source, you'll need to either:
    - Add a `capture(text:source:)` method to CaptureService, or
    - Save via ThoughtStore directly with the correct source.
    Follow whichever approach is simpler. Do NOT break existing callers.

    **Note on actor isolation:** FolderWatcherService should be an actor (consistent with TranscriptionService, ImageDescriptionService pattern). The DispatchSource callbacks run on a background queue — use `Task { await self.scanFolder(...) }` to bounce back into actor context.
  </action>
