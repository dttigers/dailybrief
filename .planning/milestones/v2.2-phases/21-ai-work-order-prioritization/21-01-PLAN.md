<objective>
Add AI-powered work order prioritization using Claude to analyze and rank open work orders by urgency.

Purpose: Work orders from ServiceNow emails arrive with a basic "Priority" string (e.g., "High", "Medium") but no intelligent ranking. Claude can analyze the full context (description, trade, equipment, location) to recommend which work orders should be tackled first — surfacing urgent issues (e.g., HVAC down in summer, water leaks) above routine maintenance.

Output: WorkOrderPrioritizer service + integration into generate command + AI-ranked sort order in PDF.
</objective>

<task type="auto">
  <name>Task 1: Create WorkOrderPrioritizer service</name>
  <files>Sources/DailyBrief/Services/WorkOrderPrioritizer.swift</files>
  <action>
Create a new `WorkOrderPrioritizer` actor following the same pattern as `ClaudeAIProvider` in AIService.swift (direct HTTP to Claude API via URLSession).

**Inputs:** Array of WorkOrder objects + AI config (apiKey, model)
**Output:** Array of case numbers in recommended priority order (highest urgency first)

**Claude prompt design:**
- System prompt: "You are a facilities management assistant. Analyze these work orders and rank them by urgency. Consider: safety hazards (electrical, water, gas), customer/business impact (HVAC in extreme weather, security issues, food safety), time-sensitivity (perishable equipment, active leaks), and trade complexity."
- User message: Format each work order as a numbered list with all 9 fields (caseNumber, store, shortDescription, trade, location, equipment, priority, contact, state)
- Response format: JSON array of case numbers in priority order, e.g. `["CS0353601", "CS0353598", ...]`

**Caching:** Cache results to `~/.cache/dailybrief/wo-priority-YYYY-MM-DD.json` keyed by a hash of the work order case numbers (so cache invalidates when the set of work orders changes). This avoids re-calling Claude on every dry-run or re-generate within the same day with the same work orders.

**Error handling:** On failure (API error, parse error), return nil so the caller falls back to existing sort order. Log the error via Logger. Do NOT crash the generate flow.

**Init:** Takes `AIConfig` (same config struct used by ClaudeAIProvider — has apiKey and model).
  </action>
