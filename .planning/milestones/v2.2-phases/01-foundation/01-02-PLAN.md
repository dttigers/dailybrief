<task type="auto">
  <name>Task 1: Define Thought model and database schema with migrations</name>
  <files>Sources/JarvisCore/Models/Thought.swift, Sources/JarvisCore/Storage/DatabaseManager.swift</files>
  <action>
  **Thought.swift:**
  Create a public struct `Thought` conforming to GRDB's `Codable`, `FetchableRecord`, `MutablePersistableRecord`, and `Sendable`.

  Fields:
  - `id: Int64?` (auto-increment primary key, nil before insert)
  - `content: String` (the captured thought text — required, non-empty)
  - `category: ThoughtCategory?` (nil until AI triage assigns one)
  - `confidence: Double?` (triage confidence 0.0-1.0, nil until triaged)
  - `source: CaptureSource` (text, voice, image — how it was captured)
  - `createdAt: Date` (auto-set on creation)
  - `modifiedAt: Date` (auto-set on creation and update)

  Create public enum `ThoughtCategory: String, Codable, Sendable, DatabaseValueConvertible` with cases: `task, therapy, idea, reflection, project`

  Create public enum `CaptureSource: String, Codable, Sendable, DatabaseValueConvertible` with cases: `text, voice, image`

  Implement `mutating func didInsert(_ inserted: InsertionSuccess)` to set the id after insert.
  Define `static let databaseTableName = "thoughts"` and column enum for type-safe queries.

  **DatabaseManager.swift:**
  Create a public actor `DatabaseManager` that:
  1. Holds a `DatabaseQueue` (GRDB's thread-safe database access)
  2. Initializes with a path (default: `~/Library/Application Support/Jarvis/jarvis.sqlite`)
  3. Creates parent directory if needed
  4. Runs migrations via `DatabaseMigrator`:
     - Migration "v1-thoughts": Creates `thoughts` table with all Thought columns, sets `createdAt` and `modifiedAt` defaults to `CURRENT_TIMESTAMP`
     - Migration "v1-thoughts-fts": Creates FTS5 virtual table `thoughts_fts` synchronized with `thoughts` table via content-sync, indexing `content` column with `unicode61` tokenizer
  5. Exposes `var reader: DatabaseReader` and a method `func write<T>(_ block: (Database) throws -> T) async throws -> T` for controlled access

  Use GRDB 7's `DatabaseQueue` API. For FTS5 content-sync, use:
  ```swift
  try db.create(virtualTable: "thoughts_fts", using: FTS5()) { t in
      t.synchronize(withTable: "thoughts")
      t.tokenizer = .unicode61()
      t.column("content")
  }
  ```

  Do NOT use WAL mode explicitly — GRDB's DatabaseQueue handles this by default.
  </action>
  <verify>`swift build` succeeds with no errors or warnings for JarvisCore target</verify>

<task type="auto">
  <name>Task 2: Create ThoughtStore actor with CRUD and FTS5 search</name>
  <files>Sources/JarvisCore/Storage/ThoughtStore.swift</files>
  <action>
  Create a public actor `ThoughtStore` that wraps `DatabaseManager` and provides the data access API.

  Methods:
  - `public func save(_ thought: inout Thought) async throws` — Insert or update. Sets `modifiedAt` to now on update. Uses `thought.save(db)`.
  - `public func delete(id: Int64) async throws -> Bool` — Delete by ID, return whether a row was deleted.
  - `public func fetch(id: Int64) async throws -> Thought?` — Fetch single by ID.
  - `public func fetchAll(category: ThoughtCategory? = nil, limit: Int = 100, offset: Int = 0) async throws -> [Thought]` — Fetch with optional category filter, ordered by `createdAt DESC`, with pagination.
  - `public func search(query: String, limit: Int = 50) async throws -> [Thought]` — FTS5 full-text search. Use GRDB's `Thought.matching(FTS5Pattern(matchingAllTokensIn: query))` joined with the FTS table. Return results ranked by FTS5 relevance. Handle empty query by returning empty array. Sanitize the query string to prevent FTS5 syntax errors — use `FTS5Pattern(matchingAllTokensIn:)` which safely tokenizes user input (do NOT use raw `FTS5Pattern(rawPattern:)` as it can throw on special characters).
  - `public func count(category: ThoughtCategory? = nil) async throws -> Int` — Count thoughts, optionally filtered by category.

  Initialize with `DatabaseManager` instance. All database access goes through the manager's `write` or `reader` methods.

  Follow existing codebase convention: actors for stateful services (like GmailService, SportsService pattern).
  </action>
