```
public actor APIThoughtStore: ThoughtRepository {

```

Implement all READ methods by mapping to API calls:

**Single fetch:**
- `fetch(id:)` → GET /thoughts/:id → decode ThoughtResponse → convert to Thought model

**List/Filter methods:**
- `fetchAll(category:limit:offset:)` → GET /thoughts with query params
- `fetchFiltered(category:source:after:tag:favoritesOnly:limit:offset:)` → GET /thoughts with all filter query params (use `after` ISO 8601 format for date param)
- `search(query:limit:)` → GET /thoughts?q={query}&limit={limit}
- `fetchTasks(status:limit:)` → GET /thoughts?category=task&taskStatus={status}
- `fetchTherapyThoughts(classification:limit:offset:)` → GET /thoughts?category=therapy&therapyClassification={classification}
- `fetchByTag(tag:limit:offset:)` → GET /thoughts?tag={tag}
- `fetchFavorites(limit:offset:)` → GET /thoughts?favoritesOnly=true
- `fetchTherapyThoughtsByDateRange(from:to:classification:)` → GET /thoughts?category=therapy&after={from}&before={to}&therapyClassification={classification}
- `fetchRecentTherapyThoughts(days:classification:limit:)` → compute `after` as (now - days), then same as date range query
- `fetchLinkedThoughts(thoughtId:)` → GET /thoughts/:id/links

**Count methods:**
All count methods use the same GET /thoughts endpoint but only read the `total` field from the PaginatedResponse. Set limit=0 to avoid transferring actual data:
- `count(category:)` → GET /thoughts?category={category}&limit=0 → read `total`
- `countFiltered(...)` → GET /thoughts with filters + limit=0 → read `total`
- `countTasks(status:)` → GET /thoughts?category=task&taskStatus={status}&limit=0 → read `total`
- `countTherapy(classification:)` → GET /thoughts?category=therapy&therapyClassification={classification}&limit=0 → read `total`
- `countUnclassifiedTherapy()` → GET /thoughts?category=therapy&limit=0 → read total, then GET /thoughts?category=therapy&therapyClassification=selfLearnable&limit=0 and therapyClassification=bringToTherapist&limit=0 → subtract from total. OR: add a specific API query for this (simpler: just pass therapyClassification as empty/null to mean unclassified — if the API supports it. If not, use the subtraction approach.)
- `countFavorites()` → GET /thoughts?favoritesOnly=true&limit=0 → read `total`
- `countLinks(thoughtId:)` → GET /thoughts/:id/links → count the array length

**Tags list:**
- `allUniqueTags()` → GET /tags → decode string array

**JSON ↔ Thought conversion:**
The API returns ThoughtResponse (with tags as string array, dates as ISO strings). Create a private helper `func toThought(_ response: ThoughtResponse) -> Thought` that maps API response fields to the Swift Thought struct. Key mappings:
- `response.tags` (array) → `thought.tags` (array, already matches)
- `response.createdAt` (ISO string) → `thought.createdAt` (Date) — use ISO 8601 DateFormatter
- `response.category` (string) → `thought.category` (ThoughtCategory enum) — use rawValue init
- `response.syncStatus` always set to `.synced` (API-backed thoughts don't need local sync tracking)
- `response.isFavorited` (0/1 int or bool) → `thought.isFavorited` (Bool)

Define a Codable struct `APIThoughtResponse` that matches the exact JSON shape from the API (tags as [String], dates as String, isFavorited as Int). Decode with this, then convert to Thought model.

<task type="auto">
  <name>Task 2: Implement write operations</name>
  <files>Sources/JarvisCore/Storage/APIThoughtStore.swift</files>
  <action>
Add all WRITE methods to APIThoughtStore:

**CRUD writes:**
- `saveThought(_ thought: Thought) -> Thought` → POST /thoughts with body {content, source, category, tags}. Decode response to Thought.
- `save(_ thought: inout Thought)` → Call saveThought, then mutate the inout parameter with the returned Thought (sets id, timestamps, etc.)
- `update(_ thought: Thought) -> Thought` → PUT /thoughts/:id with changed fields. Decode response to Thought.
- `delete(id:) -> Bool` → DELETE /thoughts/:id. Return true on 204, false on 404.

**Task status:**
- `updateTaskStatus(id:status:) -> Thought` → PUT /thoughts/:id with {taskStatus: status.rawValue}

**Tags:**
- `addTag(id:tag:) -> Thought?` → POST /thoughts/:id/tags with {tag}. Decode response.
- `removeTag(id:tag:) -> Thought?` → DELETE /thoughts/:id/tags/:tag. Then fetch updated thought.

**Favorites:**
- `toggleFavorite(id:) -> Thought?` → PUT /thoughts/:id/favorite. Decode response.

**Links:**
- `linkThoughts(sourceId:targetId:) -> ThoughtLink?` → POST /thoughts/:id/links with {targetId}. Decode response.
- `unlinkThoughts(sourceId:targetId:)` → DELETE /thoughts/:id/links/:linkedId.

**Bulk operations:**
- `bulkDelete(ids:) -> Int` → POST /thoughts/bulk/delete with {ids}. Return `updated` count.
- `bulkUpdateCategory(ids:category:) -> Int` → POST /thoughts/bulk/recategorize with {ids, category}. Return `updated` count.
- `bulkAddTag(ids:tag:) -> Int` → POST /thoughts/bulk/tag with {ids, tag, action: "add"}. Return `updated` count.
- `bulkRemoveTag(ids:tag:) -> Int` → POST /thoughts/bulk/tag with {ids, tag, action: "remove"}. Return `updated` count.
- `bulkUpdateTherapyClassification(ids:classification:) -> Int` → POST /thoughts/bulk/therapy-classify with {ids, classification}. Return `updated` count.

**Request body encoding:**
Use Encodable structs for each request body type. Keep them private within the file. Use snake_case JSON encoding to match the API conventions where needed (though the Vigil Core API uses camelCase — match what the API expects).

**Error mapping:**
- 404 responses → return nil or false as appropriate (not found is not an error for optional returns)
- 400 responses → throw descriptive error
- 503 responses → throw `.serverUnavailable`
