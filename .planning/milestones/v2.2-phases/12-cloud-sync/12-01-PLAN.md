Purpose: CloudKit sync requires a stable UUID per thought (for CKRecord IDs) and a sync status flag to track what needs uploading. Without these fields, the sync engine has no way to know which records are dirty or how to map local rows to remote records.
Output: Extended Thought model with `cloudKitRecordID`, `syncStatus`, `lastSyncedAt` fields. New `SyncStatus` enum. Database migration v2 that adds columns and backfills UUIDs for existing rows.

<task type="auto">
  <name>Task 2: Create v2 database migration for sync columns</name>
  <files>Sources/JarvisCore/Storage/DatabaseManager.swift</files>
  <action>
  **Add migration "v2-sync-fields"** in `runMigrations(on:)` after existing migrations:

  ```
  migrator.registerMigration("v2-sync-fields") { db in
      // Add sync columns to thoughts table
      try db.alter(table: "thoughts") { t in
          t.add(column: "cloudKitRecordID", .text).notNull().defaults(to: "")
          t.add(column: "syncStatus", .text).notNull().defaults(to: "pending")
          t.add(column: "lastSyncedAt", .datetime)
      }

      // Backfill UUIDs for existing rows (empty string default isn't valid)
      let rows = try Row.fetchAll(db, sql: "SELECT id FROM thoughts")
      for row in rows {
          let id: Int64 = row["id"]
          let uuid = UUID().uuidString
          try db.execute(
              sql: "UPDATE thoughts SET cloudKitRecordID = ? WHERE id = ?",
              arguments: [uuid, id]
          )
      }

      // Add unique index on cloudKitRecordID for efficient lookups during sync
      try db.create(index: "idx_thoughts_cloudKitRecordID",
                    on: "thoughts",
                    columns: ["cloudKitRecordID"],
                    unique: true)
  }
  ```

  Key points:
  - Use `.defaults(to: "")` for cloudKitRecordID since SQLite ALTER TABLE requires a default for NOT NULL columns, then backfill immediately
  - Each existing row gets a fresh UUID
  - Unique index on cloudKitRecordID enables O(1) lookup when processing CloudKit records
  - `syncStatus` defaults to "pending" so all existing thoughts will be uploaded on first sync
  - `lastSyncedAt` is nullable (nil = never synced)
  </action>
