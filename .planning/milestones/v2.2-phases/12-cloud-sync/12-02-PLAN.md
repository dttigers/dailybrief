  **Create CloudKitManager actor** (`Sources/JarvisCore/Services/CloudKitManager.swift`):

  ```swift
  import CloudKit
  ```

  **Constants:**
  - `static let containerID = "iCloud.com.jamesonmorrill.jarvis"`
  - `static let zoneName = "ThoughtsZone"` — custom zone for incremental sync
  - `static let thoughtRecordType = "Thought"` — CKRecord type name

  **Properties:**
  - `private let container: CKContainer`
  - `private let database: CKDatabase` (container.privateCloudDatabase)
  - `let zoneID: CKRecordZone.ID`

  **Initialization:**
  - `public init()` — create CKContainer with containerID, get privateCloudDatabase, create zoneID

  **Zone setup method:**
  - `public func createZoneIfNeeded() async throws`
  - Create `CKRecordZone(zoneID: zoneID)` and save via `database.save(_:)`
  - Catch `CKError.serverRecordChanged` or zone already exists — that's fine, swallow it
  - This is called once on first sync

  **Record mapping — Thought → CKRecord:**
  - `public func record(from thought: Thought) -> CKRecord`
  - CKRecord.ID: `CKRecord.ID(recordName: thought.cloudKitRecordID, zoneID: zoneID)`
  - Set fields: `content` (String), `category` (String? via rawValue), `confidence` (Double? as NSNumber), `source` (String via rawValue), `createdAt` (Date), `modifiedAt` (Date)
  - Do NOT store `id` (local auto-increment), `syncStatus`, or `lastSyncedAt` — those are local-only

  **Record mapping — CKRecord → Thought values:**
  - `public func thoughtValues(from record: CKRecord) -> ThoughtCloudData`
  - Create a simple struct `ThoughtCloudData` with: `cloudKitRecordID: String`, `content: String`, `category: ThoughtCategory?`, `confidence: Double?`, `source: CaptureSource`, `createdAt: Date`, `modifiedAt: Date`
  - Extract from record fields, using defaults where appropriate
  - `cloudKitRecordID` = `record.recordID.recordName`

  **Make ThoughtCloudData** a public struct in the same file — it's the intermediary for creating/updating local Thoughts from remote records.

  **Database access helper:**
  - `public var privateDatabase: CKDatabase { database }` — exposed for SyncService to use
  </action>
