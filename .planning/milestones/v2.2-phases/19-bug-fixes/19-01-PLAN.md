<task type="auto">
  <name>Task 1: Fix duplicate thoughts in FTS5 search</name>
  <files>Sources/JarvisCore/Storage/ThoughtStore.swift</files>
  <action>
In ThoughtStore.search() (around line 114), the FTS5 JOIN can return duplicate Thought rows when a thought matches multiple search tokens. Add `.distinct()` before the `.joining()` call to deduplicate results. Also add `.order(Thought.Columns.createdAt.desc)` to maintain consistent sort order.

Change:
```swift
return try Thought
    .joining(required: Thought.thoughtsFts.matching(pattern))
    .limit(limit)
    .fetchAll(db)
```

To:
```swift
return try Thought
    .distinct()
    .joining(required: Thought.thoughtsFts.matching(pattern))
    .order(Thought.Columns.createdAt.desc)
    .limit(limit)
    .fetchAll(db)
```
  </action>
  <verify>swift build succeeds with no errors</verify>
  <done>Search results are deduplicated and sorted by creation date descending</done>
</task>

  <action>
**Bug:** FolderWatcherService.processAudioFile() and processImageFile() both call triageService.triage() but only log the result — they never persist the category and confidence back to the database. The correct pattern exists in DashboardViewModel.processFiles() (lines ~277-289).

**Fix FolderWatcherService:**

1. Add a `thoughtStore: ThoughtStore` property to the actor (alongside the existing captureService, triageService, etc.).

2. Update the init to accept and store a `thoughtStore` parameter:
```swift
public init(
    transcriptionService: TranscriptionService,
    imageDescriptionService: ImageDescriptionService?,
    captureService: CaptureService,
    triageService: TriageService?,
    thoughtStore: ThoughtStore,
    config: AppConfig.FolderWatchingConfig
)
```

3. In processAudioFile() (around line 234-242), after `let result = try await triageService.triage(text)`, add the persist step:
```swift
if var t = try await thoughtStore.fetch(id: thought.id!) {
    t.category = result.category
    t.confidence = result.confidence
    _ = try thoughtStore.update(t)
}
```

4. Apply the same fix in processImageFile() (around line 264-272) — same pattern, persist triage result after computing it.

**Fix AppDelegate wiring:**

In AppDelegate.swift where FolderWatcherService is instantiated (around line 114), pass the ThoughtStore instance. The ThoughtStore (`store` variable) is already created earlier in applicationDidFinishLaunching. Add `thoughtStore: store` to the FolderWatcherService init call.
