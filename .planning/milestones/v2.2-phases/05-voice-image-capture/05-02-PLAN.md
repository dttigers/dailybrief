<task type="auto">
  <name>Task 1: Create ImageDescriptionService actor with multimodal Claude API</name>
  <files>Sources/JarvisCore/Services/ImageDescriptionService.swift</files>
  <action>
  Create ImageDescriptionService actor in JarvisCore/Services/. Follow the same URLSession + raw JSON pattern as TriageService (no external SDK).

  Public API:
  - `init(apiKey: String, model: String)` — same pattern as TriageService.
  - `func describe(imageData: Data, mediaType: ImageMediaType) async throws -> String` — sends image to Claude and returns text description.

  ImageMediaType enum (public, Sendable):
  - `.jpeg`, `.png`, `.gif`, `.webp`
  - `var mimeType: String` computed property returning "image/jpeg", "image/png", etc.

  ImageDescriptionError enum (public, LocalizedError):
  - `.apiError(String)` — HTTP/network errors
  - `.parseError(String)` — response parsing failures
  - `.imageTooLarge` — image data exceeds 20MB (Claude's limit)

  Implementation of `describe()`:
  1. Validate imageData.count < 20_000_000 (20MB limit). Throw `.imageTooLarge` if exceeded.
  2. Base64-encode the image data: `imageData.base64EncodedString()`.
  3. Build Claude messages API request to `https://api.anthropic.com/v1/messages`:
     - Same headers as TriageService: content-type, x-api-key, anthropic-version 2023-06-01.
     - Body JSON:
       ```
       {
         "model": model,
         "max_tokens": 300,
         "messages": [{
           "role": "user",
           "content": [
             {
               "type": "image",
               "source": {
                 "type": "base64",
                 "media_type": mediaType.mimeType,
                 "data": "<base64string>"
               }
             },
             {
               "type": "text",
               "text": "Describe this image concisely in 1-2 sentences. Focus on what is shown and any text visible in the image. This will be stored as a thought capture."
             }
           ]
         }]
       }
       ```
     - Use JSONSerialization.data(withJSONObject:) for the request body (same as TriageService).
  4. Parse response: extract `content[0].text` from Claude's response JSON (same two-level pattern as TriageService, but here we just take the text directly — no inner JSON to parse).
  5. Trim whitespace from the description. If empty, throw `.parseError`.
  6. Return the description string.

  Also add a convenience method:
  - `func describe(imageURL: URL) async throws -> String` — reads Data from URL, detects media type from file extension (jpg/jpeg → .jpeg, png → .png, gif → .gif, webp → .webp, default to .jpeg), calls describe(imageData:mediaType:).

  Use Foundation only — no additional imports needed beyond what JarvisCore already has.
  </action>
