<task type="auto">
  <name>Task 1: Implement OAuth2 authorization flow</name>
  <files>Sources/DailyBriefMonitor/GoogleCalendarAuth.swift</files>
  <action>
  Create GoogleCalendarAuth.swift in DailyBriefMonitor with an OAuth2 flow using localhost redirect:

  1. `@MainActor final class GoogleCalendarAuth` (not actor — needs MainActor for NSWorkspace browser open):
     - Static func `authorize(clientId: String, clientSecret: String) async throws -> CalendarTokens`
     - Implementation:
       a. Start a temporary HTTP server on localhost (find available port, e.g., 8089-8099 range)
       b. Build Google OAuth2 authorization URL:
          - `https://accounts.google.com/o/oauth2/v2/auth`
          - client_id, redirect_uri=http://localhost:{port}, response_type=code, scope=https://www.googleapis.com/auth/calendar.readonly, access_type=offline, prompt=consent
       c. Open URL in system browser via `NSWorkspace.shared.open(url)`
       d. Wait for redirect callback (the localhost server receives the auth code)
       e. Exchange auth code for tokens: POST to `https://oauth2.googleapis.com/token` with code, client_id, client_secret, redirect_uri, grant_type=authorization_code
       f. Parse response: access_token, refresh_token, expires_in
       g. Create CalendarTokens with expiresAt = Date() + expires_in seconds
       h. Save tokens via CalendarTokens.save()
       i. Shut down temporary HTTP server
       j. Return tokens

  For the localhost HTTP server, use a simple approach:
  - Create a Socket/NWListener on localhost that handles ONE request
  - Parse the `code` query parameter from the redirect URL
  - Return a simple HTML page: "Authorization successful! You can close this tab."
  - Use `NWListener` from Network framework (available on macOS 14+) or a simpler `DispatchIO`-based approach
  - Alternatively, use a continuation-based wrapper around a basic socket listener

  Keep it simple — this is a one-shot server that handles a single redirect callback.

  Error handling: timeout after 120 seconds if no callback received. Throw descriptive errors for each failure point.
  </action>
  <verify>swift build compiles without errors.</verify>
  <done>GoogleCalendarAuth.authorize() compiles. OAuth2 flow implemented with localhost redirect, token exchange, and persistence.</done>
</task>

  <files>Sources/DailyBriefMonitor/Settings/SettingsView.swift, Sources/DailyBriefMonitor/Settings/SettingsViewModel.swift</files>
  <action>
  1. Update SettingsViewModel:
     - Add properties: googleCalendarEnabled (Bool), googleCalendarClientId (String), googleCalendarClientSecret (String), selectedCalendarIds ([String]), isAuthorized (Bool), isAuthorizing (Bool), availableCalendars ([(id: String, name: String)])
     - Load google calendar config in loadConfig() from config.googleCalendar
     - Check isAuthorized by checking CalendarTokens.load() != nil
     - Add `func authorizeGoogleCalendar() async` that:
       a. Sets isAuthorizing = true
       b. Calls GoogleCalendarAuth.authorize(clientId:clientSecret:)
       c. On success: sets isAuthorized = true, fetches calendar list via GoogleCalendarService
       d. On failure: sets appropriate error
       e. Sets isAuthorizing = false
     - Add `func fetchAvailableCalendars() async` that uses GoogleCalendarService.fetchCalendarList()
     - Add `func disconnectGoogleCalendar()` that deletes token file and resets state
     - Include googleCalendar config in save() method's AppConfig construction

  2. Update SettingsView:
     - Add a `calendarTab` computed property:
       - Toggle "Enable Google Calendar" bound to googleCalendarEnabled
       - If enabled:
         - TextField for Client ID, SecureField for Client Secret

         - If authorizing: ProgressView with "Authorizing..."

           - Green checkmark "Connected"
           - "Disconnect" button
           - If availableCalendars loaded: List of toggleable calendars (multi-select which calendar IDs to include)
           - "Refresh Calendars" button
     - Add calendarTab to TabView with Label("Calendar", systemImage: "calendar")
     - Widen window from 600x420 to 600x460 if needed to fit new tab content

  Follow existing SettingsView patterns (Form, consistent spacing, bottom bar save).
