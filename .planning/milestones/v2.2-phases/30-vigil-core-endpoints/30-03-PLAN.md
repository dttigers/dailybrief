  <files>vigil-core/src/routes/brief.ts</files>
  <action>
  Create src/routes/brief.ts exporting a Hono instance with:

  **GET /brief** — Aggregated daily brief data
  This is the primary endpoint the Even G2 home screen will call. Returns everything needed in one request.

  Response structure:
  ```json
  {
    "date": "2026-04-04",
    "counts": {
      "total": 37,
      "byCategory": { "task": 10, "therapy": 5, "idea": 8, ... },
      "tasksByStatus": { "open": 4, "inProgress": 2, "done": 4 },
      "favorites": 3,
      "unprocessed": 2
    },
    "openTasks": [
      { "id": 1, "content": "...", "taskStatus": "open", "createdAt": "...", "tags": [] }
    ],
    "recentThoughts": [
      { "id": 5, "content": "...", "category": "idea", "source": "voice", "createdAt": "...", "tags": ["work"] }
    ],
    "recentTherapy": [
      { "id": 8, "content": "...", "therapyClassification": "bringToTherapist", "createdAt": "...", "tags": [] }
    ],
    "todayCaptures": 3
  }
  ```

  Queries:
  1. Count queries — reuse same pattern as summary.ts (total, byCategory, tasksByStatus, favorites)
  2. Unprocessed count: `WHERE category IS NULL AND syncStatus != 'pendingDeletion'`
  3. Open tasks: `WHERE category = 'task' AND taskStatus IN ('open', 'inProgress') AND syncStatus != 'pendingDeletion' ORDER BY createdAt DESC LIMIT 10` — include full thought fields with parsed tags
  4. Recent thoughts: `ORDER BY createdAt DESC LIMIT 5` with parsed tags (same as summary.ts but with all fields)
  5. Recent therapy: `WHERE category = 'therapy' AND syncStatus != 'pendingDeletion' ORDER BY createdAt DESC LIMIT 5` with parsed tags
  6. Today's capture count: `WHERE date(createdAt) = date('now') AND syncStatus != 'pendingDeletion'`

  Note: This intentionally overlaps with /summary but returns more data and is structured for display. /summary remains as the lightweight health-check-adjacent endpoint.
  </action>
  <verify>curl localhost:3001/v1/brief returns JSON with all sections populated (counts, openTasks, recentThoughts, recentTherapy, todayCaptures)</verify>
  <done>Brief endpoint returns complete aggregated data in one call. All counts accurate. Open tasks and recent items include full fields with parsed tags.</done>

<task type="auto">
  <name>Task 2: Create bulk operation routes and mount all new routes</name>
  <files>vigil-core/src/routes/bulk.ts, vigil-core/src/index.ts</files>
  <action>
  Create src/routes/bulk.ts exporting a Hono instance with:

  **POST /thoughts/bulk/delete** — Bulk soft delete
  - Body: { ids: number[] }
  - Validate ids is non-empty array of numbers
  - Use transaction: UPDATE thoughts SET syncStatus = 'pendingDeletion', modifiedAt = {now} WHERE id IN (...)
  - Use parameterized query with placeholder expansion (one ? per id)
  - Return { deleted: number } with count of affected rows
  - 400 if ids empty or invalid

  **POST /thoughts/bulk/recategorize** — Bulk category change
  - Body: { ids: number[], category: ThoughtCategory }
  - Validate category is valid ThoughtCategory
  - Use transaction: UPDATE thoughts SET category = ?, syncStatus = 'pending', modifiedAt = {now} WHERE id IN (...) AND syncStatus != 'pendingDeletion'
  - Return { updated: number }
  - 400 if ids empty or category invalid

  **POST /thoughts/bulk/tag** — Bulk add/remove tag
  - Body: { ids: number[], tag: string, action: "add" | "remove" }
  - For "add": For each thought, read tags JSON, parse, add tag (deduplicate), stringify back
  - For "remove": For each thought, read tags JSON, parse, filter out tag, stringify back
  - Wrap in transaction for atomicity
  - Update modifiedAt and syncStatus for each affected thought
  - Return { updated: number }
  - 400 if ids empty, tag empty, or action invalid

  Mount both new route modules in src/index.ts:
  ```
  app.route("/v1", brief);
  app.route("/v1", bulk);
  ```

  Important for bulk tag operations: Since SQLite JSON functions don't support in-place array modification easily, read-modify-write each row. Use a transaction so it's atomic. For bulk operations with many IDs, the transaction ensures consistency.
  </action>
  <verify>

  - curl -X POST localhost:3001/v1/thoughts/bulk/recategorize -H 'Content-Type: application/json' -d '{"ids":[1,2],"category":"idea"}' — returns updated count
  - curl -X POST localhost:3001/v1/thoughts/bulk/tag -H 'Content-Type: application/json' -d '{"ids":[1,2],"tag":"urgent","action":"add"}' — returns updated count
  Kill server after testing.
  </verify>
