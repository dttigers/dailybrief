<task type="auto">
  <name>Task 1: Install Drizzle ORM + PostgreSQL driver and define schema</name>
  <files>vigil-core/package.json, vigil-core/src/db/schema.ts</files>
  <action>
  1. Install dependencies:
     ```
     cd vigil-core && npm install drizzle-orm postgres && npm install -D drizzle-kit @types/pg
     ```
     Use `postgres` (porsager's postgres.js) as the driver — it's the recommended driver for drizzle-orm/postgres-js, lightweight, and works well with connection strings for Railway deployment later.

  2. Create `vigil-core/src/db/schema.ts` with Drizzle pgTable definitions:

     **thoughts table:**
     - id: serial('id').primaryKey()
     - content: text('content').notNull()
     - category: text('category') — nullable, one of: task, therapy, idea, reflection, project
     - confidence: doublePrecision('confidence') — nullable
     - source: text('source').notNull() — one of: text, voice, image
     - createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull()
     - modifiedAt: timestamp('modified_at', { withTimezone: true }).defaultNow().notNull()
     - cloudKitRecordID: text('cloudkit_record_id').notNull().unique()
     - syncStatus: text('sync_status').notNull().default('pending')
     - lastSyncedAt: timestamp('last_synced_at', { withTimezone: true })
     - taskStatus: text('task_status') — nullable
     - therapyClassification: text('therapy_classification') — nullable
     - tags: jsonb('tags').$type<string[]>() — use native JSONB instead of TEXT
     - isFavorited: boolean('is_favorited').notNull().default(false) — use native boolean instead of integer
     - searchVector: Use `sql` to define a generated tsvector column for FTS:
       Add a GIN index on the search vector for fast full-text search.
       NOTE: The tsvector column should be maintained via a PostgreSQL trigger (created in the migration SQL), not a Drizzle generated column, since Drizzle doesn't natively support tsvector generated columns. Define it as: `customType` or just handle it in raw migration SQL.

     **thought_links table:**
     - id: serial('id').primaryKey()
     - sourceThoughtId: integer('source_thought_id').notNull().references(() => thoughts.id, { onDelete: 'cascade' })
     - targetThoughtId: integer('target_thought_id').notNull().references(() => thoughts.id, { onDelete: 'cascade' })
     - createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull()
     - Add unique constraint on (sourceThoughtId, targetThoughtId)

     **Indexes to create:**
     - idx_thoughts_cloudkit_record_id (unique, on cloudKitRecordID) — already via .unique()
     - idx_thoughts_created_at on createdAt
     - idx_thoughts_category on category
     - idx_thoughts_sync_status on syncStatus
     - idx_thought_links_source on sourceThoughtId
     - idx_thought_links_target on targetThoughtId

     **Column naming:** Use snake_case in PostgreSQL (created_at, sync_status) but camelCase in TypeScript via Drizzle's mapping. This follows PostgreSQL conventions.

  3. Update `vigil-core/src/db/types.ts`:
     - Export inferred types from schema: `type Thought = typeof thoughts.$inferSelect`
     - Export insert types: `type NewThought = typeof thoughts.$inferInsert`
     - Keep existing type unions (ThoughtCategory, CaptureSource, etc.) for validation
     - Remove the old `Thought` interface (replaced by schema inference)
  </action>

  <done>Schema file exports thoughts and thoughtLinks tables with all columns, indexes, and proper TypeScript types. Types file exports inferred select/insert types.</done>

<task type="auto">
  <name>Task 2: Create database connection module and migration config</name>
  <files>vigil-core/src/db/connection.ts, vigil-core/drizzle.config.ts, vigil-core/.env.example</files>
  <action>
  1. Create `vigil-core/src/db/connection.ts`:
     ```typescript
     import { drizzle } from 'drizzle-orm/postgres-js';
     import postgres from 'postgres';
     import * as schema from './schema';
     ```
     - Read DATABASE_URL from process.env (required)
     - Create postgres client: `const client = postgres(process.env.DATABASE_URL!)`
     - Export db instance: `export const db = drizzle(client, { schema })`
     - Export a `testConnection()` async function that runs `SELECT 1` to verify connectivity
     - Add a `closeConnection()` function for graceful shutdown: `await client.end()`

     Do NOT create a lazy/singleton pattern like the old getDb(). Drizzle with postgres.js handles connection pooling internally. Just export the db instance directly.

  2. Create `vigil-core/drizzle.config.ts`:
     ```typescript
     import { defineConfig } from 'drizzle-kit';
     export default defineConfig({
       schema: './src/db/schema.ts',
       out: './drizzle',
       dialect: 'postgresql',
       dbCredentials: {
         url: process.env.DATABASE_URL!,
       },
     });
     ```

  3. Create `vigil-core/.env.example`:
     ```
     DATABASE_URL=postgresql://localhost:5432/vigil
     ANTHROPIC_API_KEY=your-key-here
     ```

  4. Add scripts to package.json:
     ```json
     "db:generate": "drizzle-kit generate",
     "db:migrate": "drizzle-kit migrate",
     "db:push": "drizzle-kit push",
     "db:studio": "drizzle-kit studio"
     ```

  5. Generate initial migration:
     ```
     cd vigil-core && DATABASE_URL=postgresql://localhost:5432/vigil npx drizzle-kit generate
     ```
     This creates SQL migration files in vigil-core/drizzle/ directory.

  6. Manually append to the generated migration SQL (or create a separate custom migration):
     - CREATE the tsvector trigger for full-text search:
       ```sql
       -- Add tsvector column
       ALTER TABLE thoughts ADD COLUMN search_vector tsvector
         GENERATED ALWAYS AS (to_tsvector('english', coalesce(content, ''))) STORED;

       -- Create GIN index for fast FTS
       CREATE INDEX idx_thoughts_search_vector ON thoughts USING gin(search_vector);
       ```
       NOTE: If Drizzle's generate already created the column from the schema definition, just add the GIN index. If using a generated column in the schema didn't work, add it via custom SQL in the migration.

  7. Do NOT modify the old `vigil-core/src/db/index.ts` yet — it will be removed in Plan 37-04 after all routes are migrated. Both modules can coexist temporarily.
  </action>
  <verify>
  - cd vigil-core && npx tsc --noEmit (all files compile)
  - ls vigil-core/drizzle/*.sql (migration files exist)
  - cat vigil-core/drizzle.config.ts (config exists and is valid)
  </verify>
  <done>Connection module exports db instance with postgres.js driver. drizzle-kit config exists. Initial migration generated with thoughts + thought_links tables + FTS tsvector column + GIN index. Package.json has db scripts.</done>
