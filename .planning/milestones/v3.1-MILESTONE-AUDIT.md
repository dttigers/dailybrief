## Cross-Phase Integration Findings

Integration paths traced across the four v3.1-close phases (83, 84, 86, 87):

**Status:** WIRED and VERIFIED.

- Phase 83-01 created `app_settings` table + `GET/PUT /v1/settings/print-schedule` via `createSettingsRouter` DI factory.

- **No contract drift** — Phase 83-04's `BriefScheduler.reschedule(hour:minute:enabled:)` overload is unchanged; Phase 86 does not touch `BriefScheduler`.

### 2. Phase 83 (MenuBarExtra-only) ↔ Phase 86 (StatusChecker staleness UI)

**Status:** WIRED and VERIFIED.

- Zero `setActivationPolicy` regressions from Phase 86's menubar additions (grep clean per 86-VERIFICATION).
- **StatusChecker log-marker inference (86-06)** correctly drives the UI surface Phase 83-02 built; `runNow()` contract (MainActor write of `lastExitCode`) preserved byte-identical.

### 3. Phase 83 (MenuBarExtra-only / LSUIElement=true) ↔ Phase 87 (AppIcon.icns)

**Status:** WIRED and VERIFIED — regression guard held.
