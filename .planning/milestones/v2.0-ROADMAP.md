# Milestone v2.0: Vigil Platform

**Status:** ✅ SHIPPED 2026-04-04
**Phases:** 29-36
**Total Plans:** 22

## Overview

Extract intelligence into a platform-agnostic API (Vigil Core), build first external client (Even G2 smart glasses), and migrate the Mac app to be a thin client. Mac app remains functional throughout migration. Vigil Core runs on localhost only for v2.0 (no server deployment, no auth). API versioned from day one (/v1/ prefix on all routes).

## Milestone Summary

**Key Decisions:**
- Hono over Express for API framework (lightweight, TypeScript-first, edge-ready)
- better-sqlite3 for direct Jarvis DB access (no ORM, shared SQLite file)
- Protocol abstraction pattern for Mac app migration (ThoughtRepository + AI service protocols)
- Config toggle (vigil.useAPI) for gradual migration vs big-bang switch
- snake_case config keys for Swift JSONEncoder compatibility
- Circular navigation state machine for G2 (simple, predictable UX)
- Daily file cache for AI endpoints (affirmation, prioritization) to avoid redundant API calls

**Issues Resolved:**
- Swift async throws migration required for protocol-based network operations
- Even Hub SDK bridge pattern for TypeScript integration
- G2 display constraints (576x288, 4-bit greyscale, max 8 containers/page)
- Config key format (snake_case vs camelCase) for cross-platform compatibility

**Issues Deferred:**
- G2 hardware testing (awaiting physical glasses delivery)
- Server deployment (Vigil Core runs localhost-only for v2.0)
- CKSubscription push notifications (polling-based sync still in use)

**Technical Debt Incurred:**
- Dual code paths (local + API) maintained via config toggle — cleanup when API is proven stable
- G2 plugin uses mock fallback when API unreachable — should show error state in production
