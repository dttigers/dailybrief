## Decisions Made

- **VITE_API_BASE silencing form: COMMENTED (not removed, not empty-value).** Plan 02 allowed any of three: (a) remove line, (b) comment with `#`, or (c) set to empty `VITE_API_BASE=`. Chose (b) — `# VITE_API_BASE=` with an inline explanation of why it must not be uncommented. Reasoning: matches the self-documenting LOCAL-ONLY convention from Phase 107.1 Plan 06 `vigil-core/.env.example`; a removed line leaves no audit trail for a future maintainer who wonders if the key was ever supposed to be there.
- **Rule 2 auto-add on .gitignore.** The existing `.env.bak.*` + `*.env.bak.*` patterns didn't match `.env.local.bak*` variants. This plan creates a new backup (`.env.local.bak.20260422-125525`), so I extended the pattern family. Both pre-existing and new backups are now covered. Scope boundary check: caused by this plan's actions (new backup file introduction), therefore in-scope.
- **Pre-existing TS6305 stale-dts errors logged as deferred.** `npx tsc --noEmit` exits 2 with 64 TS6305 errors about stale `.d.ts` output files. Verified pre-existing via `git stash push vite.config.ts && npx tsc --noEmit` — same 64 errors before my edit. Root cause: `tsconfig.app.json` has `composite: true` which conflicts with running `tsc --noEmit` at the root level. This is a pre-existing infra issue, not caused by my edit. Logged to deferred-items for a future hygiene plan. Plan's other verification (`npm run build` exit 0) compensates — build cleanly accepts the function-form config and emits dist/ with PWA service worker.
