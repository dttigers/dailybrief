Every successful response sets PostHog person properties (`email`, `createdAt`) for the
authenticated `userId` so the PostHog person record stays fresh on every PWA app load
(Phase 104 D-15: PWA hits `/v1/me` on mount) and every Mac/CLI vk_ call (D-11: vk_ →
seed user mapping makes the seed user identify on every Mac action).

on login). Server-side identify is the canonical fresh source; the PWA's existing
`identifyUser(userId, email)` (Phase 104 D-12) stays as the fast client-side identification.
Last-write-wins, no race because both pass the same userId — and now the server adds
`createdAt` for cohort analysis.
