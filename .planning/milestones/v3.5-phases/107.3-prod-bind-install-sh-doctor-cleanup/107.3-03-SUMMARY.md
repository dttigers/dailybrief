## Decisions Made

- **Retirement signal = in-repo marker file, not env var or external state.** The file `107.1-daemon-retirement.md` is already the canonical record of the daemon retirement (created by Plan 107.1-04). Presence-checking it via `[[ -f "$RETIREMENT_MARKER" ]]` means the retirement signal travels with the repo — no host-specific state to configure, no drift between machines.
- **Informational printf instead of print_row.** `print_row` requires a MATCH column (`✓`/`✗`). The retired state is not a binary match — it's a third state. Using a two-column `printf "%-38s | %s\n"` matches the D-13 local-DB row idiom already established in this script (commit `ad37bd5`). Keeps the formatting conventions consistent.
- **Three-way branch, not row removal.** Dropping the row entirely (the simplest possible fix) would lose the ability to detect accidental plist deletion. The third `else` branch preserves that safety net.
- **`$REPO_DIR` in scope — no fallback needed.** The plan offered a `$SCRIPT_DIR/..` fallback in case `$REPO_DIR` wasn't defined; `grep -n '^REPO_DIR=' Scripts/dailybrief-doctor.sh` confirmed line 25 sets it, so the primary form was used.
