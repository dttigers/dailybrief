    Run these commands in Terminal.app on **both** the iMac and the MacBook Pro (per D-03):

    ```bash
    # 1. Install postgresql@16 (Homebrew formula, version 16.13 — matches Railway prod per RESEARCH Summary)
    brew install postgresql@16

    # 2. Start the brew service (sticky — persists across reboots per D-16; one-time only)
    brew services start postgresql@16

    # 3. Add keg-only binaries to PATH for this shell (permanent PATH update can go in ~/.zshrc later)
    export PATH="/usr/local/opt/postgresql@16/bin:$PATH"

    # 4. Wait briefly for Postgres to accept connections, then create the database
    sleep 2
    createdb vigil_dev

    # 5. Verify
    psql -h localhost -d vigil_dev -c "SELECT version();"
    ```

    **Per D-11:** Postgres listens on 5432 (default); vigil-core will listen on 3001 (separate concern, Plan 05). Confirm no collision with `lsof -iTCP:5432 -sTCP:LISTEN` — should show `postgres` as the listener.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| brew installer → local filesystem | Homebrew formula runs `initdb` with default settings; trusts formula publisher (acceptable — brew is the primary package manager on macOS dev) |
| Postgres listener → local network | postgresql@16 binds to localhost by default; never exposed to LAN/WAN |
| .env.example → developer .env | Template is a starting point; developer fills real values and `.env` is gitignored |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-LOCAL-1 | I (Info Disclosure) | vigil-core/.env rewrite | accept (deferred to Plan 06) | Plan 01 only updates `.env.example` — the prod-values-in-.env rewrite is Plan 06's job. This plan does not touch `.env`. |
| T-LOCAL-4 | S (Spoofing / Allowlist bypass) | VIGIL_ALLOWED_EMAILS template | mitigate | `.env.example` pins VIGIL_ALLOWED_EMAILS to the real user's seed email; Postgres binds `listen_addresses = 'localhost'` (brew default, unchanged) — `vigil_dev` not reachable from LAN even if allowlist were public knowledge |
| T-LOCAL-5 | I (Secret commit risk) | vigil-core/.env.example | mitigate | Template contains only placeholders (`replace-me-with-*`, `sk-ant-dev-workspace-key-here`) — no real secrets; .env is already in root .gitignore (line 9-12) and that coverage is verified in Plan 06 Task 1 acceptance_criteria |
</threat_model>
