**popup.html changes (per D-05, D-07):**

- Add `.category-badge` style: inline-block, background rgba(29,158,117,0.15), color #1D9E75, border-radius 12px, padding 2px 10px, font-size 12px, font-weight 500.
- Add `.shortcut-hint` style: font-size 11px, color #666, text-align center, margin-top 4px.
- Add `.analyzing` style: color #999, font-size 13px (for the "Analyzing..." state).

**popup.js changes:**

1. **DOM references** — Add refs for: `includeUrlCheckbox` (#include-url), `successText` (#success-text).

2. **Keyboard shortcut (per D-08, D-09):** Add a keydown listener on `contentInput`:
   ```
   contentInput.addEventListener('keydown', (e) => {
     if (e.key === 'Enter' && (e.metaKey || e.ctrlKey)) {
       e.preventDefault();
       captureBtn.click();
     }
   });
   ```
   Plain Enter naturally inserts newlines in a textarea (D-09) -- no code needed for that.

3. **URL inclusion (per D-05, D-06):** In the capture button onclick handler, after getting `content` from the textarea:
   ```
   let finalContent = content;
   if (includeUrlCheckbox.checked) {
     const [tab] = await chrome.tabs.query({ active: true, currentWindow: true });
     if (tab?.url) {
       finalContent += `\n\n${tab.title || 'Page'}: ${tab.url}`;
     }
   }
   ```
   Use `finalContent` in the POST body instead of `content`.

4. **Triage feedback flow (per D-01, D-02, D-03, D-04):** After the POST /v1/thoughts succeeds (201):
   - Parse the response to get `thought.id`.
   - Show success area with `successText.innerHTML = '<span class="analyzing">Analyzing...</span>'` (D-01).
   - Start a polling loop: every 800ms, call `GET /v1/thoughts/${thought.id}` with the auth header. Check if `response.category` is non-null.
   - **On category found (D-02, D-03):** Display `successText.innerHTML = '<span class="checkmark">&#10003;</span> Captured! <span class="category-badge">${capitalize(category)}</span>'`. Then `setTimeout(() => window.close(), 1500)`.
   - **On timeout (D-04):** After 5s total from the POST response, stop polling. Show `successText.innerHTML = '<span class="checkmark">&#10003;</span> Captured!'` and `setTimeout(() => window.close(), 1500)`.
   - **On poll error:** Treat as timeout -- show "Captured!" fallback.
   - Use `AbortController` or a simple flag to cancel polling on timeout.

   Implementation pattern:
   ```javascript
   const thought = await res.json();
   captureSuccess.hidden = false;
   successText.innerHTML = '<span class="analyzing">Analyzing...</span>';

   const startTime = Date.now();
   const pollInterval = setInterval(async () => {
     if (Date.now() - startTime > 5000) {
       clearInterval(pollInterval);
       successText.innerHTML = '<span class="checkmark">&#10003;</span> Captured!';
       setTimeout(() => window.close(), 1500);
       return;
     }
     try {
       const pollRes = await fetch(`${API_BASE}/v1/thoughts/${thought.id}`, {
         headers: { 'Authorization': `Bearer ${apiKey}` },
       });
       if (pollRes.ok) {
         const updated = await pollRes.json();
         if (updated.category) {
           clearInterval(pollInterval);
           const cat = updated.category.charAt(0).toUpperCase() + updated.category.slice(1);
           successText.innerHTML = `<span class="checkmark">&#10003;</span> Captured! <span class="category-badge">${cat}</span>`;
           setTimeout(() => window.close(), 1500);
         }
       }
     } catch { /* ignore poll errors, timeout will handle */ }
   }, 800);
   ```

5. **Remove the old static success display** -- the old `captureSuccess.hidden = false` + `setTimeout(close, 1500)` block is replaced by the new triage feedback flow above. Remove the old static 1.5s auto-close.

6. **Reset button state** after POST completes (whether entering poll or hitting error): `captureBtn.textContent = 'Capture'; captureBtn.disabled = false;`

**Do NOT:**
- Add any background scripts or service workers (unnecessary for this scope).
- Change the manifest permissions -- `activeTab` + `storage` are sufficient.
- Modify any vigil-core server code -- the existing fire-and-forget triage + GET endpoint are all that's needed.

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-94-02 | Spoofing | API key in storage | accept | Existing pattern from Phase 84; chrome.storage.local is per-extension isolated |
