## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding (`px-2 py-0.5`) |
| sm | 8px | Compact element spacing, gap between header actions |
| md | 16px | Default element spacing, card internal padding (`p-4`) |
| lg | 24px | Section padding, gap between page sections (`mb-6`, `mt-6 pt-6`) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Regenerate button touch target: minimum 32px height (`py-1.5` at `text-xs`) — acceptable for a secondary action adjacent to primary content.
- "New Chat" header button: `text-xs` text-only link — no explicit touch target minimum; this is an existing established pattern in ChatPage.

Source: `InsightsPage.tsx`, `TherapyPage.tsx`, `ChatPage.tsx` — all use `p-4` for cards, `mb-6` for header bottom margin, `mt-6 pt-6` for section dividers.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F9F9F8` (`color-surface`) | Page background |
| Secondary (30%) | `#2C2C2A` (`color-gray-900`) | Cards (`bg-gray-900/50`), input fields (`bg-gray-900/80`), session list items |
| Accent (10%) | `#0F6E56` (`color-teal-600`) | See reserved list below |
| Destructive | `bg-red-900/50` / `text-red-300` | Error banners only |

Accent (`teal-600`) reserved for:
1. Primary CTA generate/regenerate buttons (`bg-teal-600`)
2. Active chat session highlight (`bg-teal-600/20 text-teal-400`)
3. Session prep "Suggested focus" block border (`border-teal-600/30`) and label (`text-teal-400`)
4. User chat bubble background (`bg-teal-600 text-white`)
5. Focused input border (`focus:border-teal-600`)

The Regenerate button (secondary action shown only when cache exists) uses `bg-gray-900/80` — NOT teal — to visually subordinate it to the primary generate action. Source: `90-RESEARCH.md` code example for the Regenerate button.

Hover state on Regenerate: `hover:bg-gray-800`.

## Component Inventory

### New: Regenerate button + timestamp row

Shown in the header of InsightsPage and TherapyPage when `isCached === true`.

```
[Generated 2h ago]   [Regenerate]
```

- Container: `flex items-center gap-3`
- Timestamp: `text-xs text-gray-400`
- Button: `bg-gray-900/80 hover:bg-gray-800 disabled:opacity-40 text-gray-100 px-3 py-1.5 rounded-lg text-xs font-medium transition-colors border border-gray-400/20`

When `isCached === false` AND `isLoading === false`: show the existing primary teal "Generate Insights" / "Analyze Patterns" / "Generate Prep" button (no change from current).

When `isLoading === true` (regenerate in progress):
- Clear existing result list (empty array in state per D-05)
- Show existing skeleton/pulse loading state (no new component needed)
- Button disabled at `opacity-40`

### Modified: InsightsPage header

Before (current):
```
[h1: Insights]                    [Generate Insights button]
[subheading: Analyzing last 7d]
```

After (Phase 90, when cached):
```
[h1: Insights]           [Generated 2h ago]  [Regenerate]
[subheading: Analyzing last 7d]
```

After (Phase 90, first visit / no cache):
```
[h1: Insights]                    [Generate Insights button]
[subheading: Analyzing last 7d]
```

Same structural change applies to both TherapyPage sections (Therapy Patterns + Session Prep).

### Modified: ChatPage auto-resume behavior

On mount with existing sessions: no visible UI change. The most recent session loads automatically. Prior messages appear in the message list.

"New Chat" button remains in header — same position, same style (`text-xs text-gray-400 hover:text-gray-100`).

No loading indicator for auto-resume (sessions are small; load is near-instant). If sessions[0] fails to load, fall back silently to blank state (error handling already in useChat).

## Interaction States

### Insights / Therapy pages — state machine

| State | Trigger | UI |
|-------|---------|-----|
| First visit, no cache | Mount, server returns fresh result | Auto-generate; show existing skeleton → cards |
| Cached result | Mount, server returns cache hit | Cards display instantly; Regenerate + timestamp row visible |
| Regenerating | User clicks Regenerate | Cards clear; existing skeleton shown; button disabled |
| Regenerate complete | Server returns new result | New cards replace skeleton; timestamp updates |
| Error | Network or AI failure | Existing `bg-red-900/50 text-red-300` error banner; Regenerate button re-enabled |

### Chat page — auto-resume state machine

| State | Trigger | UI |
|-------|---------|-----|
| PWA opens, sessions exist | Mount | Most recent session messages load silently |
| PWA opens, no sessions | Mount | Existing blank state: "Ask Vigil anything..." |
| User clicks "New Chat" | User action | Clears messages, clears activeSessionId (existing clearChat behavior) |

## Relative Timestamp Format

Inline utility — no external library. Implement as a pure function `formatRelativeTime(iso: string): string`:

| Elapsed | Output |
|---------|--------|
| < 1 minute | `just now` |
| 1–59 minutes | `{N}m ago` |
| 1–23 hours | `{N}h ago` |
| 1+ days | `{N}d ago` |

Source: `90-RESEARCH.md` "Don't Hand-Roll" note confirms `date-fns` is absent from `vigil-pwa/package.json`. Use inline calculation.
