## Layout & Interaction Contract

### Week Header

**Location:** Inserted between CategoryTabs and the ThoughtList div (inside the `mt-4` div in ThoughtsPage.tsx). Lives in ThoughtsPage.tsx, not ThoughtList.tsx.

**Structure:**
```
[compact single-row label]
This week  ·  Wed Apr 9 – Tue Apr 15
```

**Exact HTML element:** `<p>` tag with role="status" aria-live="polite" so screen readers announce when it swaps to the search header.

**Dimensions:** height 20px, margin-bottom 8px (mb-2). Full width, no border, no background. Left-aligned, matching the left edge of the thought list.

**Visibility rule:**
- When `debouncedQuery === ''` (no active search): show week header.
- When `debouncedQuery !== ''` (search active): show search header. Week header is not rendered (conditional render, not CSS hide).

### Search Header

**Location:** Same slot as week header — mutually exclusive render.

**Structure:**
```
Search  ·  all time
```

**Transition:** No animation. Instant swap on the same render cycle as the query debounce update (300ms after keystroke). Clears instantly when input clears back to empty.

### Date Formatting

**Format pattern:** `{3-letter day abbrev} {Month abbrev} {D}` using `Intl.DateTimeFormat` with options `{ weekday: 'short', month: 'short', day: 'numeric' }` in the user's configured timezone.

**Examples:**
- `Wed Apr 9 – Tue Apr 15`
- `Wed Apr 16 – Tue Apr 22`

**Locale:** Use `navigator.language` for locale-aware formatting (matches existing browser environment; server returns tz string, client formats for display).

**Separator:** ` – ` (space, en-dash, space) between start and end dates.

### Empty State

**Trigger:** `thoughts.length === 0` AND `isLoading === false` AND `error === null` AND `debouncedQuery === ''` (week window is active, not search).

**Location:** Rendered inside ThoughtList.tsx, replacing the existing `"No thoughts found"` string.

**Layout:** Vertically centered in the list area (`py-12` = 48px top/bottom padding), text centered (`text-center`).

**Search-active empty state:** When `thoughts.length === 0` and a search query IS active, use the existing generic copy "No thoughts found" — do not use the week-specific copy in search context.

## Copywriting Contract

All copy is locked. No ad-hoc strings. Executor must use these exact strings.

| Element | Exact Copy |
|---------|------------|
| Week header (label portion) | `This week` |
| Week header (separator) | ` · ` |
| Week header (date range) | `{Wed MMM D} – {Tue MMM D}` (formatted per date contract above) |
| Search header (label portion) | `Search` |
| Search header (separator + scope) | ` · all time` |
| Empty state heading | `No thoughts this week yet` |
| Empty state body | `Capture one above to get started.` |
| Empty state hint | `Looking for older thoughts? Search above.` |
| Loading state (existing, unchanged) | `Loading thoughts...` |
| Error state (existing, unchanged) | `{error}` (passed directly from hook) |
| Generic empty (search active, no results) | `No thoughts found` (existing — unchanged) |

**Formatting rules:**
- No trailing period on the week header or search header.
- Empty state uses sentence case throughout.
- "Search above" in the hint is literal instruction — do not change to "use search" or "try searching."
- The hint line is secondary and visually de-emphasized (smaller opacity, same size as empty state body).

**Destructive actions:** None in this phase. No confirmation dialogs, no destructive copy required.

New and modified components this phase. No new dependencies introduced.

**New element in ThoughtsPage.tsx (exact structure):**

```tsx
{/* Week / Search context header */}
{debouncedQuery === '' ? (
  <p
    className="text-xs mb-2 text-left"
    role="status"
    aria-live="polite"
  >
    <span className="text-teal-400 font-medium">This week</span>
    <span className="text-gray-400"> · {formattedStart} – {formattedEnd}</span>
  </p>

  <p
    className="text-xs mb-2 text-left"
    role="status"
    aria-live="polite"
  >
    <span className="text-teal-400 font-medium">Search</span>
    <span className="text-gray-400"> · all time</span>
  </p>
)}
```

**Modified empty state in ThoughtList.tsx:**

```tsx
if (thoughts.length === 0) {
  // isSearchActive prop: true when caller has an active query
  if (isSearchActive) {
    return (
      <div className="text-gray-400 text-center py-12">
        No thoughts found
      </div>
    )
  }
  return (
    <div className="text-center py-12 space-y-1">
      <p className="text-gray-400 text-sm">No thoughts this week yet</p>
      <p className="text-gray-400 text-sm">Capture one above to get started.</p>
      <p className="text-gray-400/70 text-xs mt-2">Looking for older thoughts? Search above.</p>
    </div>
  )
}
```

ThoughtList requires a new `isSearchActive: boolean` prop to enable the branching. Planner must account for this prop addition in the interface and callsite.
