<!-- Backend chat endpoint contract (from vigil-core/src/routes/chat.ts) -->

POST /v1/chat
Request body:
```json
{
  "messages": [{ "role": "user" | "assistant", "content": "string" }],
  "includeContext": true,       // optional, default true
  "contextLimit": 20            // optional, 1-50, default 20
}
```

Response (200):
```json
{
  "response": "string",        // Claude's reply text
  "contextUsed": 0             // number of thoughts injected
}
```

Error responses: 400 (bad body), 502 (AI failure), 503 (AI unavailable)

NOTE: This is NOT a streaming endpoint. It returns the full response in one shot.
The PWA should show a loading/typing indicator while waiting, then render the
full response when it arrives.

   Append a new "Chat API" section after the Work Orders section. Define:
   ```typescript
   export interface ChatMessage {
     role: 'user' | 'assistant'
     content: string
   }

   export interface ChatResponse {
     response: string
     contextUsed: number
   }

   export async function sendChatMessage(
     messages: ChatMessage[],
     includeContext = true,
   ): Promise<ChatResponse> {
     const res = await vigilFetch('/v1/chat', {
       method: 'POST',
       body: JSON.stringify({ messages, includeContext }),
     })
     if (!res.ok) throw new Error(`Chat failed: ${res.status}`)
     return res.json()
   }
   ```

2. **Create `vigil-pwa/src/hooks/useChat.ts`** (per CHAT-01):
   Manage in-memory conversation history. Exports:
   ```typescript
   export function useChat(): {
     messages: ChatMessage[]
     isLoading: boolean
     error: string | null
     contextUsed: number
     sendMessage: (content: string) => Promise<void>
     clearChat: () => void
   }
   ```
   Implementation:
   - `messages` state: `ChatMessage[]` starting empty
   - `sendMessage(content)`: appends `{ role: 'user', content }` to messages, sets isLoading=true, calls `sendChatMessage` with the full messages array (including the new user message), appends `{ role: 'assistant', content: response.response }` to messages, updates contextUsed, sets isLoading=false. On error, set error string and remove the user message that failed (or keep it and show error inline).
   - `clearChat()`: resets messages to [], contextUsed to 0, error to null
   - Use `useCallback` for sendMessage and clearChat to keep stable references
   - History is in-memory only — no localStorage persistence (per phase scope "in-memory is fine for v2.5")
