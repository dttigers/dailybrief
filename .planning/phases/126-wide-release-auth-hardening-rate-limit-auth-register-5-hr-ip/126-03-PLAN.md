- Import `import * as Sentry from "@sentry/node"`.
- Module-scope `let initialized = false`.
- `export function initSentry(): void`:
  - Read `process.env["SENTRY_DSN"]`. If absent, return (no-op).
  - Call `Sentry.init({ dsn, environment: process.env["NODE_ENV"] ?? "development", tracesSampleRate: 0 })`.
  - Set `initialized = true`.
  - Do NOT enable `sendDefaultPii` (RESEARCH §Security Domain — Bearer leak via breadcrumbs mitigation).
- `export function captureToSentry(userId: number | string | null, err: unknown, context: Record<string, string | number | boolean | undefined> = {}): void`:

  - Normalize `error = err instanceof Error ? err : new Error(String(err))`.
  - Use v10 functional API: `Sentry.withScope((scope) => { if (userId !== null) scope.setUser({ id: String(userId) }); scope.setContext("request", context); Sentry.captureException(error); })`.
  - Mirror parameter shape from `analytics/posthog.ts:181-192` exactly so call sites in `index.ts` can call both with identical args.

Do NOT use `Sentry.Hub`, `Sentry.configureScope`, or any pre-v8 API (R9). Do NOT call `Sentry.flush()` on every capture (perf — let SDK batch).
