<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server response → PWA `body.code` | PWA trusts server-emitted code field; resolveApiError treats unknown codes as forward-compat (raw error fallback) |
| ERROR_CODE_MAP → user UX | Static lookup table; no user input crosses this boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-126-07-01 | Information Disclosure | Server error string contains stack trace or PII | mitigate | Server side (Plan 05) emits curated `error` strings (existing convention — auth.ts lines 96-226 verified). PWA falls back to raw `error` only when code is unknown; if server changes shape, fallback string is shown. Operator review pre-launch. |
| T-126-07-02 | Tampering | Future code added to ERROR_CODE_MAP that ships a malicious URL via ctaHref | accept | Source-controlled; PR review catches. mailto:hello@vigilhub.io and internal `/auth` `/settings` are the only outbound targets today; absolute URLs disallowed by convention. |
| T-126-07-03 | Spoofing | Server emits an unknown code claiming to be sensitive | mitigate | Unknown-code path falls back to raw `error` string. Server is the source of truth; if compromised, both code AND error are untrusted equally. No new attack surface. |
| T-126-07-04 | Repudiation | A locked code is silently removed from ERROR_CODE_MAP | mitigate | Wave 0 test `AUTH-126-CODE-MAP-LOCKED-ENUM` asserts all 9 LOCKED keys exist. Removal breaks CI. |
| T-126-07-05 | Information Disclosure | @sentry/react bundles DSN into JS — DSN leak via DevTools | accept | DSNs are public-by-design (Sentry threat model — RESEARCH §Security Domain). DSN ≠ secret. Documented in Plan 09 (where init lives). |
| T-126-07-06 | Information Disclosure | "Invalid email or password" UX surfaces for a token-subject-not-found condition (would expose D-04 enum confusion) | mitigate | INVALID_TOKEN_SUBJECT maps to "Session expired" UX with sign-in CTA. Plan 04 emits the distinct code; this plan maps it. No conflation with INVALID_CREDENTIALS. |
</threat_model>
