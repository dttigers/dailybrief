## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → POST /auth/register | Untrusted client input crosses here; gates apply in order: rate-limit → captcha → shape → allowlist → DB |
| /auth/register handler → Cloudflare siteverify | Outbound TLS POST; secret env-only; failure = 503 (no fail-open) |
| /auth/register handler → users DB write | Last gate; protected by argon2 hash + allowlist + captcha + rate-limit upstream |
| Environment → isAllowlistedEmail | VIGIL_ALLOWED_EMAILS env var is the kill-switch; default unset = closed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-126-05-01 | Denial of Service | Bot-scripted /auth/register flood | mitigate | RATE_LIMIT_MAX_IP=20/hr + RATE_LIMIT_MAX_EMAIL=5/hr dual-counter (D-03). Turnstile (D-01) adds anti-automation layer. Defense in depth. |
| T-126-05-02 | Denial of Service | IP-rotating distributed bots | mitigate | Email-axis cap of 5/hr catches one-email/many-IPs attack. Turnstile siteverify per attempt makes large-scale automation costly. |
