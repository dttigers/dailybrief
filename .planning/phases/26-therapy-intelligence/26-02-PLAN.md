**DashboardViewModel.swift:**
1. Add `private let therapyClassificationService: TherapyClassificationService?` property. Accept in init.
2. Create private helper `classifyTherapyIfNeeded(_ thought: Thought) async` that:
   - Checks `thought.category == .therapy && thought.therapyClassification == nil`
   - If true, calls `therapyClassificationService.classify(thought.content)`
   - Updates the thought with `therapyClassification` and stores the reasoning (log it for now)
   - Calls `store.update(thought)`
3. Call `classifyTherapyIfNeeded` after every successful triage that results in `.therapy` category. Wire into these existing flows:
   - After single `reTriageThought` (around line 429-434)
   - After bulk `bulkRetriage` (around line 565-569)
   - After file import triage (around lines 355-360 and 388-392)
4. Add `reClassifyTherapy(_ thought: Thought) async` method for manual re-classification of a therapy thought. Follow the `reTriageThought` pattern — set a `reclassifyingThoughtId` state var, call classify, update, reload.

**FolderWatcherService.swift:**
1. Add optional `therapyClassificationService: TherapyClassificationService?` property. Accept in init.

After triage succeeds in `processAudioFile` and `processImageFile` (lines ~248-254, ~283-289), if the triage result category is `.therapy`, call `therapyClassificationService.classify()` and update the thought's `therapyClassification` field. **ThoughtRowView.swift:**
1. For therapy-category thoughts that have a `therapyClassification`, display a small badge/pill next to the existing category pill:
   - `selfLearnable`: Green pill with "Self-work" text and a book icon (SF Symbol: `book.closed`)
   - `bringToTherapist`: Orange pill with "Therapist" text and a person icon (SF Symbol: `person.fill.questionmark`)
2. If therapy thought has no classification yet, show a subtle "Classifying..." or no badge (it will auto-classify on next triage).
3. Add the `reasoning` display: when the user hovers over the therapy classification badge, show the reasoning string as a tooltip. Store reasoning as a transient display value — since we don't persist reasoning in DB, show it only during session when available. For persisted thoughts, just show the badge without tooltip.
4. Add a "Re-classify" context menu item for therapy thoughts (alongside existing "Re-triage"). Wire to `onReClassify` callback.

**DashboardView.swift:**
1. In the sidebar, under the existing "Therapy" category row, add two indented sub-filter rows (only visible when Therapy is the selected category or always visible under Therapy):
   - "Self-work" with count badge — filters to `therapyClassification == .selfLearnable`
   - "Therapist" with count badge — filters to `therapyClassification == .bringToTherapist`
   - "Unclassified" with count badge — filters to therapy thoughts with nil classification
2. Use SF Symbols matching the badges: `book.closed` for self-work, `person.fill.questionmark` for therapist.
3. Wire the filter selection to DashboardViewModel.

**DashboardViewModel.swift:**
1. Add `@Published var therapyFilter: TherapyClassification?` (nil = all therapy thoughts).
2. Add therapy classification counts: `selfLearnableCount`, `bringToTherapistCount`, `unclassifiedTherapyCount`.
3. In `loadCounts()`, add queries for therapy sub-counts (count where category=therapy AND therapyClassification = each value, plus count where category=therapy AND therapyClassification IS NULL).
4. In `loadThoughts()`, when `selectedCategory == .therapy` and `therapyFilter != nil`, filter results by `therapyClassification`.
5. Wire the `reClassifyTherapy` callback from ThoughtRowView through DashboardView.

**Color/styling notes:**
- Self-work green: `.green` (matches reflection category vibe — independent growth)
- Therapist orange: `.orange` (warm, attention-drawing but not alarming)
- Keep badges small — they're supplementary to the main category pill, not replacing it
- Add a small disclaimer text at the bottom of the therapy sub-filter section: "AI suggestions — not clinical advice" in `.secondary` color, `.caption` font
