# Phase 999.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
>
> **Profile:** integration-heavy operator phase. Almost no unit-testable surface —
> the change is a 3-line plist + a portal capability + a signing pipeline edit.
> Sampling rate is therefore "after each install attempt" rather than "after each
> commit," and the canonical probes are shell commands (`codesign`, `swift run`,
> log greps) rather than test framework runs.

---

## Sampling Rate

- **After every entitlements/Plist edit:** Run `swift build -c release` to catch obvious breakage
- **After every install.sh edit:** Run `./Scripts/install.sh` end-to-end on the throwaway branch
- **After install completes:** Run the canonical UBIQ probe sequence (UBIQ-01 → UBIQ-05 in order)
- **Before `/gsd:verify-work`:** All probes green; HEIC drop test produces materialization in <10s
- **Max feedback latency:** ~90s per install/probe cycle

## Per-Requirement Verification Map

Each phase requirement maps to one or more deterministic probes. The probes are ordered: a later requirement should not be checked until earlier ones pass.

| REQ | Probe | Expected Output | Failure Mode |
|-----|-------|-----------------|--------------|
| UBIQ-01 | `codesign -d --entitlements - ~/.local/bin/DailyBriefMonitor.app` | `<key>com.apple.developer.ubiquity-container-identifiers</key>` + `<array><string>iCloud.com.jamesonmorrill.dailybriefmonitor</string></array>` present | Empty `<dict></dict>` → entitlements file wasn't passed to codesign |
| UBIQ-04 | `codesign --verify --verbose ~/.local/bin/DailyBriefMonitor.app` | `valid on disk` + `satisfies its Designated Requirement` | Signature broken — most likely cause: missing or invalid `embedded.provisionprofile` |
| UBIQ-04 (extra) | `spctl -a -vv ~/.local/bin/DailyBriefMonitor.app` | `accepted` + `source=Developer ID` (notarized) or `source=Developer ID` (if not notarized yet) | `rejected` → Gatekeeper / notarization regression |
| UBIQ-05 | `launchctl print gui/$(id -u)/com.jamesonmorrill.dailybriefmonitor` (10s after install) | `state = running`, `last exit code = (never exited)`, no `last exit reason` field | `last exit code = 9 (KILLED)` or amfid `Invalid Code Signature` in `log show --predicate 'process == "amfid"' --last 5m` → restricted-entitlement violation, profile mismatch |
| UBIQ-02 | `swift run --package-path . DailyBriefMonitor --probe-ubiquity` (one-shot probe added to main.swift, or run `tail -f ~/Library/Logs/DailyBrief/monitor-stderr.log` for new "ubiquity container = /Users/.../Mobile Documents/iCloud~com~jamesonmorrill~dailybriefmonitor/Documents") | `ubiquity container URL: /Users/jamesonmorrill/Library/Mobile Documents/iCloud~com~jamesonmorrill~dailybriefmonitor/Documents` | `nil` → entitlement claimed but not authorized by macOS (rare — profile got stripped) |
| UBIQ-03 | Drop fresh HEIC into `~/Library/Mobile Documents/com~apple~CloudDocs/Notebook`. Watch `monitor-stderr.log` and run `xattr -l <file>` periodically. | Within ~5s: log shows `FolderWatcherService: triggered iCloud download for IMG_XXXX.HEIC`; within ~10s: `mdls -name kMDItemFSSize` returns full size; `xattr` no longer contains `com.apple.ubiquity.placeholder.metadata` | Timeout >30s → ubiquity API still no-op; brctl line should appear as belt-and-suspenders fallback |
| UBIQ-06 (optional) | After UBIQ-01..05 green: remove brctl block at `FolderWatcherService.swift:361-376`. Repeat UBIQ-03 probe. | No `brctl` log lines; HEIC still materializes in <10s | Materialization fails → restore brctl block (do not merge optional UBIQ-06) |

The probe sequence is what gets committed to `.planning/phases/999.1-.../probes/verify-ubiquity.sh` during Wave 0 of execution.
