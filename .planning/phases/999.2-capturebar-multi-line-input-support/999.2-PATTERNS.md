#### Pattern 5: Auto-grow recipe (RESEARCH §"Pattern 2") — NEW, no in-repo precedent

ThoughtRow does NOT auto-grow; this is genuinely new logic. The canonical recipe from RESEARCH is the source-of-truth (verified against React 19 + DOM spec):

```typescript
// SOURCE — RESEARCH.md §"Pattern 2: scrollHeight auto-grow recipe"
useLayoutEffect(() => {
  const el = textareaRef.current
  if (!el) return
  el.style.height = '0px'
  el.style.height = `${el.scrollHeight}px`
}, [input])
```

Place inside CaptureBar component body. Dependency array on `[input]` covers both D-03 (grow on type/paste) and D-04 (reset on post-submit `setInput('')`).

**Pitfall reminder (RESEARCH Pitfall 4):** The empty-textarea case is handled correctly by this recipe because an empty textarea's `scrollHeight` is one line of text (padding + line-height). No special `if (input === '')` branch.

#### Pattern 6: Existing submit/triage block (CaptureBar.tsx:14-45) — PRESERVE VERBATIM

Per D-09: the fire-and-forget triage logic, error handling, `setIsSubmitting`, and `createThought` call all stay as-is. Only the input element and the keydown handler change.

```typescript
// PRESERVE — vigil-pwa/src/components/CaptureBar.tsx:14-45
async function handleSubmit() {
  const trimmed = input.trim()                 // D-07: empty/whitespace-only rejected
  if (!trimmed || isSubmitting) return         // D-08: trim() preserves interior \n

  setIsSubmitting(true)
  setError(null)

  try {
    const thought = await createThought(trimmed)
    onCapture(thought)
    setInput('')                                // triggers useLayoutEffect → height reset (D-04)
    setIsSubmitting(false)

    // Fire-and-forget triage: never block capture on triage result
    triageThought(trimmed)
      .then((result) =>
        updateThought(thought.id, {
          category: result.category,
          ...(result.tags ? { tags: result.tags } : {}),
          ...(result.therapyClassification ? { therapyClassification: result.therapyClassification } : {}),
        }).then(() =>
          onCategoryUpdate(thought.id, result.category),
        ),
      )
      .catch(() => {
        // Silently swallow 503/502 — thought stays uncategorized
      })
  } catch (err) {
    setError(err instanceof Error ? err.message : 'Failed to save thought')
    setIsSubmitting(false)
  }
}
```

#### Pattern 5: Required test coverage (per CONTEXT.md Discretion + RESEARCH §"Validation Architecture")

The new test file MUST cover at minimum these seven behaviors (RESEARCH §"Code Examples" example 2 is the literal scaffold the planner should adopt):

1. **CAP-MULTI-02:** `user.paste('line one\nline two\nline three')` → assert textarea value preserves `\n`.
2. **CAP-MULTI-01 / D-01:** `user.paste('a\nb')` then `user.keyboard('{Meta>}{Enter}{/Meta}')` → assert `createThought` called with `'a\nb'`.
3. **CAP-MULTI-01 / D-01 (trap-test for Pitfall 3):** `user.type('first{Enter}second')` → assert textarea value is `'first\nsecond'` AND `createThought` was NOT called. **Never delete this test.**
4. **CAP-MULTI-04 / D-07:** type whitespace + plain Enters, then Cmd+Enter → assert `createThought` NOT called.
5. **D-04:** type then Cmd+Enter → `vi.waitFor` that textarea value clears.
6. **CAP-MULTI-05 / D-05:** assert textarea has `enterKeyHint="enter"` attribute.
7. **D-02:** Save button is disabled when empty and when whitespace-only, enabled with real content.

**Note (Pitfall 1):** Do NOT assert on `textarea.style.height` — jsdom returns `scrollHeight = 0`. Height is a visual-UAT concern, not a unit test.
