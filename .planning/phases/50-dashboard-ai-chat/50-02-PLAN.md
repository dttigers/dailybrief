  Add to the existing APIAIServices.swift file (which already contains other API service actors):

  **1. ChatProviding protocol** (public, Sendable):
  ```swift
  public protocol ChatProviding: Sendable {
      func chat(messages: [ChatMessage], includeContext: Bool) async throws -> ChatResponse
  }
  ```

  **2. ChatMessage model** (public struct, Codable, Sendable):
  - `role`: String ("user" or "assistant")
  - `content`: String

  **3. ChatResponse model** (public struct, Decodable, Sendable):
  - `response`: String
  - `contextUsed`: Int

  **4. ChatError enum** (public, Error, LocalizedError):
  - `apiError(String)`
  - `emptyResponse`

  **5. APIChatService actor** (public, conforms to ChatProviding):
  - Takes `VigilAPIClient` in init (same pattern as other API services)
  - `chat()` implementation: POST to `/chat` with body `{ messages: [...], includeContext: bool, contextLimit: 20 }`
  - Decode response as ChatResponse
  - Throw ChatError.emptyResponse if response string is empty after trimming
  - Wrap VigilAPIError as ChatError.apiError

  Follow the exact same actor + DTO pattern used by APITriageService, APIInsightService, etc. in this file.
  </action>

  <done>ChatProviding protocol and APIChatService actor compile and follow established service patterns</done>

  <action>
  **ChatViewModel.swift** — New file. `@MainActor @Observable final class`:

  Properties:
  - `messages: [DisplayMessage]` — array of (id: UUID, role: String, content: String, timestamp: Date)
  - `inputText: String` — bound to text field
  - `isLoading: Bool` — shows typing indicator while waiting for response
  - `includeContext: Bool = true` — toggle for thought context injection
  - `error: String?` — transient error display
  - Private: `chatService: (any ChatProviding)?`

  Methods:
  - `init(chatService:)` — takes optional ChatProviding
  - `sendMessage()` — guards non-empty input, appends user message to messages, clears input, sets isLoading=true, builds ChatMessage array from messages (user/assistant only), calls chatService.chat(), appends assistant response, sets isLoading=false. On error: set error string, remove the user message that failed, restore inputText.
  - `clearConversation()` — resets messages array
  - `canSend: Bool` — computed: !inputText.trimmed.isEmpty && !isLoading && chatService != nil

  **ChatView.swift** — New file. SwiftUI view:

  Layout (VStack filling detail area):
  1. **Header bar**: "Chat with Vigil" title + context toggle (Toggle bound to viewModel.includeContext) + clear button (trash icon, calls clearConversation)
  2. **Messages area** (ScrollViewReader + ScrollView):
     - ForEach over viewModel.messages
     - User messages: right-aligned, blue/accent bubble, white text
     - Assistant messages: left-aligned, gray/secondary bubble, primary text — render content as Markdown (use SwiftUI Text with AttributedString or just plain Text for MVP)
     - If isLoading: show typing indicator (3 animated dots or ProgressView with "Thinking...")
     - Auto-scroll to bottom when new messages arrive (use .onChange of messages.count to scroll to last message id)
  3. **Input area** (HStack at bottom):
     - TextField("Ask anything...", text: $viewModel.inputText, axis: .vertical) with lineLimit 1...5
     - Send button (arrow.up.circle.fill icon), disabled when !canSend
     - Submit on Enter key (onSubmit modifier), Shift+Enter for newline
  4. **Error banner**: If viewModel.error is non-nil, show dismissable error at top of messages area
  5. **Empty state**: When messages is empty, show centered prompt suggestions:
     - "What should I focus on today?"
     - "Break down my open tasks"
     - "What patterns do you see in my thoughts?"
     - Tapping a suggestion fills inputText and sends

  Style notes: Match the existing dashboard aesthetic — use system colors, .controlBackgroundColor for bubbles, standard padding. Keep it clean and native-feeling.
