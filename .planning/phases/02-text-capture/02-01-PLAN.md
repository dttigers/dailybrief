Create new directory `Sources/JarvisCore/Services/` and add `CaptureService.swift`.

CaptureService is a public actor that wraps ThoughtStore for capture-specific operations:

```
public actor CaptureService {
    private let store: ThoughtStore

    public init(store: ThoughtStore)

    /// Captures a text thought. Returns the saved Thought with assigned ID.
    /// Throws if content is empty after trimming.
    public func captureText(_ content: String) async throws -> Thought
}
```

Implementation of `captureText`:
1. Trim whitespace/newlines from content
2. Guard non-empty — throw a descriptive error if empty
3. Create `Thought(content: trimmed, source: .text)` — category and confidence stay nil (untriaged)
4. Call `store.save(&thought)` — this sets modifiedAt and persists (satisfies CAPT-05 timestamps)
5. Return the saved thought (now has id assigned via didInsert)

Create a `CaptureError` enum (public, LocalizedError) with case `.emptyContent`.

Keep it minimal — no extra validation, no categories. Zero required fields beyond the text itself.

**CapturePanel.swift** — NSPanel subclass for the floating capture window:

```
final class CapturePanel: NSPanel
```

Configuration in init:
- Style mask: `.titled`, `.closable`, `.fullSizeContentView`, `.nonactivatingPanel`
- `isFloatingPanel = true` (stays above other windows)
- `level = .floating`
- `titleVisibility = .hidden`
- `titlebarAppearsTransparent = true`
- `isMovableByWindowBackground = true`
- `backgroundColor = .windowBackgroundColor`
- `isReleasedWhenClosed = false` (reuse panel instance)
- Frame size: 480 width x 160 height
- `hidesOnDeactivate = false` (don't hide when switching apps during capture)
- `becomesKeyOnlyIfNeeded = false` (allow keyboard input)

Add methods:
- `toggle()` — if panel is visible and key, call `close()`. Otherwise call `center()` then `makeKeyAndOrderFront(nil)` and `NSApp.activate(ignoringOtherApps: true)`.
- `showPanel()` — center, makeKeyAndOrderFront, activate app
- `hidePanel()` — close

**CaptureView.swift** — SwiftUI view hosted in the panel:

```
struct CaptureView: View
```

Properties:
- `@State private var text = ""`
- `@State private var isSaving = false`
- `@State private var showSuccess = false`
- `var onCapture: (String) async throws -> Void` (closure — decoupled from service)
- `var onDismiss: () -> Void`

Layout:
- VStack with 12pt padding
- `TextEditor(text: $text)` — the primary input, about 100pt height, styled with `.font(.body)` and a subtle border. Use `.scrollContentBackground(.hidden)` and a custom background for clean look.
- HStack at bottom:
  - Left: Text showing character count (subtle, `.foregroundStyle(.secondary)`)
  - Right: "Capture" button (`.keyboardShortcut(.return, modifiers: .command)`) — disabled when text is empty or isSaving
- When showSuccess is true, overlay a brief "Captured!" label that auto-dismisses

Behavior:
- Cmd+Enter triggers capture (via keyboard shortcut on button)
- Escape triggers onDismiss
- On capture: set isSaving=true, call onCapture(text), on success set showSuccess=true, after 0.4s delay clear text and call onDismiss. On error, show inline error text.
- Focus TextEditor on appear (use `@FocusState`)

Do NOT use `.onSubmit` on TextEditor (it doesn't fire for multiline). The Cmd+Enter keyboard shortcut on the button is the correct pattern.

Do NOT make CaptureView depend on CaptureService — it takes a closure. This keeps the view testable and decoupled.
