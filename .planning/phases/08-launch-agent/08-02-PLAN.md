<task type="auto">
  <name>Task 1: Create BriefScheduler with daily timer</name>
  <files>Sources/DailyBriefMonitor/BriefScheduler.swift</files>
  <action>
Create a new `BriefScheduler` class (marked @Observable) that:

1. Takes a reference to StatusChecker (to call `runNow()`) and a scheduled hour/minute (default 6:00).
2. On init, calculates the time interval until the next scheduled run.
3. Uses a Foundation `Timer` (scheduled on RunLoop.main) that fires at the scheduled time.
4. When the timer fires: calls `checker.runNow()`, then reschedules for the next day.
5. Exposes `nextRunTime: Date?` (computed from the timer) for display in the menu bar.
6. Exposes `isScheduleEnabled: Bool` (default true) to allow toggling the schedule on/off.
7. Has a `reschedule(hour: Int, minute: Int)` method to change the schedule time.

Track whether today's brief has already been generated (check StatusChecker's lastRunTime against today's date) to avoid duplicate runs if the monitor restarts after the scheduled time.

Use `Calendar.current.nextDate(after: Date(), matching: DateComponents(hour: hour, minute: minute), matchingPolicy: .nextTime)` to compute next fire time.

Keep it simple — no persistence of schedule state (timer recomputes on launch).
  </action>

  <done>BriefScheduler class exists with daily timer, next-run computation, and toggle</done>
