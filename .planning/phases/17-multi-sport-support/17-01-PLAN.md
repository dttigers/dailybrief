<task type="auto">
  <name>Task 2: Refactor AppConfig for multi-sport support</name>
  <files>Sources/JarvisCore/Config/AppConfig.swift</files>
  <action>
Refactor `AppConfig.SportsConfig` to support multiple sports. Replace the single MLB-only config with a multi-sport structure:

```swift
public struct SportsConfig: Codable, Sendable {
    // Keep existing MLB fields for backward compatibility during decode
    public var mlb: SportLeagueConfig
    public var nfl: SportLeagueConfig
    public var nba: SportLeagueConfig
    public var nhl: SportLeagueConfig

    // Generic per-league config
    public struct SportLeagueConfig: Codable, Sendable {
        public var enabled: Bool
        public var teamId: Int
        public var divisionId: Int
        public var conferenceId: Int  // leagueId for MLB
        public var teamName: String
        public var divisionName: String
    }
}
```

**Backward compatibility is CRITICAL.** The custom `init(from decoder:)` must handle the old flat config format (teamId, divisionId, leagueId at the top level) by mapping it to `mlb` config with enabled=true and all other sports disabled. This way existing config.json files keep working.

Default values:
- MLB: enabled=true, teamId=116, divisionId=202, conferenceId=103, "Detroit Tigers", "AL Central"
- NFL: enabled=false, teamId=8, divisionId=0, conferenceId=9, "Detroit Lions", "NFC North"
- NBA: enabled=false, teamId=8, divisionId=0, conferenceId=1, "Detroit Pistons", "Central"
- NHL: enabled=false, teamId=6, divisionId=0, conferenceId=1, "Detroit Red Wings", "Atlantic"

Also add a computed property `enabledSports: [String]` that returns the list of enabled sport keys (e.g., ["mlb", "nfl"]).

Update any references to `config.sports.teamId` etc. in the codebase to use `config.sports.mlb.teamId` — check SportsService init and DailyBrief.swift. The SportsService should still receive just the MLB config (SportLeagueConfig) for now — Plan 02 will add the ESPN service for other sports.
