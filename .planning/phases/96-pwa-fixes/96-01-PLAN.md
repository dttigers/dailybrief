2. Examine the response body — the server returns a specific error string ("Invalid JSON body", "messages array is required", or "Last message must have role 'user'"). This tells you which validation path is failing.

3. Common root causes to investigate in order:
   a. **Hono double body consumption**: If another middleware (e.g., logging, rate limiter) calls `c.req.json()` or reads the body before the chat handler, `c.req.json()` in chat.ts will fail with a parse error. Check if any middleware in index.ts reads request bodies.
   b. **Request body not reaching handler**: The `timeout()` middleware or `cors()` middleware could interfere. Test with a minimal request.
   c. **Message validation failing**: If the client sends messages in an unexpected shape (e.g., `messages` key not at top level, or extra wrapping).
   d. **Anthropic SDK rejection**: If `callClaudeConversation` throws before the 502 catch, but after validation passes — though this should return 502, not 400.

4. Apply the fix to the identified root cause. If the fix is in:
   - `vigil-core/src/routes/chat.ts` — fix the handler
   - `vigil-pwa/src/api/client.ts` — fix the request payload construction
   - `vigil-core/src/index.ts` — fix middleware ordering

5. After fixing, verify with the same curl command that you get a 200 response with `{"response":"...","contextUsed":N}`.
