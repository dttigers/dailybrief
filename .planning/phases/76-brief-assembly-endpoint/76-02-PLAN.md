## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> POST /brief/generate | Authenticated via bearer token; no request body to validate |
| client -> GET /brief/:date | Authenticated; date param is untrusted user input |
| filesystem -> GET /brief/:date | Reading file by path stored in DB; path derived from validated date |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-76-04 | Tampering | GET /brief/:date | mitigate | Validate `:date` param with `/^\d{4}-\d{2}-\d{2}$/` regex before any DB query or file read — prevents path traversal |
| T-76-05 | Information Disclosure | POST /brief/generate error | mitigate | Catch errors and return generic "Brief generation failed" — do not expose stack traces or internal paths to client |
| T-76-06 | Denial of Service | POST /brief/generate | accept | 30-second global timeout protects against runaway generation; single-user system makes DoS low risk |
| T-76-07 | Elevation of Privilege | Route registration | mitigate | Both routes are behind existing bearerAuth middleware — no new auth bypass paths |
</threat_model>
