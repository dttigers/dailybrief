<objective>
Build the in-process generate scheduler that fires daily brief generation at the user-configured time in the user's timezone, dedupes across restarts/DST, performs a 7-day retention sweep inline after each successful generate, and cleans up on SIGTERM.

Purpose: This is the heart of Phase 86 — it decouples "when a brief gets generated" from "when the Mac prints it". The Mac CLI no longer has to be awake or correct about generation timing; the server is the single source of truth (D-13).

Output: `createGenerateScheduler({ db, assemblerFactory, now?, logFn? })` with `start()` / `stop()` methods. Wired into `vigil-core/src/index.ts` after route registration with SIGTERM teardown.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Scheduler → DB | Scheduler reads settings and writes briefs rows as a privileged in-process caller |
| Scheduler → FS | Retention sweep calls fs.unlink on pdfFilename paths stored in briefs table |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-86-06 | Tampering | briefs.pdfFilename used by retention sweep | mitigate | Only unlink paths already stored by the server itself (brief-generate.ts wrote them); wrap each unlink in try/catch; best-effort per D-21. The path is already server-trusted (not user-supplied). |
| T-86-07 | DoS | Scheduler throwing crashes Node process | mitigate | tick() wraps all logic in try/catch; `.catch` on `setInterval` callback; scheduler failures never propagate to server runtime |
| T-86-08 | Repudiation | Silent failures masking cron problems | mitigate | Explicit `logFn` channel: info on success, warn on dedupe-skip, error on throw. Always prints to stdout/stderr (visible in Railway logs) |
| T-86-09 | DoS | Double-fire in multi-instance deploy | accept | Current Railway plan = 1 instance (confirmed). Dedupe window (10min) blunts damage even if it happens. Comment in index.ts warns future operators. |
| T-86-10 | EoP | Scheduler could in theory hit server-internal endpoints | mitigate | Scheduler invokes `brief-assembly-service` directly as a function call — does NOT self-HTTP. No auth bypass surface. |
</threat_model>
