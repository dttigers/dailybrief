<decision_record>
## Decisions resolved by this plan

### DR-01: Inline Swift `cp` vs `install.sh --no-launchctl` flag
**Decision:** Inline the binary copy in Swift using `FileManager`. Do NOT add a `--no-launchctl` flag to `install.sh`.

**Rationale:**
- Removes the subprocess coupling entirely — UpdateService is no longer dependent on install.sh's internals at all
- Two `copyItem` calls in Swift are smaller than the round-trip surface of a flag + a second code path inside install.sh
- VERIFICATION.md hints at this ("simpler, removes coupling")
- A flag still leaves install.sh as a foot-gun for any future caller that forgets the flag

### DR-02: Plist regeneration during update
**Decision:** Do NOT touch the plist during a normal update. The plist is generated by `install.sh` only on first install / terminal bootstrap.

**Rationale:** Reading `install.sh` lines 56-89, the plist is fully self-contained (it references `$INSTALL_DIR/DailyBriefMonitor`, which is a stable path). It does not embed any per-build metadata. The trampoline's `launchctl kickstart -k` re-execs the binary at the same plist path, which is exactly what we need. Plist regeneration is a first-install concern, not an update concern.

### DR-03: Regression assertion for `runs = 1, last exit code = 0`
**Decision:** Include as a required item in the human-verify checkpoint (Task 3). Do NOT add an automated test in this phase — the checkpoint exists precisely for this kind of launchd-state assertion that is hard to script reliably.

### DR-04: Warning location in install.sh
**Decision:** Add the warning as a block comment immediately above the launchctl bootout/bootstrap block (between current lines 91 and 92). This is where any future maintainer searching for the bootout call will land.
</decision_record>

```swift
    /// Inline replacement for `Scripts/install.sh`'s cp steps. Safe to call from
    /// inside the managed launchd process because it does NOT touch launchctl.
    /// The trampoline (spawnDetachedReloadHelper) handles respawn via `launchctl kickstart -k`.
    private func installBuiltBinaries() -> (success: Bool, output: String) {
        let releaseDir = RepoLocation.releaseBuildDir
        let installDir = NSString("~/.local/bin").expandingTildeInPath
        var log = ""

        // Ensure ~/.local/bin exists (parity with install.sh step 2: `mkdir -p`)
        do {
            try FileManager.default.createDirectory(
                atPath: installDir,
                withIntermediateDirectories: true,
                attributes: nil
            )
        } catch {
            return (false, "Failed to create \(installDir): \(error.localizedDescription)")
        }

        for binary in ["DailyBrief", "DailyBriefMonitor"] {
            let src = (releaseDir as NSString).appendingPathComponent(binary)
            let dst = (installDir as NSString).appendingPathComponent(binary)

            guard FileManager.default.fileExists(atPath: src) else {
                return (false, log + "\nMissing source binary: \(src)")
            }

            // FileManager.copyItem fails if dst exists — remove first (parity with `cp -f`)
            if FileManager.default.fileExists(atPath: dst) {
                do {
                    try FileManager.default.removeItem(atPath: dst)
                } catch {
                    return (false, log + "\nFailed to remove existing \(dst): \(error.localizedDescription)")
                }
            }

            do {
                try FileManager.default.copyItem(atPath: src, toPath: dst)
                log += "Installed \(binary): \(src) → \(dst)\n"
            } catch {
                return (false, log + "\nFailed to copy \(binary): \(error.localizedDescription)")
            }
        }

        return (true, log)
    }
```
