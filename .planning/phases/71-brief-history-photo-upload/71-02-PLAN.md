<interfaces>
<!-- Backend process-photo API (from vigil-core/src/routes/process-photo.ts) -->

POST /v1/process-photo?preview=true — Preview mode (no DB save)
  Body: { image: string (base64), mediaType: "image/jpeg"|"image/png"|"image/gif"|"image/webp", forcePaperType?: "lined"|"gridded" }
  Response 200: { paperType: string, confidence: number, thoughts: Array<{ id: null, content: string, source: "image", confidence: number, projectId: null }> }

POST /v1/process-photo — Commit mode (saves to DB)
  Body: same as preview
  Response 201: { paperType: string, confidence: number, thoughts: ThoughtApiResponse[] }

Validation:
  - image required, base64 string, max ~5MB
  - mediaType required, must be one of image/jpeg, image/png, image/gif, image/webp
  - forcePaperType optional, must be "lined" or "gridded" if provided
  - Returns 413 if image too large
  - Returns 502 if AI processing fails
  - Returns 503 if AI service unavailable

<!-- Existing PWA patterns -->
API client: vigilFetch(path, init?) — adds Bearer auth + Content-Type
File input: <input type="file" accept="image/*"> with onChange handler
Base64 conversion: FileReader.readAsDataURL, strip data URL prefix
</interfaces>

1. Create vigil-pwa/src/hooks/usePhotoUpload.ts:

   - State machine with phases: "idle" | "selecting" | "previewing" | "committing" | "done" | "error".
   - `selectFile(file: File)`: validates file is image/* and under 5MB, reads as base64 via FileReader, strips the `data:image/...;base64,` prefix, derives mediaType from file.type (map to the 4 valid types, reject others). Transitions to "previewing".
   - `preview()`: calls `vigilFetch('/v1/process-photo?preview=true', { method: 'POST', body: JSON.stringify({ image, mediaType }) })`. Stores response in `previewResult` state: `{ paperType, confidence, thoughts }`. Transitions from "previewing" to show results. On error transitions to "error" with message.
   - `commit(forcePaperType?: "lined" | "gridded")`: calls `vigilFetch('/v1/process-photo', { method: 'POST', body: JSON.stringify({ image, mediaType, forcePaperType }) })`. On success transitions to "done". On error transitions to "error".
   - `reset()`: clears all state back to "idle".
   - Returns: `{ phase, file, imagePreviewUrl, previewResult, commitResult, error, selectFile, commit, reset }`.
   - `imagePreviewUrl`: store the full data URL from FileReader before stripping prefix, so the page can show a thumbnail via `<img src={imagePreviewUrl}>`.

2. Create vigil-pwa/src/pages/PhotoUploadPage.tsx:

   - Uses `usePhotoUpload()` hook.
   - **Idle state**: Big drop zone / file picker button. Text: "Upload a photo of handwritten notes". `<input type="file" accept="image/jpeg,image/png,image/gif,image/webp" capture="environment">` (capture="environment" prompts camera on mobile).

   - **Preview result state**: Show the photo thumbnail (imagePreviewUrl), paper type badge (e.g., "Lined" / "Gridded" with confidence percentage), and list of detected thoughts as editable-looking cards. Two buttons: "Save Thoughts" (calls commit) and "Cancel" (calls reset). If paperType is not what user expects, show a small "Override: Lined / Gridded" toggle that sets forcePaperType on commit.
   - **Committing state**: Loading spinner with "Saving thoughts..."
   - **Done state**: Success message showing how many thoughts were saved. "Upload Another" button (calls reset).
   - **Error state**: Error message with "Try Again" button (calls reset).
   - Style with Tailwind dark theme. Drop zone: dashed border-slate-600, hover:border-indigo-500. Photo thumbnail: max-h-48 rounded. Thought cards: bg-slate-800 p-3 rounded.
