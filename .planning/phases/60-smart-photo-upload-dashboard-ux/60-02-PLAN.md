<state_machine>
DashboardViewModel photo-upload state machine (this is the D-10 test target):

```swift
enum PhotoPreviewState: Identifiable {
    case analyzing(fileURL: URL, indexInBatch: Int, totalInBatch: Int)
    case awaitingUserDecision(PhotoPreviewPayload)
    case committing(PhotoPreviewPayload, forced: PaperType?)
    // Terminal states fall out of the enum — either commit success appends thoughts
    // and advances to next file, or cancel/error advances without committing.

    var id: String { ... } // some stable id for .sheet(item:) presentation
}

struct PhotoPreviewPayload {
    let fileURL: URL
    let indexInBatch: Int
    let totalInBatch: Int
    let detectedPaperType: PaperType      // what the backend originally returned
    let confidence: Double
    let thoughts: [String]                // content strings, read-only
    let currentForcePaperType: PaperType  // what the segmented picker is showing
    let showUncertaintyBanner: Bool       // true iff original confidence < 0.5
    let userDefaultPaperType: PaperType   // from SettingsViewModel, for the banner text
}

enum PaperType: String, Codable { case lined, gridded }
```

Transitions (must be covered by unit tests in Task 6):

1. **T1 Analyze → Preview shown** — processFiles starts photo N, calls processPhoto(preview:true), receives {paperType:"lined", confidence:0.9, thoughts:[a,b,c]}, transitions to `awaitingUserDecision` with `currentForcePaperType = .lined`, `showUncertaintyBanner = false`
2. **T2 Preview → Commit (no override)** — user clicks Commit with picker still on detected type → calls processPhoto(preview:false) WITHOUT forcePaperType → receives committed ThoughtApiResponse array → appends to dashboard list → advances to next file
3. **T3 Preview → Override → Re-preview** — user flips picker from lined to gridded → view-model issues processPhoto(preview:true, forcePaperType: .gridded) → receives 1-thought response → updates payload.thoughts and payload.currentForcePaperType, state remains awaitingUserDecision
4. **T4 Preview → Cancel** — user clicks Cancel → state cleared → advances to next file → no processPhoto(preview:false) call
5. **T5 Analyze fails** — processPhoto(preview:true) throws (network, 413, 502, 503, 500) → push filename + D-08-mapped reason onto `photoBatchErrors` (existing error-banner field) → advance to next file → no preview shown
6. **T6 Low confidence (< 0.5)** — processPhoto(preview:true) returns confidence:0.3 → view-model IMMEDIATELY issues a second preview call with forcePaperType = settings.photoUploadDefaultPaperType → receives re-shaped thoughts → transitions to awaitingUserDecision with `currentForcePaperType = userDefault`, `showUncertaintyBanner = true`, `detectedPaperType = what Claude said`, `userDefaultPaperType = settings default`
7. **T7 Multi-photo sequential** — photo 1 commits → state reset → photo 2 analyze starts → photo 2 preview shown (fresh, no sticky override from photo 1's choice)
8. **T8 Commit API error** — user clicks Commit but processPhoto(preview:false) throws → push error onto photoBatchErrors, advance to next file (do NOT retry, do NOT leave modal stuck)

Override while uncertainty banner is showing (T6 → T3): user can further override off the user-default choice → standard T3 re-preview path applies. The banner text remains the same (`using your default: {userDefaultPaperType}`) because that IS why the current state exists — the user has simply then overridden further.
</state_machine>

<api_client_contract>

Mirror the existing `describeSubjects` shape — JSON POST with the bearer token auto-applied by `VigilAPIClient.patch` / `post` (confirm the method name used by describeSubjects and use the same).

```swift
struct ProcessedPhotoResponse: Codable {
    let paperType: String       // "lined" | "gridded" | "unknown"
    let confidence: Double
    let thoughts: [PreviewThought]
}

struct PreviewThought: Codable {
    let id: Int?                // null in preview mode, populated in commit mode
    let content: String
    let source: String          // always "image"
    let confidence: Double?
    let projectId: Int?
}

extension APIImageDescriptionService {
    /// Calls POST /v1/process-photo with optional preview + forcePaperType.
    /// - preview: true → ?preview=true → no DB insert, PreviewThought.id will be nil
    /// - forcePaperType: optional "lined" | "gridded" body field (nil = omit)
    /// Throws on non-2xx; caller maps HTTP status to user-facing strings per D-08.
    func processPhoto(
        imageData: Data,
        preview: Bool,
        forcePaperType: PaperType?
    ) async throws -> ProcessedPhotoResponse
}
```

- Base64-encode `imageData` client-side (use `imageData.base64EncodedString()`).
- mediaType: derive from file extension — `.jpg`/`.jpeg` → `image/jpeg`, `.png` → `image/png`, `.gif` → `image/gif`, `.webp` → `image/webp`. If unknown, throw a typed error the caller maps to "Image format not supported" (400).
- Query param: if `preview == true`, append `?preview=true` to the path; otherwise no query string at all (NOT `?preview=false` — be byte-identical to Phase 59 commit behavior).
- Body: always include `image` and `mediaType`; include `forcePaperType` only if non-nil. Use `JSONEncoder` with conditional encoding.
- Decoder: on non-2xx status, throw a typed error carrying the HTTP status code so the view-model can map it per D-08. Do NOT include the server error body in user-facing surfaces — log it for debug only.
- Keep `describeSubjects` unchanged.
</api_client_contract>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File picker → DashboardViewModel.processFiles | The macOS file picker hands the dashboard URLs to local files. URLs come from the user directly, but file contents are untrusted input from the filesystem (could be anything with an image extension). |
| DashboardViewModel → `/v1/process-photo` | Base64-encoded image bytes + the user's forcePaperType selection + a bearer token cross here. Auth is bearer (Phase 38); the user's selection is trusted enum validated client-side before send. |

| UserDefaults ↔ SettingsViewModel.photoUploadDefaultPaperType | Local, single-user. Not a security boundary — preference only, no PII, no secret. |
