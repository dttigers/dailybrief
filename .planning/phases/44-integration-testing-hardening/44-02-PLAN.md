<task type="auto">
  <name>Task 1: Create comprehensive API smoke test script</name>
  <files>vigil-core/scripts/smoke-test.ts</files>
  <action>
Create `vigil-core/scripts/smoke-test.ts` that tests all key API endpoints against production:

1. Read API_URL and API_KEY from env vars (default URL: https://vigil-core-production.up.railway.app)
2. Test each endpoint category, printing pass/fail for each:

   **Health (no auth):**
   - GET /v1/health → 200, body has status "ok" and database "connected"

   **Auth enforcement:**
   - GET /v1/thoughts (no auth header) → 401
   - GET /v1/thoughts (invalid key) → 401

   **Thoughts CRUD:**
   - POST /v1/thoughts with `{ "rawContent": "Smoke test thought" }` → 201, returns thought with id
   - GET /v1/thoughts → 200, array includes the created thought
   - GET /v1/thoughts/:id → 200, matches created thought
   - DELETE /v1/thoughts/:id → 200 or 204

   **Summary & Brief:**
   - GET /v1/summary → 200, has `total` and `byCategory`
   - GET /v1/brief → 200, has `date` and `counts`

   **Tags:**
   - GET /v1/tags → 200, returns array

   **Triage (if thought exists):**
   - POST /v1/triage with valid thought ID → 200 (or skip if no thoughts)

3. Print summary: X/Y tests passed
4. Exit with code 0 if all pass, 1 if any fail

Pattern: Follow existing script style (tsx runner, like generate-key.ts and migrate-sqlite.ts).
Add `"smoke-test"` script to package.json: `"smoke-test": "tsx scripts/smoke-test.ts"`
  </action>
  <verify>
    cd vigil-core && API_KEY=<key> npx tsx scripts/smoke-test.ts — all tests pass
  </verify>
  <done>Smoke test script runs against production, all endpoints respond correctly, clean pass/fail output.</done>
