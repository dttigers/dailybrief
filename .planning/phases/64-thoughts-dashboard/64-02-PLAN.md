## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-64-04 | Tampering | CaptureBar input | mitigate | Trim whitespace and reject empty strings before calling createThought. Server also validates non-empty content. |
| T-64-05 | Tampering | ThoughtRow inline edit | mitigate | Same trim+empty check before calling updateThought. Only include `content` field in PUT body (not category or other fields). |
| T-64-06 | Information Disclosure | Triage error handling | mitigate | Catch all triage errors silently — do not expose server error details (503/502 response bodies) to the user. |
| T-64-07 | Spoofing | Category from triage | accept | Category string comes from our own server — not user-controllable. Rendered as text node (React auto-escapes). Low risk. |
</threat_model>
