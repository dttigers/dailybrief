  <name>Task 1: Add capture mode UI to CaptureView</name>
  <files>Sources/DailyBriefMonitor/CaptureView.swift</files>
  <action>
  Extend CaptureView with three capture modes: text (existing), voice, and image.

  1. Add CaptureMode enum at top of file:
     ```
     enum CaptureMode: String, CaseIterable {
         case text, voice, image
         var label: String { rawValue.capitalized }
         var icon: String {
             switch self {
             case .text: return "text.cursor"
             case .voice: return "mic.fill"
             case .image: return "photo"
             }
         }
     }
     ```

  2. Add new @State properties to CaptureView:
     - `@State private var captureMode: CaptureMode = .text`
     - `@State private var isRecording = false`
     - `@State private var recordingDuration: TimeInterval = 0`
     - `@State private var recordingTimer: Task<Void, Never>?`
     - `@State private var isTranscribing = false`
     - `@State private var selectedImageURL: URL?`
     - `@State private var isDescribing = false`

  3. Add new closure properties:
     - `var onVoiceCapture: ((URL) async throws -> Thought)?` — called with audio URL after recording stops. Caller handles transcription + save + triage.
     - `var onImageCapture: ((URL) async throws -> Thought)?` — called with image URL after selection. Caller handles description + save + triage.

  4. Add mode picker at top of the VStack (above the existing TextEditor):
     ```
     Picker("Mode", selection: $captureMode) {
         ForEach(CaptureMode.allCases, id: \.self) { mode in
             Label(mode.label, systemImage: mode.icon).tag(mode)
         }
     }
     .pickerStyle(.segmented)
     .labelsHidden()
     ```

  5. Replace the fixed TextEditor with a conditional view based on captureMode:

     **Text mode** (existing): Show the TextEditor + character count + Capture button exactly as they are now. No changes to text capture flow.

     **Voice mode**: Show centered content:
     - Large mic button (SF Symbol "mic.circle.fill" / "stop.circle.fill" when recording).
       Size: 60x60. Color: .red when recording, .accentColor when idle.
     - Below button: recording duration label formatted as "0:00" (updated every 0.1s via timer Task).
     - Below duration: status text — "Tap to record" / "Recording..." / "Transcribing..." / "Captured!".
     - When recording starts: set isRecording=true, start duration timer Task.
     - When recording stops: set isRecording=false, cancel timer, set isTranscribing=true, call onVoiceCapture closure. On success: show triage result (reuse existing overlay). On error: show errorMessage.
     - Cmd+Enter shortcut toggles recording (start if idle, stop if recording).

     **Image mode**: Show centered content:
     - "Choose Image" button with photo icon. On tap: calls onImageCapture flow (picker + describe).
     - If selectedImageURL is set: show filename label below button.
     - If isDescribing: show ProgressView + "Analyzing image..." text.
     - On success: show triage result overlay (reuse existing).
     - On error: show errorMessage.

  6. The success overlay, triage result overlay, and category picker overlay remain shared across all modes — they already work as ZStack overlays on top of the content.

  7. Ensure Escape key still dismisses in all modes. If recording when Escape pressed, stop recording first, then dismiss.

  Keep the CaptureView frame size the same. The mode picker adds ~30px of height which is fine for the panel.
  Do NOT change existing text capture behavior — text mode must work exactly as before.

     a. Calls transcriptionService.transcribe(audioURL:) to get text

     d. Returns the saved Thought
     e. On error: logs and rethrows

     **onImageCapture**: Closure that takes image URL and:
     a. Calls ImagePicker.pickImage() — wait, no. The URL is passed in from CaptureView which already called ImagePicker. Actually, reconsider: the closure should handle the full flow from URL.
     b. Calls imageDescriptionService.describe(imageURL:) to get text description
     c. Calls captureService.capture(description, source: .image) to save thought
     d. Runs triage on the description text
     e. Returns the saved Thought
     f. On error: logs and rethrows

     Actually, for voice: CaptureView manages the recording UI (start/stop via VoiceCaptureService closures). The onVoiceCapture closure receives the audio URL AFTER recording stops. So AppDelegate needs to also pass VoiceCaptureService to CaptureView for start/stop control.

     Revised approach — add two more closures to CaptureView:
     - `var onStartRecording: (() async throws -> Void)?` — calls voiceCaptureService.startRecording()
     - `var onStopRecording: (() async throws -> URL)?` — calls voiceCaptureService.stopRecording()

     Then onVoiceCapture becomes the post-recording handler that transcribes + saves + triages.

     Alternatively, simpler: pass a single `onVoiceCapture` closure that CaptureView calls when the user taps stop. The closure receives nothing — it calls stopRecording() internally, transcribes, saves, triages, returns Thought. CaptureView calls startRecording via a separate closure when record button tapped.

     Simplest approach:
     - `var onStartRecording: (() async throws -> Void)?`
     - `var onStopRecording: (() async throws -> Thought)?` — stops recording, transcribes, saves with .voice source, triages, returns Thought
     - `var onImageCapture: (() async throws -> Thought)?` — opens picker, describes, saves with .image source, triages, returns Thought

     This way CaptureView just calls closures and shows results. All orchestration in AppDelegate.

     Wire these in the CaptureView initializer alongside existing closures. Make them optional (nil when services unavailable — voice/image buttons disabled).

  5. For image flow, the onImageCapture closure should:
     a. Call `ImagePicker.pickImage()` — if nil (user cancelled), return early (throw or return nil... better to make it optional return)
     
     Actually, let's make onImageCapture non-optional-return: `(() async throws -> Thought?)?`
     - Calls ImagePicker.pickImage()
     - If user cancels, returns nil
     - Otherwise describes image, saves thought, triages, returns Thought

  6. In CaptureView, update voice mode:
     - Record button tap: `Task { try await onStartRecording?() }`, set isRecording = true
     - Stop button tap: set isRecording = false, cancel timer, call `let thought = try await onStopRecording?()` — then flow into triage display (reuse existing overlay logic)

  7. In CaptureView, update image mode:
     - "Choose Image" button tap: set isDescribing = true, call `let thought = try await onImageCapture?()` — if nil (cancelled), reset. Otherwise flow into triage display.

  8. Disable voice/image mode segments if respective closures are nil (services unavailable).
