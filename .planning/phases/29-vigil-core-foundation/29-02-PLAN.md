Database schema (from Swift GRDB migrations):

Table: thoughts
- id (INTEGER PRIMARY KEY AUTOINCREMENT)
- content (TEXT NOT NULL)
- category (TEXT) — task|therapy|idea|reflection|project
- confidence (REAL) — 0.0-1.0
- source (TEXT NOT NULL) — text|voice|image
- createdAt (DATETIME NOT NULL)
- modifiedAt (DATETIME NOT NULL)
- cloudKitRecordID (TEXT NOT NULL UNIQUE)
- syncStatus (TEXT NOT NULL) — pending|synced|pendingDeletion
- lastSyncedAt (DATETIME)
- taskStatus (TEXT) — open|inProgress|done
- therapyClassification (TEXT) — selfLearnable|bringToTherapist
- tags (TEXT) — JSON array stored as string
- isFavorited (INTEGER NOT NULL DEFAULT 0)

Table: thought_links
- id (INTEGER PRIMARY KEY AUTOINCREMENT)
- sourceThoughtId (INTEGER NOT NULL, FK → thoughts.id CASCADE)
- targetThoughtId (INTEGER NOT NULL, FK → thoughts.id CASCADE)
- createdAt (DATETIME NOT NULL)
- UNIQUE(sourceThoughtId, targetThoughtId)

Virtual table: thoughts_fts (FTS5 on content column)

<task type="auto">
  <name>Task 2: Create summary endpoint</name>
  <files>vigil-core/src/routes/summary.ts, vigil-core/src/index.ts</files>
  <action>
    Create `src/routes/summary.ts`:
    - Export a Hono instance with GET /summary route
    - Query the database for:
      1. Total thought count: `SELECT COUNT(*) FROM thoughts WHERE syncStatus != 'pendingDeletion'`
      2. Counts by category: `SELECT category, COUNT(*) as count FROM thoughts WHERE syncStatus != 'pendingDeletion' GROUP BY category`
      3. Task counts by status: `SELECT taskStatus, COUNT(*) as count FROM thoughts WHERE category = 'task' AND syncStatus != 'pendingDeletion' GROUP BY taskStatus`
      4. Recent thoughts (last 5): `SELECT id, content, category, source, createdAt FROM thoughts WHERE syncStatus != 'pendingDeletion' ORDER BY createdAt DESC LIMIT 5`
      5. Favorites count: `SELECT COUNT(*) FROM thoughts WHERE isFavorited = 1 AND syncStatus != 'pendingDeletion'`
      6. Linked thoughts count: `SELECT COUNT(DISTINCT sourceThoughtId) FROM thought_links`

    - Response shape:
      ```json
      {
        "total": 142,
        "byCategory": { "task": 45, "therapy": 30, "idea": 25, "reflection": 22, "project": 20 },
        "tasksByStatus": { "open": 12, "inProgress": 3, "done": 30 },
        "favorites": 8,
        "linkedThoughts": 5,
        "recent": [{ "id": 142, "content": "...", "category": "task", "source": "text", "createdAt": "..." }]
      }
      ```
    - Exclude thoughts with syncStatus = 'pendingDeletion' from all counts (soft-deleted)
    - Parse tags JSON in recent thoughts if present (tags column is JSON array stored as TEXT)
    - If DB not available, return 503 with `{ error: "Database not available" }`

    Mount in src/index.ts: `app.route('/v1', summary)` alongside health routes.
  </action>
  <verify>cd vigil-core && npx tsx src/index.ts & sleep 2 && curl -s http://localhost:3001/v1/summary | python3 -m json.tool && kill %1</verify>
  <done>GET /v1/summary returns JSON with total count, category breakdown, task status breakdown, favorites count, linked count, and 5 most recent thoughts from the live Jarvis database.</done>
</task>
