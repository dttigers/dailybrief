<task type="auto">
  <name>Task 1: Create Vigil Core project scaffold</name>
  <files>vigil-core/package.json, vigil-core/tsconfig.json, vigil-core/.gitignore, vigil-core/src/index.ts</files>
  <action>
    Create `vigil-core/` directory at the repo root (sibling to Sources/).

    1. Initialize `package.json` with:
       - name: "vigil-core"
       - type: "module" (ESM)
       - scripts: "dev": "tsx watch src/index.ts", "build": "tsc", "start": "node dist/index.js"
       - dependencies: hono, @hono/node-server, better-sqlite3
       - devDependencies: tsx, typescript, @types/better-sqlite3, @types/node

    2. Create `tsconfig.json`:
       - target: ES2022, module: NodeNext, moduleResolution: NodeNext
       - outDir: "./dist", rootDir: "./src"
       - strict: true, esModuleInterop: true, skipLibCheck: true

    3. Create `.gitignore`: node_modules/, dist/, *.js.map

    4. Create `src/index.ts`:
       - Import Hono and @hono/node-server serve
       - Create Hono app instance
       - Mount health routes at /v1 (imported from routes/health.ts — created in Task 2)
       - Start server on port 3001 (use PORT env var with 3001 default)
       - Log "Vigil Core API running on port {port}" at startup

    Use port 3001 to avoid conflicts with common dev servers on 3000. Do NOT use Express — Hono is lighter and better suited for this use case.

    Run `cd vigil-core && npm install` after creating files.
  </action>
  <verify>cd vigil-core && ls package.json tsconfig.json src/index.ts && node -e "require('./node_modules/hono/package.json')" 2>/dev/null || npx tsx --eval "import('hono')"</verify>
  <done>vigil-core/ exists with package.json, tsconfig.json, .gitignore, src/index.ts. All dependencies installed.</done>
</task>
