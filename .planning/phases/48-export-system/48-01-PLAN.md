<task type="auto">
  <name>Task 1: Create API export endpoint</name>
  <files>vigil-core/src/routes/export.ts, vigil-core/src/index.ts</files>
  <action>
Create `vigil-core/src/routes/export.ts` with a `GET /export` route that returns all matching thoughts in the requested format.

**Query parameters:**
- `format` (required): `json`, `csv`, or `markdown`
- `category`: filter by category (task, therapy, idea, reflection, project)
- `from`: ISO date string — only thoughts created on/after this date
- `to`: ISO date string — only thoughts created on/before this date
- `tag`: filter by tag
- `q`: full-text search query

**Implementation:**
- Reuse the same WHERE clause pattern from `thoughts.ts` (syncStatus != pendingDeletion, category filter, date range via gte/lte on createdAt, tag jsonb containment, full-text search)
- NO pagination — fetch all matching rows (export is for bulk download). Use a hard limit of 10000 rows to prevent runaway queries.
- Order by createdAt ascending (chronological for export readability)

**Format handlers:**
- **JSON**: Return `Content-Type: application/json` with array of thought objects (same shape as ThoughtApiResponse but as a flat array, not paginated). Set `Content-Disposition: attachment; filename="vigil-export-YYYY-MM-DD.json"`.
- **CSV**: Return `Content-Type: text/csv` with header row: `id,content,category,source,createdAt,modifiedAt,taskStatus,tags,isFavorited`. Escape content field properly (wrap in double quotes, escape internal quotes by doubling). Set `Content-Disposition: attachment; filename="vigil-export-YYYY-MM-DD.csv"`.
- **Markdown**: Return `Content-Type: text/markdown`. Format as sections grouped by category with headers. Each thought as a bullet with date prefix: `- **2026-04-05** — Content here`. Uncategorized thoughts in their own section. Set `Content-Disposition: attachment; filename="vigil-export-YYYY-MM-DD.md"`.

**Register the route:** Import and mount in `index.ts` as `app.route("/v1", exportRoute)` in the protected routes section, after the briefHistory route.

**Do NOT:**
- Add streaming — the 10k row limit keeps responses manageable
- Add new dependencies — use built-in string operations for CSV/Markdown formatting

**Step 1: Add raw data fetch to VigilAPIClient**

Add a `getRawData` method to `VigilAPIClient` that returns raw `Data` instead of decoding JSON. This is needed because CSV and Markdown responses aren't JSON.

```swift
public func getRawData(path: String, query: [String: String] = [:], accept: String = "application/json") async throws -> Data
```

Same pattern as `get<T>` but:
- Sets Accept header to the `accept` parameter value
- Returns raw Data after validation (no JSON decoding)

**Step 2: Add Export subcommand**

Add `Export` to the subcommands array in DailyBrief's CommandConfiguration.

Create `Export` struct following the same pattern as `History`:

```swift
struct Export: AsyncParsableCommand {
    static let configuration = CommandConfiguration(
        abstract: "Export thoughts as JSON, CSV, or Markdown."
    )

    @Option(help: "Output format: json, csv, or markdown (default: json)")
    var format: String = "json"

    @Option(help: "Filter by category (task, therapy, idea, reflection, project)")
    var category: String?

    @Option(help: "Start date filter (YYYY-MM-DD)")
    var from: String?

    @Option(help: "End date filter (YYYY-MM-DD)")
    var to: String?

    @Option(help: "Output file path (default: ~/Documents/DailyBrief/vigil-export-DATE.FORMAT)")
    var output: String?

    @Option(help: "Path to config file")
    var configPath: String?
}
```

**In run():**
1. Validate format is one of json, csv, markdown
2. Validate date formats if provided (YYYY-MM-DD regex)
3. Build query params dict from options
4. Determine Accept header: `application/json` for json, `text/csv` for csv, `text/markdown` for markdown
5. Call `apiClient.getRawData(path: "/export", query: params, accept: acceptHeader)`
6. Determine output path: use --output if provided, otherwise `~/Documents/DailyBrief/vigil-export-YYYY-MM-DD.{format extension}`
7. Write data to file
8. Print confirmation: "Exported to {path} ({byte count} bytes)"

**File extension mapping:** json → .json, csv → .csv, markdown → .md
