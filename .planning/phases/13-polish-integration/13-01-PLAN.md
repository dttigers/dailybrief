  - In `openDashboard()`, pass insightService to DashboardViewModel init (add parameter)

  - Add `private let insightService: InsightService?` property
  - Add `var insights: [Insight] = []` published state
  - Add `var isLoadingInsights = false` state

  - Add `loadInsights()` method:
    - Guard insightService exists, guard config.insights.enabled
    - Load config for lookbackDays
    - Fetch recent thoughts from store (use fetchAll with limit based on lookback)
    - Call `insightService.generateInsights(thoughts:lookbackDays:)`
    - Set `insights` from result
    - Wrap in do/catch, log errors, set insights to empty on failure
  - Call `await loadInsights()` at end of `refresh()` method

  **DashboardView.swift:**
  - Add an "Insights" section in the detail view, ABOVE the "Today's Schedule" section
  - Only show when `!viewModel.insights.isEmpty`
  - Each insight row: icon based on InsightType (lightbulb for pattern, link for connection, arrow.right.circle for actionPrompt, chart.line.uptrend for trend), title in bold, message below in secondary style
  - Use a VStack with `.padding(.horizontal, 12).padding(.vertical, 6)` matching the calendar events section style
  - Show a subtle loading indicator when `viewModel.isLoadingInsights` and insights are empty
