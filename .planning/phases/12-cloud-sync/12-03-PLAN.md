<task type="auto">
  <name>Task 2: Create SyncService actor with push, pull, and conflict resolution</name>
  <files>Sources/JarvisCore/Services/SyncService.swift</files>
  <action>
  **Create SyncService actor** (`Sources/JarvisCore/Services/SyncService.swift`):

  ```swift
  import CloudKit
  import Foundation
  ```

  **Properties:**
  - `private let cloudKit: CloudKitManager`
  - `private let store: ThoughtStore`
  - `private var serverChangeToken: CKServerChangeToken?` — persisted to UserDefaults
  - `private let changeTokenKey = "com.jarvis.cloudkit.changeToken"`
  - `public private(set) var isSyncing: Bool = false`
  - `public private(set) var lastSyncDate: Date?`

  **Initialization:**
  - `public init(cloudKit: CloudKitManager, store: ThoughtStore)`
  - Load persisted change token from UserDefaults on init (decode Data → CKServerChangeToken via NSKeyedUnarchiver)

  **`public func sync() async throws`:**
  - Guard `!isSyncing` (prevent concurrent syncs)
  - Set `isSyncing = true`, defer `isSyncing = false`
  - Call `try await cloudKit.createZoneIfNeeded()` (idempotent)
  - Call `try await pushChanges()`
  - Call `try await pullChanges()`
  - Set `lastSyncDate = Date()`

  **`private func pushChanges() async throws`:**
  1. Fetch pending thoughts: `try await store.fetchPendingSync()`
  2. Convert each to CKRecord via `cloudKit.record(from:)`
  3. Batch upload via `CKModifyRecordsOperation`:
     - `recordsToSave` = the CKRecords
     - `savePolicy = .changedKeys` (only upload modified fields)
     - Use `cloudKit.privateDatabase.modifyRecords(saving:deleting:)` (async/await API available on macOS 14+)
  4. On success for each record: call `try store.markSynced(id:)`
  5. Fetch pending deletions: `try await store.fetchPendingDeletions()`
  6. Convert to CKRecord.IDs (using cloudKitRecordID + zoneID)
  7. Delete via same `modifyRecords(saving:deleting:)` call (or separate batch)
  8. On success for each deletion: call `try store.deletePermanently(id:)`

  **`private func pullChanges() async throws`:**
  1. Use `CKFetchRecordZoneChangesOperation` with the custom zone and persisted `serverChangeToken`
  2. On macOS 14+, use the async/await `CKDatabase.recordZoneChanges(inZoneWith:since:)` API which returns `(modificationResultsByID, deletions, changeToken)`
  3. For each changed record:
     - Convert to ThoughtCloudData via `cloudKit.thoughtValues(from:)`
     - **Conflict resolution (last-write-wins):** Compare remote `modifiedAt` with local `modifiedAt`
       - If remote is newer OR local doesn't exist: `try store.upsertFromCloud(data)`
       - If local is newer AND syncStatus is `.pending`: skip (local change will be pushed next cycle)
       - If local is `.synced`: always accept remote (no local changes since last sync)
  4. For each deletion:
     - Look up local thought by cloudKitRecordID
     - If found: permanently delete via `store.deletePermanently(id:)`
  5. Persist new `serverChangeToken` to UserDefaults (encode via NSKeyedArchiver → Data)

  **Change token persistence helpers:**
  - `private func loadChangeToken() -> CKServerChangeToken?` — UserDefaults.standard.data → NSKeyedUnarchiver
  - `private func saveChangeToken(_ token: CKServerChangeToken?)` — NSKeyedArchiver → UserDefaults.standard.set

  **Error handling:**
  - Wrap CloudKit errors in a `SyncError` enum: `pushFailed(Error)`, `pullFailed(Error)`, `zoneSetupFailed(Error)`
  - Log errors but don't crash — sync is best-effort
  - If `CKError.notAuthenticated`: log clearly that iCloud sign-in is required
  - If `CKError.networkUnavailable` or `CKError.networkFailure`: silently skip (will retry next cycle)
  </action>
