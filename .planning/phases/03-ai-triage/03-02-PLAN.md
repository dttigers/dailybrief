  <name>Task 1: Show triage result in CaptureView after capture</name>
  <files>Sources/DailyBriefMonitor/CaptureView.swift, Sources/DailyBriefMonitor/AppDelegate.swift</files>
  <action>
Modify CaptureView to show the triage result after a thought is captured, instead of immediately dismissing.

**New flow (replaces current "Captured!" → 400ms → dismiss):**
1. User presses Cmd+Enter → thought saves → show "Captured!" briefly (200ms)
2. Transition to triage state: show small spinner + "Categorizing..."
3. When triage completes: show category badge (colored pill with category name) + confidence percentage
4. Auto-dismiss after 1.5 seconds, OR user can press Escape/click away to dismiss immediately
5. If triage fails: show "Captured!" only (no error shown — triage failure is silent)

**CaptureView changes:**
- Add new closure: `var onTriage: (String) async throws -> TriageResult` (takes thought content, returns TriageResult)
  - Alternative: change onCapture to return the TriageResult directly if triage succeeds. This is simpler — `onCapture` returns `TriageResult?` (nil if triage fails or is unavailable)
  - Actually, best approach: add a new callback `var onTriage: ((String) async -> TriageResult?)` that AppDelegate provides. Keep onCapture as-is for save. After save succeeds, call onTriage to get the result.
- Add state: `@State private var triageResult: TriageResult?` and `@State private var isTriaging = false`
- Update performCapture():
  1. Save thought (existing onCapture call)
  2. Show "Captured!" briefly
  3. Set isTriaging = true, call onTriage
  4. On success: set triageResult, show result for 1.5s, then dismiss
  5. On failure/nil: dismiss immediately after "Captured!"

**Triage result display:**
- Category pill: rounded rectangle with category color + category name (capitalize first letter)
  - Colors: task=blue, therapy=purple, idea=orange, reflection=green, project=indigo
- Confidence: small text next to pill showing percentage (e.g., "87%")
- Layout: horizontal stack centered in the overlay area where "Captured!" currently shows

**AppDelegate changes:**
- Move triage from fire-and-forget to a callback that CaptureView awaits
- onCapture closure: save thought, return it
- onTriage closure: call triageService.triage(content), update thought in DB, return TriageResult
- The capture still saves immediately — triage is a separate step that the UI awaits for display purposes
- Update onCapture signature: `(String) async throws -> Thought` (returns saved thought)
- Add onTriage: `(String) async -> TriageResult?` (returns nil on failure)
