  <action>
1. Add a `saveBriefSnapshot` method to VigilAPIClient that POSTs to `/v1/briefs`:
   - Accepts: date (String, YYYY-MM-DD), summary data (Codable struct with counts, categories, task summaries), pdfFilename (String?), thoughtCount (Int), taskCount (Int)
   - POSTs JSON to the API, handles errors silently (brief saving is non-critical — don't fail the generate command if this POST fails)

2. Create a `BriefSnapshot` Codable struct (in VigilAPIClient or a new file) that captures:
   - Category counts (from DailyBriefData thoughts)
   - Open task count and top task summaries (first 5 task titles)
   - Whether therapy data was present
   - Sports summary (team name, game result if any)
   - Affirmation text
   - Calendar event count
   - Work order count

3. In Generate.run(), after PDF generation succeeds, call `saveBriefSnapshot()` with the aggregated data. Use `try?` so failures don't break the generate flow. Log success/failure.

Do NOT include full thought content in the snapshot — just counts, summaries, and metadata. Keep it lightweight.
