Schedule fetch pattern for DailyBriefMonitorApp:
```swift
// Inside .task { } modifier on the MenuBarExtra body:
.task {
    // Scheduler already initialized with defaults in .onAppear.
    // Now update asynchronously from the API.
    guard let config = try? ConfigLoader.load() else { return }
    guard let url = URL(string: "\(config.apiBaseUrl)/settings/print-schedule") else { return }
    var req = URLRequest(url: url, timeoutInterval: 5)
    req.setValue("Bearer \(config.apiKey)", forHTTPHeaderField: "Authorization")
    guard let (data, resp) = try? await URLSession.shared.data(for: req),
          let http = resp as? HTTPURLResponse,
          http.statusCode == 200,
          let decoded = try? JSONDecoder().decode(PrintScheduleResponse.self, from: data)
    else { return }  // silent fallback — scheduler stays on defaults
    scheduler?.reschedule(hour: decoded.hour, minute: decoded.minute, enabled: decoded.enabled)
}

// Decodable helper (define in DailyBriefMonitorApp.swift or a new small file):
private struct PrintScheduleResponse: Decodable {
    let hour: Int
    let minute: Int
    let enabled: Bool
}
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-83-13 | Tampering | API returns malformed JSON for print-schedule | mitigate | `JSONDecoder().decode(PrintScheduleResponse.self, ...)` wrapped in `try?` — malformed response causes silent fallback to defaults. No crash, no corrupt state. |
| T-83-14 | Denial of Service | API unreachable at launch — app hangs waiting | mitigate | `URLRequest(timeoutInterval: 5)` — 5-second cap. The `.task` modifier runs concurrently; it cannot block the menu bar from rendering. BriefScheduler defaults already initialized. |
| T-83-15 | Information Disclosure | VIGIL_API_KEY sent in clear text in URLSession request | accept | All requests go to `https://api.vigilhub.io` (TLS). The Bearer token is already used by the doctor command and AppDelegate for all API calls. No regression. |
| T-83-16 | Spoofing | API base URL in config.json points to attacker server | accept | ConfigLoader reads from `~/.config/dailybrief/config.json` — user-controlled file. If compromised, the entire API surface is already accessible to the attacker. This is an existing condition, not a new one introduced by this phase. |
