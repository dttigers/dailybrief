Why retire: Mac apps all point at `https://api.vigilhub.io/v1` ([Sources/JarvisCore/Config/AppConfig.swift:28](../../Sources/JarvisCore/Config/AppConfig.swift)). Retirement unblocks Plan 107.1-05's unified `npm run dev` which binds the same port 3001.

## Retirement sequence applied

On each dev machine (LaunchAgent in `gui/$UID` domain, Aqua session — no `sudo` required):

```bash
launchctl bootout gui/$UID/com.jamesonmorrill.vigilcore
rm ~/Library/LaunchAgents/com.jamesonmorrill.vigilcore.plist
```

Execution channel:
- **iMac:** run locally on the console (current working machine during this plan execution).
- **MacBook Pro:** run over SSH via LAN (192.168.1.136). RESEARCH Pitfall 2 flagged Aqua-session risk for SSH; live probe showed `launchctl print gui/$UID/...` works over SSH on this host/macOS version, so `bootout` is attempted via SSH and verified. If silent failure surfaces, fallback is running in Terminal.app locally on the MacBook Pro.
