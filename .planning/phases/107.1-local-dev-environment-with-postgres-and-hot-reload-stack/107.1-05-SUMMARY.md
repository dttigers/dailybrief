**Thin-orchestrator root `package.json` (NOT a monorepo, D-13) chains `scripts/preflight-check.sh && concurrently --kill-others --names 'core,pwa' --prefix-colors 'blue,magenta'` so one command boots both dev servers; vigil-core dev script upgraded to `tsx --env-file=.env watch src/index.ts` (RESEARCH finding #1) so DATABASE_URL will actually load once Plan 06 repoints `.env` to localhost.**

## Accomplishments

- **Root orchestrator shipped.** `npm run dev` at repo root now chains preflight (Plan 03) → concurrently dual-boot of vigil-core + vigil-pwa. Replaces the two-terminal, two-directory, forget-to-bootout-the-daemon workflow that's been the daily dev friction for weeks.
- **Silent `tsx` bug preempted.** vigil-core dev script is now `tsx --env-file=.env watch src/index.ts`. tsx 4.x does NOT auto-load `.env` — without this flag, once Plan 06 rewrites `vigil-core/.env` to localhost values, the server would start on :3001 but every DB query would silently 500 because `process.env.DATABASE_URL` is undefined.
