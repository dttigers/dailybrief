Create a multi-stage Dockerfile in vigil-core/:

**Stage 1 (builder):**
- FROM node:20-alpine AS builder
- WORKDIR /app
- COPY package*.json ./
- RUN npm ci (includes devDeps for tsc)
- COPY . .
- RUN npm run build (runs tsc, outputs to dist/)

**Stage 2 (production):**
- FROM node:20-alpine
- WORKDIR /app
- COPY package*.json ./
- RUN npm ci --omit=dev (production deps only: hono, drizzle-orm, postgres, @anthropic-ai/sdk, @hono/node-server)
- COPY --from=builder /app/dist ./dist
- COPY drizzle ./drizzle (migration SQL files needed at runtime)
- EXPOSE 3001
- CMD ["node", "dist/index.js"]

**Notes:**
- Do NOT hardcode PORT in EXPOSE — Railway sets PORT dynamically, and the app already reads process.env.PORT
- EXPOSE 3001 is just documentation (matches the default fallback in index.ts)
- Include drizzle/ folder for production migrations

Create .dockerignore:

<task type="auto">
  <name>Task 2: Create programmatic migration script</name>
  <files>vigil-core/src/db/migrate.ts, vigil-core/package.json</files>
  <action>
Create vigil-core/src/db/migrate.ts that runs Drizzle migrations programmatically using drizzle-orm (a production dependency), NOT drizzle-kit (a devDependency).

```ts
import { drizzle } from "drizzle-orm/postgres-js";
import { migrate } from "drizzle-orm/postgres-js/migrator";
import postgres from "postgres";

const databaseUrl = process.env.DATABASE_URL;
if (!databaseUrl) {
  console.error("[migrate] DATABASE_URL not set");
  process.exit(1);
}

const client = postgres(databaseUrl, { max: 1 });
const db = drizzle(client);

console.log("[migrate] Running migrations...");
await migrate(db, { migrationsFolder: "./drizzle" });
console.log("[migrate] Migrations complete");
await client.end();
```

Add script to package.json:
- "db:migrate-prod": "node dist/db/migrate.js"

This script:
1. Requires only drizzle-orm and postgres (both production deps)
2. Connects with max 1 connection (migration only)
3. Runs all pending SQL migrations from drizzle/ folder
4. Exits cleanly after completion

Update Dockerfile CMD to run migrations before starting the app:
```dockerfile
CMD ["sh", "-c", "node dist/db/migrate.js && node dist/index.js"]
```

This ensures migrations run on every deploy, which is idempotent (Drizzle tracks applied migrations in a __drizzle_migrations table).
