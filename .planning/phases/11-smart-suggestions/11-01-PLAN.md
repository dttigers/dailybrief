  <action>
  **Insight model** (`Sources/JarvisCore/Models/Insight.swift`):
  - Create `InsightType` enum: `pattern`, `connection`, `actionPrompt`, `trend` (String, Codable, Sendable)
  - Create `Insight` struct (Codable, Sendable):
    - `type: InsightType` — what kind of insight
    - `title: String` — short headline (e.g., "Recurring Theme Detected")
    - `message: String` — the insight body (1-2 sentences)
    - `confidence: Double` — 0.0-1.0
    - `relatedThoughtIds: [Int64]` — IDs of thoughts that informed this insight (empty array default)

  **InsightService actor** (`Sources/JarvisCore/Services/InsightService.swift`):
  - Follow TriageService pattern exactly: actor with `apiKey` and `model` properties
  - Create `InsightError` enum with `apiError(String)` and `parseError(String)` cases (like TriageError)
  - Single public method: `func generateInsights(thoughts: [Thought], lookbackDays: Int) async throws -> [Insight]`
  - Claude API call structure (same HTTP pattern as TriageService):
    - System prompt: "You are a personal insight engine for someone with ADHD. Analyze their recent captured thoughts and surface useful patterns, connections between ideas, and actionable suggestions. Focus on being genuinely helpful, not generic. Return a JSON array of insights."
    - Include the thought content, categories, and dates in the user message as a formatted list
    - Ask Claude to return JSON array: `[{"type": "pattern|connection|actionPrompt|trend", "title": "...", "message": "...", "confidence": 0.0-1.0, "related_thought_ids": []}]`
    - max_tokens: 1024
  - Parse response: extract text from Claude API response, parse JSON array into [Insight]
  - Filter insights below 0.5 confidence before returning
  - If fewer than 3 thoughts provided, return empty array (not enough data for meaningful insights)
