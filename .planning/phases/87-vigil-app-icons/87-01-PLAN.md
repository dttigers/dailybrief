  <action>
    Use rsvg-convert if available (`brew install librsvg` if not — Claude's Discretion per CONTEXT), fall back to `sips` + SVG-to-PNG via `qlmanage -t`. Preferred command form shown:

    1. Generate standard "any" variants (mark at 60% of canvas — direct rasterize of master):
       ```
       for size in 192 256 384 512; do
         rsvg-convert -w $size -h $size brand/source/vigil-mark.svg \
           -o brand/pwa/pwa-${size}x${size}.png
       done
       ```

    2. Generate maskable 512 variant (D-04 — mark scaled to 80% safe zone):
       - Create `brand/source/vigil-mark-maskable.svg`: same teal bg at 1024×1024, but wrap the mark in a `<g transform="translate(128,128) scale(0.8)">` wrapping the original mark path/group so the mark occupies the inner 80% (instead of 60%) — this compensates for aggressive Android crops.
       - Actually inverse: the CANVAS stays 100% teal, the MARK art scales to 80% of the original 60% placement = keep the mark within the inner 80%×80% safe zone per D-04 + specifics section. Simplest correct form: keep mark at original size, because 60% < 80% safe zone. So maskable PNG can just be a rerender of the master SVG at 512. If any corner or edge detail extends past 80%, create a zoomed-out variant.
       - Render: `rsvg-convert -w 512 -h 512 brand/source/vigil-mark.svg -o brand/pwa/pwa-512x512-maskable.png`
       - Verify safe zone: the teal corners extend fully to the canvas edge (full bleed), and the white mark is entirely within the inner 80% (410×410 centered box). If mark strays, adjust source SVG transform and re-render.

    3. Apple touch icon (180×180, no alpha — D-05 + specifics):
       ```
       rsvg-convert -w 180 -h 180 brand/source/vigil-mark.svg -o /tmp/apple-touch.png
       # Flatten alpha onto solid teal (iOS ignores alpha anyway, but be explicit):
       sips -s format png /tmp/apple-touch.png --out brand/pwa/apple-touch-icon.png
       ```

    4. Favicon SVG (inline teal bg + white mark for browser tabs):
       `cp brand/source/vigil-mark.svg brand/pwa/favicon.svg`

    5. Favicon.ico (32×32 multi-res for legacy browsers):
       ```
       rsvg-convert -w 16 -h 16 brand/source/vigil-mark.svg -o /tmp/fav16.png
       rsvg-convert -w 32 -h 32 brand/source/vigil-mark.svg -o /tmp/fav32.png
       rsvg-convert -w 48 -h 48 brand/source/vigil-mark.svg -o /tmp/fav48.png
       # Use ImageMagick (available as `magick` or `convert`). If neither available,
       # brew install imagemagick. Alternative: use `png2ico` or `icotool`.
       magick /tmp/fav16.png /tmp/fav32.png /tmp/fav48.png brand/pwa/favicon.ico
       ```
  </action>
