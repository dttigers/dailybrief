<objective>
Wire the SSE fan-out path that closes the VOICE-06 8-second user-perceived round-trip. After Plan 02 lands the server-side `bus.emitThoughtCreated` call site, this plan: (1) implements the `emit/on/off` triple on `agent-events-bus.ts` with a three-channel joint-cleanup gate; (2) extends `agent-stream.ts` to multiplex `thought-created` frames; (3) adds a PWA SSE subscriber that dispatches the existing `vigil:thought-created` window event on receipt (which `useThoughts.ts:127` already listens for); (4) lands the three VOICE_TRANSCRIBE_* operator-copy entries in `api-error-codes.ts`; (5) lands the D8 round-trip integration test.

Purpose: Without SSE the dashboard polls every 30s and VOICE-06 is structurally unreachable (DRIFT-02). With SSE the floor is `stop→HTTP_ms (1.88s median) + SSE propagation (~50ms) + React render (~16ms) ≈ 2s` — comfortably under 8s. Multiplexing onto `/v1/agent-stream` per RESEARCH Gray Area #3 avoids a second SSE endpoint.
Output: Cross-device path works — G2 records → server transcribes + emits → SSE delivers → PWA dispatches window event → existing `useThoughts.ts:127` refetches.
</objective>
