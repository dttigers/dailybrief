<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Vigil API | API key sent as Bearer token in Authorization header over HTTPS |
| localStorage | API key stored client-side; accessible to any JS on same origin |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-63-01 | Information Disclosure | localStorage API key | accept | Single-user PWA on user's own device; no multi-user risk. XSS is the real vector — mitigated by T-63-02 |
| T-63-02 | Information Disclosure | XSS reading localStorage | mitigate | React JSX auto-escapes output; no use of dangerouslySetInnerHTML; no dynamic script injection. Vite CSP headers can be added in vercel.json if needed later |
| T-63-03 | Elevation of Privilege | CORS wildcard in production | mitigate | Set CORS_ORIGINS=https://app.vigilhub.io on Railway env var during Plan 02 deployment |
| T-63-04 | Spoofing | API key validation against wrong endpoint | mitigate | Use GET /v1/summary (authenticated) not /v1/health (public). Verified in vigil-core/src/index.ts line 61 |
</threat_model>
