LOAD-BEARING canonical SDK detail (EVEN-SKILLS.md:38):
  "Any field with a zero/default value (0, false, empty string) will be undefined, not the zero value.
   This affects: sysEvent.eventType — single click is 0, but arrives as undefined;
   listEvent.currentSelectItemIndex — first item is 0, but arrives as undefined.
   Always use nullish coalescing: event.sysEvent.eventType ?? 0."

Current NAV_EVENTS membership (vigil-g2-plugin/src/main.ts:66-70):
  SCROLL_TOP_EVENT, SCROLL_BOTTOM_EVENT, DOUBLE_CLICK_EVENT
  CLICK_EVENT (=0) is NOT a member.

Current routing pattern (vigil-g2-plugin/src/main.ts:221-260):
  - L224: list-item CLICK_EVENT branch — `event.listEvent?.eventType === OsEventTypeList.CLICK_EVENT && event.listEvent.currentSelectItemIndex != null`
    Note: strict === comparison; if eventType arrives as undefined (per SDK quirk), this branch never fires.
  - L233: list NAV branch — `event.listEvent?.eventType && NAV_EVENTS.has(event.listEvent.eventType)`
    Note: truthy check; eventType === 0 (CLICK_EVENT as the numeric value) would short-circuit falsy. eventType === undefined also short-circuits falsy.
  - L239: text NAV branch — `event.textEvent?.eventType && NAV_EVENTS.has(event.textEvent.eventType)` — same truthy issue.
  - L245-249: sys NAV branch — `const eventType = event.sysEvent.eventType; if (eventType && NAV_EVENTS.has(eventType))` — same truthy issue; relevant to Companion (sys events).
