```markdown
# Vigil Capture — iOS Shortcut

One-tap capture from iOS share sheet to Vigil. Works from Safari, Notes, Messages, and any app that shares URLs, text, or Safari web pages.

## Install

### One-tap (recommended)

Open this link on your iPhone or Mac:

<ICLOUD_URL>

Tap **Add Shortcut**. That's it.

### Manual (fallback)

1. Download `Vigil Capture.shortcut` from this directory.
2. Open the file — Shortcuts.app will launch and prompt you to add it.
3. Tap **Add Shortcut**.

## First-run setup

The Shortcut ships with a blank API key. You must paste your Vigil API key before it will work.

1. Open Shortcuts.app → tap **Vigil Capture** to edit (or long-press → **Edit**).
2. Find the **first Text action** at the top. Its label is `api_key`.
3. Tap the text body and paste your API key.
4. (Optional) The second Text action is `base_url`, pre-filled with `https://api.vigilhub.io`. Only change this if you're running your own vigil-core.
5. Tap **Done**.

## Usage

From any app's share sheet:
- **Safari page** → posts `{title}\n{url}\n\n{note}` to Vigil
- **Selected text / plain text** → posts the raw text (+ optional note)
- **URL only** → posts `{url}\n\n{note}`

When the Shortcut runs it prompts for an optional note — leave empty and hit Done to skip.

Accepted share-sheet input types: **URLs**, **Text**, **Safari web pages**. Images and files are not supported in v1.

## Sharing this Shortcut with someone else

**IMPORTANT:** Before you re-share this Shortcut (via iCloud link, AirDrop, or file export), **blank the `api_key` Text action first**. Otherwise the recipient inherits your Vigil API key and can post to your account.

To blank it:
1. Edit the Shortcut → find the first Text action (`api_key`) → delete the contents → Done.
2. Share the now-keyless Shortcut.
3. Re-paste your key back into the local copy.

This is a known limitation of Shortcuts.app — it has no per-install secret storage.

## Troubleshooting

### The Shortcut doesn't appear in the share sheet

1. Open Shortcuts.app → tap **Vigil Capture** → tap ⓘ (info) → confirm **Show in Share Sheet** is ON.
2. Confirm **Share Sheet Types** includes URLs, Text, and Safari web pages.
3. On iOS, force-close the app you're sharing from and reopen — the share sheet cache is sticky.

### "Nothing to capture" notification

You shared empty content and didn't type a note. Expected behavior — nothing was sent to the API.

### "Vigil capture failed: HTTP 401"

Your API key is wrong or blank. Edit the Shortcut → first Text action → paste a fresh key.

### "Vigil capture failed: HTTP 4xx/5xx"

- 400 → payload shape changed. Check that source is `text` inside the Get Contents of URL JSON body.
- 429 → rate limited (100 req/60s). Wait a minute.
- 5xx → vigil-core issue. Check https://api.vigilhub.io/v1/health in a browser.

### "Vigil capture failed: network error"

No connectivity, or base_url typo. Edit the second Text action (`base_url`). Should be `https://api.vigilhub.io` for the default hosted instance.

## What this Shortcut does (action-by-action)

For reference / future maintenance, see `screenshots/` for images of the action chain, or see `.planning/phases/85-ios-shortcut/85-01-PLAN.md` in the main repo for the full step-by-step recipe used to author it.

Core flow:
1. Text actions hold `api_key` and `base_url`, assigned to variables.
2. Ask for optional note.
3. If Shortcut Input is a URL → Get Details of Safari Web Page (Name) → build `title\nurl\n\nnote` content. Otherwise → build raw text + note content.
4. Empty-content guard: show "Nothing to capture" and stop.
5. POST to `{base_url}/v1/thoughts` with `Authorization: Bearer {api_key}` and JSON body `{content, source: "text"}`.
6. Show success or failure notification.

## API contract

This Shortcut uses the same endpoint as the [Vigil browser extension](../vigil-extension/):

- **Endpoint:** `POST {base_url}/v1/thoughts`
- **Headers:** `Authorization: Bearer <api_key>`, `Content-Type: application/json`
- **Body:** `{"content": "<string>", "source": "text"}`

Valid `source` values in vigil-core are `text`, `voice`, `image`. This Shortcut always uses `text`.

## Related

- [vigil-extension/](../vigil-extension/) — Chrome browser extension (same capture contract)
- [vigil-safari-extension/](../vigil-safari-extension/) — Safari extension wrapper
- Phase plan: `.planning/phases/85-ios-shortcut/`
```
