## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Plan recipes assuming feature parity across Shortcuts.app versions | Plan 85-01 assumed `Shortcut Input is URL` type comparison and direct `Status Code` property would exist. User's macOS Shortcuts only offers `has any value`/`does not have any value` on Shortcut Input. Forced mid-recipe pivot to `Get URLs from Input → Count > 0` branching. | advisory | When writing Shortcuts.app recipes, avoid branching on Shortcut Input type. Always route via `Get URLs from Input` + Count >/= 0, which normalizes across versions. |
| Magic-variable confusion between display label and underlying variable | Shortcuts chips may display a human-readable label (e.g. "Vigil capture failed") but point at an entirely different underlying variable. Happened twice this session: once in the If condition (chip labeled "Vigil capture failed" instead of Dictionary/Status Code), once in notification body. Cost significant debugging time. | advisory | When a chip's color/label seems odd, delete it and re-pick from the variable picker. Don't trust chip labels to match their variable source. |
