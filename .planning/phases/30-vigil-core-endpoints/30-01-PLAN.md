  4. Add `PaginatedResponse<T>` interface: { data: T[], total: number, limit: number, offset: number }

<task type="auto">
  <name>Task 2: Create thoughts CRUD + search routes</name>
  <files>vigil-core/src/routes/thoughts.ts, vigil-core/src/index.ts</files>
  <action>
  Create src/routes/thoughts.ts exporting a Hono instance. Follow the same pattern as summary.ts (import getDb, check for null → 503). All queries MUST exclude `syncStatus = 'pendingDeletion'` rows.

  Endpoints:

  **GET /thoughts** — List with filters
  Query params: category, source, taskStatus, therapyClassification, tag, favoritesOnly (boolean), q (search), limit (default 50, max 200), offset (default 0)
  - Build WHERE clause dynamically from provided params
  - If `q` param provided, use FTS5: `SELECT t.* FROM thoughts t JOIN thoughts_fts fts ON t.id = fts.rowid WHERE thoughts_fts MATCH ? AND t.syncStatus != 'pendingDeletion'`
  - If `tag` param provided, filter with: `json_each(tags)` table function — `EXISTS (SELECT 1 FROM json_each(t.tags) WHERE json_each.value = ?)`
  - If `favoritesOnly=true`, add `AND isFavorited = 1`
  - Return PaginatedResponse with total count (separate COUNT query with same WHERE)
  - ORDER BY createdAt DESC
  - Parse tags JSON to array in response

  **GET /thoughts/:id** — Single thought
  - Return 404 if not found or pendingDeletion
  - Include parsed tags array

  **POST /thoughts** — Create
  - Required body: { content: string, source: string }
  - Optional body: { category, tags }
  - Generate cloudKitRecordID as UUID (use crypto.randomUUID())
  - Set syncStatus = 'pending', createdAt/modifiedAt = ISO now
  - Tags: if provided as array, JSON.stringify for storage
  - INSERT INTO thoughts and return the created thought with 201 status
  - Use db.prepare(...).run() for insert, then fetch by lastInsertRowid

  **PUT /thoughts/:id** — Update
  - Accept partial body (any ThoughtUpdateInput fields)
  - Build SET clause dynamically from provided fields only
  - Always update modifiedAt to ISO now
  - Set syncStatus = 'pending' on any update (marks for re-sync)
  - Tags: if provided as array, JSON.stringify for storage
  - isFavorited: convert boolean to 0/1 integer
  - Return updated thought, 404 if not found

  **DELETE /thoughts/:id** — Soft delete
  - SET syncStatus = 'pendingDeletion', modifiedAt = ISO now
  - Return 204 on success, 404 if not found
  - Do NOT actually delete the row — Mac app's CloudKit sync handles permanent deletion

  Mount in index.ts: `app.route("/v1", thoughts)`

  Important implementation notes:
  - Use `c.req.query()` for query params, `c.req.json()` for body parsing
  - Use parameterized queries for ALL user input (SQL injection prevention)
  - Validate source is one of: text, voice, image. Return 400 for invalid.
  - Validate category if provided is one of: task, therapy, idea, reflection, project
  - content must be non-empty string, return 400 if missing/empty
  </action>

  - curl -X PUT localhost:3001/v1/thoughts/{id} -H 'Content-Type: application/json' -d '{"content":"updated"}' — returns updated thought
  - curl -X DELETE localhost:3001/v1/thoughts/{id} — returns 204
  - curl 'localhost:3001/v1/thoughts?category=task&taskStatus=open' — filtered list
  - curl 'localhost:3001/v1/thoughts?q=test' — FTS search results
  Kill server after testing.
