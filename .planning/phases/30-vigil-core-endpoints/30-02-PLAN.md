<task type="auto">
  <name>Task 1: Create tag and favorite routes</name>
  <files>vigil-core/src/routes/tags.ts</files>
  <action>
  Create src/routes/tags.ts exporting a Hono instance with these endpoints:

  **POST /thoughts/:id/tags** — Add tag to thought
  - Body: { tag: string }
  - Read current tags JSON from thought, parse to array, add new tag (deduplicate), JSON.stringify back
  - Update modifiedAt, set syncStatus = 'pending'
  - Return updated thought with parsed tags array
  - 404 if thought not found, 400 if tag empty

  **DELETE /thoughts/:id/tags/:tag** — Remove tag from thought
  - Read current tags JSON, parse, filter out the tag, JSON.stringify back
  - Update modifiedAt, set syncStatus = 'pending'
  - Return updated thought
  - 404 if thought not found, 404 if tag not present on thought

  **GET /tags** — List all unique tags
  - Query: `SELECT DISTINCT json_each.value as tag FROM thoughts, json_each(thoughts.tags) WHERE thoughts.syncStatus != 'pendingDeletion' AND thoughts.tags IS NOT NULL ORDER BY tag`
  - Return { tags: string[] }

  **PUT /thoughts/:id/favorite** — Toggle favorite
  - Read current isFavorited, flip it (0→1, 1→0)
  - Update modifiedAt, set syncStatus = 'pending'
  - Return updated thought with new favorite state
  - 404 if thought not found

  Pattern: Use getDb() check → 503 if null. Parameterized queries for all user input. All queries exclude pendingDeletion. URL-decode the :tag param for DELETE (tags may contain spaces/special chars).
  </action>
  <verify>
  Start server and test:
  - curl -X POST localhost:3001/v1/thoughts/{id}/tags -H 'Content-Type: application/json' -d '{"tag":"important"}' — adds tag
  - curl -X DELETE localhost:3001/v1/thoughts/{id}/tags/important — removes tag
  - curl localhost:3001/v1/tags — returns all unique tags
  - curl -X PUT localhost:3001/v1/thoughts/{id}/favorite — toggles favorite
  </verify>
  <done>Tag add/remove, tag list, and favorite toggle all working with proper validation and status codes.</done>

<task type="auto">
  <name>Task 2: Create thought link routes and mount all new routes</name>
  <files>vigil-core/src/routes/links.ts, vigil-core/src/index.ts</files>
  <action>
  Create src/routes/links.ts exporting a Hono instance with these endpoints:

  **POST /thoughts/:id/links** — Create bidirectional link
  - Body: { targetId: number }
  - Insert TWO rows: (sourceId→targetId) and (targetId→sourceId) for bidirectional access
  - Use INSERT OR IGNORE to handle duplicates gracefully
  - 404 if either thought not found
  - 400 if targetId equals :id (no self-links)
  - Return { linked: true, sourceId, targetId } with 201

  **DELETE /thoughts/:id/links/:linkedId** — Remove bidirectional link
  - Delete BOTH directions: (id→linkedId) and (linkedId→id)
  - Return 204 on success
  - 404 if link doesn't exist

  **GET /thoughts/:id/links** — List linked thoughts
  - Query both directions: thoughts linked FROM this thought and thoughts linked TO this thought
  - `SELECT DISTINCT t.* FROM thoughts t JOIN thought_links tl ON (t.id = tl.targetThoughtId AND tl.sourceThoughtId = ?) OR (t.id = tl.sourceThoughtId AND tl.targetThoughtId = ?) WHERE t.syncStatus != 'pendingDeletion'`
  - Return { links: ThoughtResponse[] } with parsed tags
  - 404 if source thought not found

  Mount both new route modules in src/index.ts:
  ```
  app.route("/v1", tags);
  app.route("/v1", links);
  ```

  Important: Wrap both INSERT statements for link creation in a transaction (db.transaction()) so both directions are created atomically.
  </action>
  <verify>

  - curl localhost:3001/v1/thoughts/2/links — also shows thought 1 (bidirectional)
  - curl -X DELETE localhost:3001/v1/thoughts/1/links/2 — removes link (204)
  Kill server after testing.
  </verify>
