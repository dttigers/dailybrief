Output: G2 plugin with tap-to-expand task details and working swipe navigation from detail view.

  5. Add new exported function:
     ```typescript
     export async function navigateToTaskDetail(
       itemIndex: number,
       bridge: EvenAppBridge,
     ): Promise<void> {
       const tasks = getLastFetchedTasks()
       const task = tasks[itemIndex]
       if (!task) return
       currentScreen = Screen.TASK_DETAIL
       const container = buildTaskDetailScreen(task)
       await bridge.rebuildPageContainer(container)
       console.log(`[vigil-g2] navigated to: task-detail (index ${itemIndex})`)
     }
     ```
  6. In `handleNavEvent`, update the switch cases to handle TASK_DETAIL:
     - When `currentScreen === Screen.TASK_DETAIL`:
       - SCROLL_TOP_EVENT → navigate to WORK_ORDERS (back to list)
       - SCROLL_BOTTOM_EVENT → navigate to AFFIRMATION (continue forward)
       - DOUBLE_CLICK_EVENT → navigate to HOME (already handled)
     - Since TASK_DETAIL is not in SCREEN_ORDER, getNextScreen/getPrevScreen won't work for it. Add explicit handling BEFORE the switch statement:
       ```typescript
       if (currentScreen === Screen.TASK_DETAIL) {
         if (eventType === OsEventTypeList.SCROLL_TOP_EVENT) {
           await navigateTo(Screen.WORK_ORDERS, bridge)
         } else if (eventType === OsEventTypeList.SCROLL_BOTTOM_EVENT) {
           await navigateTo(Screen.AFFIRMATION, bridge)
         } else if (eventType === OsEventTypeList.DOUBLE_CLICK_EVENT) {
           await navigateTo(Screen.HOME, bridge)
         }
         return
       }
       ```
