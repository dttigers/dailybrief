Extended `vigil-core/src/routes/calendar-auth.test.ts` with 5 new regression tests locking in the Plan 02 behavior (scope parsing, id_token email decoding, status column read-through, empty-scopes back-compat) so the Phase 81 UAT Test 4 gmail-scope gap cannot silently reopen. ## What Was Built

**Task 1 — Test harness + 5 new tests (commit `dc73066`):**

Three changes in `vigil-core/src/routes/calendar-auth.test.ts`:

1. **MOCK_TOKENS extended** with explicit field types and two new defaults:
   - `scope`: space-separated string `"...calendar.readonly ...gmail.readonly"`
   - `id_token`: structural JWT `eyJhbGciOiJIUzI1NiJ9.eyJlbWFpbCI6InVzZXJAZXhhbXBsZS5jb20ifQ.unused-signature` whose base64url payload decodes to `{"email":"user@example.com"}`
   - Typed as an explicit object literal so `Partial<typeof MOCK_TOKENS>` includes `id_token: null` as a legal override (required for the id_token-missing test).

2. **buildApp helper updated** to match Plan 02's 6-arg `dbUpsertFn` DI signature:
   - Second positional parameter `tokenOverrides?: Partial<typeof MOCK_TOKENS>` for per-test token shape.
   - `dbCalls` record now carries `{ provider, scopes, accountEmail }` — existing test at line 84 (`dbCalls[0].provider`) continues to pass because the shape is additive.
   - `getTokenFn` spreads `{ ...MOCK_TOKENS, ...tokenOverrides }` so each test can null/replace individual fields (e.g. `id_token: null`) without rebuilding the full fixture.

3. **Five new tests appended** at the end of the file:

   | Test | Asserts |
   |------|---------|
   | `79.1-scopes-parsed` | Callback splits `tokens.scope` on whitespace → 2-element array containing both URLs; `dbCalls[0].scopes` has `length === 2` and includes both calendar + gmail URLs. |
   | `79.1-email-decoded` | Callback base64url-decodes id_token payload → `dbCalls[0].accountEmail === "user@example.com"`. |
   | `79.1-email-null-when-idtoken-missing` | With `id_token: null` override → `accountEmail === null`, scopes path still parses both URLs (independent failure modes). |
   | `79.1-status-gmail-connected` | `/google/status` with scopes array containing `gmail.readonly` → `body.gmail === "connected"` (UAT Test 4 gap closed). |
   | `79.1-status-backcompat-preserved` | `/google/status` with `scopes: []` (post-Plan-01 default for legacy rows) → `calendar=connected`, `gmail=needs_auth`. |

One micro-refinement applied as natural consequence of TypeScript strictness (Rule 3, non-blocking):

- **[Rule 3 — Type infrastructure]** Declared `MOCK_TOKENS` with an explicit object literal type (`{ refresh_token: string; ...; id_token: string | null }`) rather than inferred. Required so `Partial<typeof MOCK_TOKENS>` admits `{ id_token: null }` as a legal override — otherwise TS would infer `id_token` as the literal string type and reject `null`.
