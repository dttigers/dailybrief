<objective>
Wire the plugin-side quiet-mode pipeline end-to-end. Three file groups:

1. **SSE shim extension** — `vigil-g2-plugin/src/lib/sse-client.ts` adds a `quiet_mode_changed` event-type branch to the existing event-type dispatch loop. New optional `onQuietMode?: (data: string) => void` callback in `SseClientOptions`. Invoked when the server emits the Phase 0 synthetic frame OR a runtime toggle frame.

2. **Companion screen filter + Q glyph** — `vigil-g2-plugin/src/screens/companion.ts` adds module-level `quietMode` ref + `setQuietMode/isQuietMode` exports. Per UI-SPEC §"Updated rightSide Priority Order": `computeRightSide()` prepends `Q` glyph when quiet mode is active. Per UI-SPEC §"What Quiet Mode Suppresses (HUD-Write Filter)": defense-in-depth filter at `buildContainers()` — non-allowlist banners DON'T render when quietMode=true. Server-side suppression queue (Plan 05) is the primary mechanism; this is belt-and-suspenders for the synthetic-frame-arrival race window.

3. **main.ts glue** — wire SSE `onQuietMode` callback → `companion.setQuietMode(enabled)` + `rebuildCurrentScreen` if currently on Companion screen (so `Q` glyph appears/disappears within ~100ms of PWA toggle).

Purpose: This is the load-bearing wire-up plan for AGENT-HUD-03 plugin side. After this plan, a `quiet_mode_changed` SSE frame arriving on the plugin updates `quietMode` ref + rebuilds the HUD with `Q` glyph (when on Companion). The hardware-retest in Plan 09 exercises this end-to-end against real G2 hardware.
