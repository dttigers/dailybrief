Define scope constants:
```typescript
const CALENDAR_SCOPE = "https://www.googleapis.com/auth/calendar.readonly";
const GMAIL_SCOPE = "https://www.googleapis.com/auth/gmail.readonly";
const REQUESTED_SCOPES = [CALENDAR_SCOPE, GMAIL_SCOPE];
```

Implement `createGoogleAuthRouter(deps?)`:

**GET /auth/google:**
- Read `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `GOOGLE_REDIRECT_URI` from env
- Create OAuth2Client
- Generate state JWT using `jose` SignJWT (per D-02):
  - Payload: `{ nonce: randomBytes(16).toString("hex") }`
  - Protected header: `{ alg: "HS256" }`
  - Set issued at, set expiration time `"5m"`
  - Sign with `new TextEncoder().encode(process.env["GOOGLE_OAUTH_STATE_SECRET"]!)`
  - Use `deps?.signStateFn` if provided (for tests)
- Call `client.generateAuthUrl()` with:
  - `access_type: "offline"`
  - `prompt: "consent"` (per D-09)
  - `scope: REQUESTED_SCOPES` (both calendar.readonly and gmail.readonly per D-09)
  - `state: stateJwt`
- Return `c.redirect(url)`

**GET /auth/google/callback:**
- Read `PWA_URL` from env (default `"http://localhost:5173"`)
- Get `code`, `error`, `state` from query params
- If `error` or no `code`: redirect to `${pwaUrl}?google_error=${encodeURIComponent(error ?? "no_code")}` (per D-07, note: `google_error` not `calendar_error`)
- Verify state JWT using `jwtVerify` (or `deps?.verifyStateFn`):
  - If verification fails (expired, tampered, missing): redirect to `${pwaUrl}?google_error=invalid_state`
  - Secret: `new TextEncoder().encode(process.env["GOOGLE_OAUTH_STATE_SECRET"]!)`
- Exchange code for tokens using `deps?.getTokenFn` or `client.getToken(code)`
- If no `refresh_token`: redirect with `?google_error=no_refresh_token`
- Encrypt refresh token via `encryptToken()`
- Determine granted scopes: `const grantedScopes = tokens.scope?.split(' ') ?? REQUESTED_SCOPES;` (defensive per RESEARCH.md)
- Upsert to DB via `deps?.dbUpsertFn` or default implementation that includes `scopes: grantedScopes` in both insert values and onConflictDoUpdate set
- On success: redirect to `${pwaUrl}?google_connected=true` (per D-07)
- On error: redirect to `${pwaUrl}?google_error=server_error`

Export production instance:
```typescript
export const googleAuth = createGoogleAuthRouter();
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-79-01 | Spoofing | OAuth state parameter | mitigate | JWT signed with HS256 using GOOGLE_OAUTH_STATE_SECRET; jwtVerify rejects forged tokens (replaces in-memory nonce) |
| T-79-02 | Tampering | JWT state in callback | mitigate | jwtVerify validates signature; tampered JWTs throw JWSInvalid error, callback redirects with google_error=invalid_state |
| T-79-03 | Replay/Elevation | Expired JWT state reuse | mitigate | JWT has 5-minute expiry via setExpirationTime("5m"); jwtVerify rejects expired tokens automatically |
| T-79-04 | Information Disclosure | JWT secret in logs | mitigate | Never log state JWT value or GOOGLE_OAUTH_STATE_SECRET; follow existing pattern from calendar-auth.ts (only log err.message) |
| T-79-05 | Information Disclosure | Refresh token storage | accept | Already mitigated by token-crypto.ts AES-256-GCM encryption (no changes needed) |
| T-79-06 | Denial of Service | Missing GOOGLE_OAUTH_STATE_SECRET at startup | mitigate | SignJWT will throw on empty/undefined secret; route returns 500 and logs error. Phase 79 STATE.md todo: verify env var in Railway before deploy |
</threat_model>
