**Step 1: Run the converter**

Execute from the project root directory:
```bash
xcrun safari-web-extension-converter \
  "$(pwd)/vigil-extension" \
  --app-name "Vigil Capture" \
  --bundle-identifier io.vigilhub.extension \
  --macos-only \
  --no-open
```

The `--no-open` flag prevents Xcode from opening automatically (we'll build from command line first).

The converter will create a directory -- likely named "Vigil Capture" in the current directory. Move it to `vigil-safari-extension/`:
```bash
# The converter creates a directory with the app name
# Move/rename it to our expected location
mv "Vigil Capture" vigil-safari-extension 2>/dev/null || true
```

If the converter puts output elsewhere, find and move it:
```bash
# Fallback: find the generated .xcodeproj
find . -maxdepth 2 -name "*.xcodeproj" -newer vigil-extension/manifest.json 2>/dev/null
```

**Step 2: Build from command line using xcodebuild**

```bash
cd vigil-safari-extension
xcodebuild -project "Vigil Capture.xcodeproj" \
  -scheme "Vigil Capture (macOS)" \
  -configuration Debug \
  -allowProvisioningUpdates \
  build 2>&1 | tail -20
```

If the build fails due to signing, try with `CODE_SIGNING_ALLOWED=NO`:
```bash
xcodebuild -project "Vigil Capture.xcodeproj" \
  -scheme "Vigil Capture (macOS)" \
  -configuration Debug \
  CODE_SIGNING_ALLOWED=NO \
  build 2>&1 | tail -20
```

**Step 3: Verify the build succeeded**

Check for the built .app:
```bash
find ~/Library/Developer/Xcode/DerivedData -name "Vigil Capture.app" -type d 2>/dev/null | head -1
```

If the xcodebuild approach has issues with signing or scheme names (converter-generated projects can vary), document the exact error and the manual Xcode steps required:
1. Open vigil-safari-extension/*.xcodeproj in Xcode
2. Select Team under Signing & Capabilities for both targets
3. Build with Cmd+B

**Important Safari gotchas from research:**
- `browser.storage.local.get()` must use `await` (already done in popup.js)
- `browser.notifications` not supported (not used)
- Converter may warn about unsupported manifest keys -- warnings are non-fatal for this simple extension
