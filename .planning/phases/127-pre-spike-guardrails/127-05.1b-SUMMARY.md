# Plan 127-05.1b: GUARD-03 route gating + onError + integration — Summary

## What was built

Closed the final three GUARD-03 glue items left from Plan 05.1a:

1. **`/v1/chat` pre-flight gate** — `vigil-core/src/routes/chat.ts` now calls
   `await requireAiBudget(userId)` immediately after `bearerAuth` resolves
   `userId`, before any AI work. Mirrors the gate pattern already wired into
   `/v1/process-audio` by Plan 05.1a.

2. **`app.onError` 429 branch (Pitfall 5 closure)** — `vigil-core/src/index.ts`
   now short-circuits `DailyBudgetExceededError` to HTTP 429 with
   `{error, code: "DAILY_AI_BUDGET_EXCEEDED"}` BEFORE the Sentry capture and
   PostHog event-capture sinks. A deliberate 429 is not an error to log; it's
   the contract surface a user hit. Without this short-circuit every budget
   trip would burn one Sentry slot per request once `requireAiBudget` starts
   firing in production.

3. **Cross-user-isolation extension** —
   `vigil-core/src/integration/cross-user-isolation.test.ts` now has two
   locks against cross-user reads of `ai_usage_daily`:
   - Integration test: insert userA's row, query as userB, assert zero rows
     (W-01 pattern, mirrors the work-orders isolation lock above it).
   - Source-grep `W-01-AI-BUDGET`: pins the literal
     `eq(aiUsageDaily.userId, userId)` in `vigil-core/src/lib/ai-budget.ts`
     so a future planner can't drop the filter without the test fighting
     them. Runs without DATABASE_URL.
