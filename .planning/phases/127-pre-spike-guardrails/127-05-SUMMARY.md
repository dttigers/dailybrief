## Re-scopes applied (vs CONTEXT.md)

- **`usd_estimate` precision** is `numeric(12, 6)` (Pitfall 9 override of CONTEXT D-03.1's `numeric(10, 4)`). Sub-cent precision is needed because a single chat turn at Sonnet 4 rates can be $0.00105 — under `(10, 4)` rounding that drops to $0.0011, silently inflating the watermark.
- **Migration filename is `0020_add_ai_usage_daily.sql`** (NOT `0020_reconcile_work_orders_107_1.sql` — Pitfall 2). The `work_orders` drift columns already landed in `drizzle/0013_work_orders_drift_repair.sql` (2026-04-22); GUARD-04 / Plan 07 will ship only the drift-detector test, not a new migration.
- **Migration apply path is `npm run db:migrate`** (which calls `drizzle-kit migrate`), NOT `drizzle-kit push` (RESEARCH §Anti-pattern 4). This bypasses the journal table; the canonical path drives the migrator which reads `drizzle/` and records applied migrations in `__drizzle_migrations`.
- **Two test-only pure helpers REQUIRED** (`__computeUsdForTest`, `__readCapUsdForTest`) — they make Tests 6/7 trivial pure-function assertions without a DB or mock.module dance. All 7 enumerated tests pass; the prior "at least 5 of 7" escape hatch is gone.

## Future risk surfaced

**Workspace AI cap ($500/mo, locked at Anthropic console per Phase 126)** is the absolute backstop; the **per-user $0.50/day** watermark this plan installs is the fair-use guardrail. They stack:
- Per-user $0.50/day = $15/mo/user worst-case → well under workspace cap divided by realistic user count.
- Workspace cap protects against bugs (e.g., infinite retry loop on a single user) that would otherwise breach the monthly $500.

Per-user-overridable caps remain **deferred** per CONTEXT — no `users.daily_ai_budget_usd` column today. If/when paid tiers exist (v3.10+), the override path is: add the column, swap `readCapUsd()` to query the column with env fallback.

**Per-user-timezone rollover is also deferred.** UTC 00:00 is the single global rollover (CURRENT_DATE in Postgres). The trade-off: a user crossing the cap at 11pm local sees AI back online by their morning regardless of TZ math, and capture (the core daily loop) is never blocked by AI cap — only AI features are.
