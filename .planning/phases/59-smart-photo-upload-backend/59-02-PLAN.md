### 1.3 — Handler body

The handler pipeline:

1. Parse JSON body. 400 on malformed.
2. Validate `body.image` is a non-empty string. 400 if not.
3. Validate `body.mediaType` is a string and is in `VALID_MEDIA_TYPES`. 400 if not.
4. Call `deps.getAIClientFn()` — if null, return 503 with the exact describe-image error message.
5. Call `deps.callClaudeFn({...})` with the image + PHOTO_PROMPT + `maxTokens: 2000`. On throw, return 502 with `{error: err.message}` (exactly matching describe-image's 502 shape). Log via `console.error("[vigil-core] /process-photo Claude call failed:", err)` — NEVER log the base64 image (T-59-04).
6. Run the returned `rawText` through `processClaudeResponse(rawText)` — already tested in Plan 01. This gives you `{paperType, confidence, thoughts: string[]}`.
7. Build the insert rows:
   ```ts
   const insertRows = result.thoughts.map((content) => ({
     content,
     source: "image" as const,
     confidence: result.confidence,
     cloudKitRecordID: crypto.randomUUID(),
   }));
   ```
   Every row gets a DISTINCT `randomUUID()` (P-7).
8. Call `deps.dbInsertFn(insertRows)` — ONE call, batched. On throw, return 500 with `{error: "Create failed"}` and `console.error(...)` log.

|-----------|----------|-----------|-------------|-----------------|
| T-59-01 | DoS (oversized base64 payload) | POST /process-photo body parsing | accept | Covered by existing `timeout(30_000)` middleware (index.ts:50) and global rate limiter.
