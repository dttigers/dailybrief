**Part A: Config toggle**

Add a `vigil` section to AppConfig:
```swift
public struct VigilConfig: Codable, Sendable {
    public var useAPI: Bool = false
    public var apiBaseURL: String = "http://localhost:3001/v1"
}
```

Add `public var vigil: VigilConfig?` to AppConfig (optional so existing configs without this field still parse). Default to `useAPI: false` when not present.

**Part B: AppDelegate wiring**

In AppDelegate where ThoughtStore is created (around line 70-75), add conditional initialization:

```swift
let repository: any ThoughtRepository
if config.vigil?.useAPI == true {
    let apiURL = URL(string: config.vigil?.apiBaseURL ?? "http://localhost:3001/v1")!
    let client = VigilAPIClient(baseURL: apiURL)
    repository = APIThoughtStore(client: client)
} else {
    let dbManager = try DatabaseManager()
    repository = ThoughtStore(database: dbManager)
}
```

IMPORTANT: Keep the existing ThoughtStore initialization as the default path. The API path is opt-in only.

The `thoughtStore` property used downstream must change type from `ThoughtStore` to `any ThoughtRepository`. Update the property declaration accordingly.

For SyncService: SyncService needs the concrete ThoughtStore (for sync methods not in the protocol). When using API mode, SyncService should be disabled (sync is handled differently with API backend). Add a conditional: only create SyncService when NOT using API mode.

Pass `repository` (as `any ThoughtRepository`) to DashboardViewModel and CaptureService.
