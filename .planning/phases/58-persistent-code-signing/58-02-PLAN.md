<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer workstation → macOS keychain | bootstrap.sh and dailybrief-doctor.sh both query the login keychain via `security find-identity` (read-only). Neither script writes to the keychain or imports certs. |
| developer workstation → stderr/stdout | Error messages printed by bootstrap.sh on missing cert are instructional (remediation command) and do not leak keychain contents or existing identity names. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-58-08 | Spoofing | bootstrap.sh cert resolution | mitigate | Same `security find-identity -v -p codesigning` + grep "Developer ID Application" as install.sh (Plan 01). The two scripts use identical cert-resolution logic, so neither can accept a cert the other would reject. |
| T-58-09 | Tampering | Removed build.sh codesign line is quietly re-added in a later commit | accept | Mitigated by comment block in build.sh explaining the Phase 58 removal reason. Not enforceable by shell-level protections. A future regression would be caught by the Plan 01 install.sh cert guard because the unsigned .build/release/DailyBrief never reaches TCC anyway (install.sh signs in ~/.local/bin/). |
| T-58-10 | Information Disclosure | dailybrief-doctor.sh prints the full Dev ID cert identity name | accept | The identity string (e.g., `Developer ID Application: Jameson Morrill (TEAMID)`) contains the developer's real name and Apple Team ID. This is PUBLIC information — it's embedded in every signed binary and visible via `codesign -dvv` on any shipped artifact. No new disclosure. |
| T-58-11 | Repudiation | Pre-flight cert check passes but the resolved identity is different from install.sh's resolution | mitigate | Both scripts use `head -1` deterministic first-match on the same query. The identity printed by bootstrap.sh pre-flight OK line is the same one install.sh will use downstream. Acceptance criterion verifies the shell variable is surfaced in the OK line for visibility. |
| T-58-12 | Denial of Service | dailybrief-doctor.sh hangs on `security find-identity` | accept | `security find-identity` is a local keychain query with no network dependency and typical runtime <100ms. 2>/dev/null suppresses noise. No timeout needed — if it hangs, the whole keychain system is compromised and the doctor script is not the right layer to fix it. |
</threat_model>
