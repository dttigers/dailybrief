3. Create `vigil-core/src/routes/work-orders.ts` with two endpoints:

**POST /work-orders/sync** — Accepts `{ workOrders: WorkOrder[] }`. For each work order, upserts into `work_orders` table by `caseNumber` (on conflict update all fields + syncedAt). Returns `{ synced: number }`.
- Mass-assignment defense: destructure only the 9 known fields from each object.
- Validate workOrders is a non-empty array.

**GET /work-orders** — Returns all work orders joined with their status from `work_order_statuses`. Response shape:
```typescript
{
  data: Array<{
    caseNumber: string;
    store: string;
    shortDescription: string;
    trade: string;
    location: string;
    equipment: string;
    priority: string;
    contact: string;
    state: string;
    status: string; // from work_order_statuses, default "open"
    syncedAt: string;
  }>
}
```
Use a LEFT JOIN on `workOrderStatuses` by `caseNumber`. If no status row exists, default to `"open"`.

4. Mount the route in `vigil-core/src/index.ts`: Import `workOrders` from `"./routes/work-orders.js"` and add `app.route("/v1", workOrders);` in the protected routes section, BEFORE the existing `workOrderStatus` route to avoid path conflicts.
