   **Option A (preferred — minimal hook change):** In `useChat.ts`, find the mount effect that calls `getChatSessions()` then `loadSession()`. Wrap the `loadSession()` call in a check:
   ```typescript
   // In useChat.ts mount effect, after getChatSessions() resolves:
   // Only auto-load the last session if no external caller has claimed control
   if (!skipAutoLoadRef?.current) {
     loadSession(sessions[0].id)
   }
   ```
   To make `skipAutoLoadRef` available: add an optional `skipAutoLoad?: React.MutableRefObject<boolean>` parameter to `useChat()`, thread it in, and ChatPage passes `skipAutoLoadRef`.

   **Option B (simpler — no hook change):** In ChatPage, use `useLayoutEffect` (runs before `useEffect`) to set `clearChat()` and the thought-handled flag BEFORE the `useChat` mount effect runs:
   ```typescript
   import { useLayoutEffect } from 'react'

   useLayoutEffect(() => {
     const state = location.state as { thoughtText?: string; thoughtId?: number } | null
     if (!state?.thoughtText) return
     skipAutoLoadRef.current = true
   }, [])
   ```

   Then the main `useEffect` handles the send:
   ```typescript
   // Auto-send thought when arriving from ThoughtsPage (D-02, D-03, D-04)
   useEffect(() => {
     if (thoughtHandledRef.current) return
     const state = location.state as { thoughtText?: string; thoughtId?: number } | null
     if (!state?.thoughtText) return

     thoughtHandledRef.current = true

     // Clear any auto-resumed session so sendMessage creates a fresh one (D-03)
     clearChat()

     // Inject thought as first user message and auto-send (D-02, D-04)
     // sendMessage reads messagesRef.current ([] after clearChat), auto-creates session
     sendMessage(state.thoughtText)

     // Clear location.state to prevent replay on tab switch
     window.history.replaceState({}, '')
     // eslint-disable-next-line react-hooks/exhaustive-deps
   }, [])
   ```

   **Choose Option B** unless the executor finds that `useChat`'s auto-load `useEffect` runs before ChatPage's `useEffect` even with `useLayoutEffect` (unlikely — layout effects fire synchronously before paint, then effects fire in mount order). If Option B doesn't prevent the race (test by checking if the old session briefly flashes before clearing), switch to Option A.

**Why `thoughtHandledRef`:** Prevents double-send if React strict mode or concurrent features re-run the effect. `window.history.replaceState` clears state for future navigations; the ref guards the current render cycle.

**Why `skipAutoLoadRef`:** Prevents the `getChatSessions()` → `loadSession()` race in `useChat`'s mount effect from overwriting the cleared state. Without this, if `loadSession()` resolves after `clearChat()`, the old session could be loaded back, and the thought message could land in the wrong session.

**Why empty deps `[]`:** Must run once on mount only. `sendMessage` and `clearChat` are stable refs. The ESLint suppression comment documents the intentional omission.
