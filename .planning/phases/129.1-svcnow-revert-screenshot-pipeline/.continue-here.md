# Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Hypothesis space too narrow on planned diagnostics | Plan 129-10 enumerated H1/H2/H3 for GAP-129-G; actual root cause (H4 dispatch-table incomplete) was outside the planned space. Diagnostic evidence correctly ruled out H1/H2/H3 but I had to read code directly to spot H4. | advisory | When diagnostic evidence rules out all planned hypotheses, immediately read the code path between the last-successful-checkpoint (e.g. `pickInitialScreen` returned correct value) and the failing observation (e.g. plugin painted wrong screen). The "between" is often the unsuspected place. Use NONE_OF_ABOVE explicitly. |
| Cross-machine git divergence during multi-day UAT | Operator's Mac was 30 commits behind origin/main while Linux dev box was 30 commits ahead. Scenario 8's setup-view never rendered until the Mac pulled. ~15 min of debugging burned before discovering the stale checkout. | advisory | Before starting hardware UAT on a different machine than where code was authored: verify `git log origin/main..HEAD` and `git rev-parse HEAD` match. Add to Session 0 of future cross-machine UAT runbooks. |
| Auto-mode classifier blocks `git push origin main` even after AskUserQuestion approval | Claude Code's classifier blocks direct pushes to default branch independently of conversational consent. Required operator to explicitly grant bypass before push could proceed. | advisory | Either pre-add `Bash(git push origin main)` to `.claude/settings.json`, or accept that pushes-to-default-branch always require operator manual step. Don't burn cycles retrying the same blocked command. |
