- [ ] CONSTRAINT: **Stale-base worktree resurrection** — The `worktree_branch_check` soft-reset updates refs but not the working tree, so executor commits can carry the worktree's original (stale) tree. On merge, stale trees resurrect files that main has since deleted. Structural mitigation: when main has cleanup commits unreflected in the worktree's branch tail (e.g., `git rm -r` of archived directories), either disable worktrees for that phase (`workflow.use_worktrees=false`) or have the check do `git reset --hard $EXPECTED_BASE` after index reconciliation.

- [ ] CONSTRAINT: **gsd-verifier polluted main working tree** — Running the read-only verifier on the main tree (no worktree isolation) resulted in 50+ unexpected modifications/deletions (vitest.config.ts, SettingsPage.test.tsx, GoogleStatusContext, etc., plus 5 archived phase directories resurrected as untracked). Root cause not determined. Structural mitigation: spawn gsd-verifier with `isolation="worktree"` from a stable base, or verify on a clean ephemeral clone. Do NOT let the verifier run against a tree that holds recent large-scale cleanup deltas.

**Do not proceed until both boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Stale-base worktree resurrection | Wave 1 worktree created from pre-archival commit; soft-reset to EXPECTED_BASE updated refs only. Merge brought back 153 deleted files and orphaned 3 executor commits. | blocking | Hard-reset (not soft) after branch-base mismatch; OR disable worktrees when main has cleanup deltas unreflected in worktree chain. |
| Verifier tree pollution | gsd-verifier run on main tree produced 50+ unexpected file deltas. Recovered via `git checkout HEAD -- .` but root cause unidentified. | blocking | Isolate gsd-verifier in a worktree or ephemeral clone; never run against the same tree executors just modified at scale. |
| Chained `&&`/`||` in recovery shell | `merge \|\| commit --amend && worktree remove && branch -D` deletes the worktree branch even when merge fails, orphaning commits. | advisory | Never combine destructive cleanup (`worktree remove`, `branch -D`) with merges in a single `&&` chain. Separate steps; verify merge succeeded first. |
