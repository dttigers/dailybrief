  <action>
Fix the LaunchAgent configuration to prevent SIGILL crashes:

1. **Update repo plist** (`LaunchAgent/com.jamesonmorrill.dailybriefmonitor.plist`):
   - Change ProgramArguments to use `~/.local/bin/DailyBriefMonitor` (the installed path, matching install.sh)
   - Add `LimitLoadToSessionType` key with value `Aqua` — this ensures the agent only loads in GUI login sessions, preventing crashes when WindowServer isn't ready
   - Add `ProcessType` key with value `Interactive` — tells launchd this is a GUI process
   - Change `KeepAlive` from bare `true` to a dict with `SuccessfulExit: false` — only restart on crash, don't restart if the app exits cleanly (exit 0). This prevents infinite restart loops during debugging.
   - Keep existing StandardOutPath, StandardErrorPath, and EnvironmentVariables

2. **Update install.sh** to match:
   - Update the generated plist in install.sh to include the same `LimitLoadToSessionType`, `ProcessType`, and `KeepAlive` dict changes
   - Use `launchctl bootout` + `launchctl bootstrap` instead of deprecated `launchctl unload`/`load` (the old API is deprecated on modern macOS). Use `gui/$(id -u)` as the domain target.
   - Add a post-install verification step: `launchctl print gui/$(id -u)/$MONITOR_LABEL` to confirm the agent loaded

  </action>
