| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F9F9F8` (`--color-surface`) | Page background |
| Secondary (30%) | `#2C2C2A` (`--color-gray-900`) | Card/section backgrounds (`bg-gray-900`) |

Accent (`#1D9E75` / teal) reserved for:
1. Checkbox `accent-teal-600` — the checked-state fill color on `<input type="checkbox">`
2. League-enabled toggle border highlight (`border-teal-600/40`) when a league is active
3. League section header teal dot or enabled indicator (if used — see Interaction Patterns)
4. "PRIMARY" badge background (`bg-teal-500/20 text-teal-400`) — only if a team has a "primary" designation (unlikely for sports; use sparingly)
5. Retry link text (`text-teal-400 hover:text-teal-300`) in error state

Accent is NOT used for:
- Disabled league sections (use `opacity-50` or `text-gray-400`)
- Team list item backgrounds (plain, no tint)
- Section dividers (use `border-gray-900/40`)

Source: `SettingsPage.tsx` and `index.css` token set. Teal is the sole Vigil brand accent per brand guidelines memory.

## Spacing and Layout

### Section placement

The Sports picker lives as a new `<section>` card in `SettingsPage.tsx`, beneath the Google Account card and above the ScheduleCard rows. It mirrors the card pattern exactly:

```
<section className="bg-gray-900 border border-gray-900/40 rounded-lg p-5 mt-4">
  <h2 className="text-lg font-medium">Sports</h2>
  <p className="text-xs text-gray-500 mb-4">
    Choose which leagues and favorite teams appear in your daily brief.
  </p>
  {/* league rows */}
</section>
```

### League row structure

Each league (MLB, NFL, NBA, NHL) occupies a block with:
1. A top-level league toggle row: `[checkbox] [League name]` — enables/disables the entire league
2. When enabled: a team picker sub-list indented beneath it

Layout:
```
[checkbox] MLB
  └─ Loading… / error / team list
      [radio] Boston Red Sox
      [radio] New York Yankees
      …
```

League rows are separated by `mt-4 border-t border-gray-900/40 pt-4` after the first league (same divider pattern as the calendars subsection).

### Team picker list

- Each team is a single-select radio within its league (one favorite team per league)
- Row pattern: `[radio] [Team full name]`
- Radio uses `accent-teal-600` for checked-state color, same as calendar checkboxes
- No color swatch (teams have no color metadata from the API at this layer; balldontlie returns name strings only)
- List is alphabetically sorted — the API returns full team name strings; sort client-side

## Interaction Patterns

### League enable/disable (top-level checkbox)

- `<input type="checkbox">` per league, `accent-teal-600`
- Optimistic toggle: checked state updates immediately on click
- Debounced save: 400ms after last change, PUT the full sports preferences object
- If save fails: rollback to last-known-good server state, show error toast ("Couldn't save sports settings — try again")
- Visual state when league disabled: team picker sub-list is hidden (not rendered, not `display:none`)
- Visual state when league enabled but no team selected: team list is visible with a helper note

### Team selection (radio within an enabled league)

- Single-select radio per league: `<input type="radio" name="team-{league}">` — one favorite team at a time
- Selecting a team within an enabled league triggers the same debounced 400ms PUT
- Optimistic selection: radio updates immediately
- Rollback behavior on failure: same as league toggle

### Save UX (matching Phase 115 pattern)

- Auto-save per interaction with 400ms debounce — NO explicit Save button
- No unsaved-state indicator
- Error toast on failure via existing `useToast()` / `ToastHost`
- Debounce timer cancelled on component unmount (same WR-02 known limitation as Phase 115 — explicitly deferred)

### Loading states

- League loading placeholder: `<p className="text-gray-400 text-sm">Loading teams…</p>` — displayed per-league while team list is being fetched
- Error state per league: `<div className="bg-red-900/20 border border-red-900/40 rounded p-3">` with Retry button, exactly matching the calendar error pattern
- If team list API is unavailable: league toggle still works; team picker shows error with Retry

### Empty/disabled league state in the brief

When all leagues are disabled, the brief omits the sports section entirely (success criterion SC#4). This is purely a server-side behavior — the PWA only needs to communicate the semantic clearly through the settings UI, not render a preview.

Notes:
- League labels include the sport name after the acronym (e.g., "MLB — Baseball") because solo-dev tooling — clarity beats terseness.
- "Standings only" helper for no-team-selected is the honest fallback: the sports-service can still show league standings without a favorite team. If the implementation cannot support this, change to: "Select a team to track game results."
- No destructive actions in this phase — no confirmation dialogs needed.

## State Machine per League

```
league_enabled=false
  → checkbox click → league_enabled=true (optimistic) → debounce PUT → [success | rollback]

league_enabled=true, team_selected=null
  → radio click → team_selected="Team Name" (optimistic) → debounce PUT → [success | rollback]

league_enabled=true, team_selected="Team Name"
  → different radio click → team_selected updated (optimistic) → debounce PUT → [success | rollback]
  → checkbox click (disable) → league_enabled=false, team_selected preserved (not cleared) → debounce PUT
```

Preservation rule: disabling a league does NOT clear the team selection. If the user re-enables the league, the previous team selection is restored from server state on next load.

## Data Shape (Frontend Contract)

The PWA stores and sends a single `sportsSelections` object per user:

```typescript
interface SportsSelections {
  enabledLeagues: Array<"mlb" | "nfl" | "nba" | "nhl">;
  favoriteTeams: {
    mlb?: string;   // team name string matching balldontlie full_name / display_name
    nfl?: string;
    nba?: string;
    nhl?: string;
  };
}
```

This is the shape sent to `PUT /v1/sports/selections` (new endpoint, modelled after `PUT /v1/calendar/selections`).

Empty `enabledLeagues: []` = no sports section in brief.
A league present in `enabledLeagues` but absent from `favoriteTeams` = standings-only for that league.

## Accessibility

- All checkboxes and radios have an associated visible `<label>` or `aria-label` — no icon-only controls
- League rows maintain minimum 44px touch target height (via `py-2` on `<li>` or `<label>`)
- Error regions use `role="alert"` for screen reader announcement — matching the existing toast pattern
- The Sports section is wrapped in `<section aria-labelledby="sports-heading">` with `id="sports-heading"` on the `<h2>` element
- Disabled team list (league unchecked) is not rendered — no `aria-disabled` needed; absence is the correct semantic
