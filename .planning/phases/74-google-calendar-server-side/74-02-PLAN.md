    **Factory function `createCalendarService(deps?)`:**

    **`getValidAccessToken()` (internal, per CAL-02):**
    - Query oauth_tokens for provider='google' (via `dbSelectFn` or real DB)
    - If no row: throw `TokenNotFoundError` (custom class extending Error)
    - Check if `expiresAt` is null or within 5 minutes of now
    - If expired: decrypt refresh token via `decryptToken()`, call `refreshFn` (or real OAuth2Client.refreshAccessToken), update DB with new access token + expiry via `dbUpdateFn`
    - If refresh fails: throw `TokenRevokedError` (custom class extending Error)
    - Return valid access token string

    **`fetchTodaysEvents()` -> `CalendarEventsResponse` (per D-10, D-11):**
    - Try `getValidAccessToken()` — catch TokenNotFoundError/TokenRevokedError -> return `{ status: "needs_reauth" }`
    - Get calendar selections from DB row. If empty array: call `fetchCalendarListRaw()` to get all calendar IDs
    - For each calendar ID, fetch today's events from Google Calendar API:
      ```
      GET https://www.googleapis.com/calendar/v3/calendars/{calendarId}/events
      ?timeMin={todayStart.toISOString()}
      &timeMax={todayEnd.toISOString()}
      &singleEvents=true
      &orderBy=startTime
      &maxResults=50
      Headers: Authorization: Bearer {accessToken}
      ```
    - `todayStart` = midnight local time today, `todayEnd` = midnight tomorrow (use `new Date()` with year/month/date constructor)
    - Normalize each event: check `start.dateTime` vs `start.date` for allDay detection. Map to CalendarEvent shape.
    - On fetch 401: return `{ status: "needs_reauth" }`
    - On fetch network error: return `{ status: "error", error: "Google Calendar API unreachable" }`
    - Return `{ status: "ok", events: [...], fetchedAt: new Date().toISOString() }`

    **`fetchCalendarList()` -> `CalendarListResponse` (per D-08):**
    - Try `getValidAccessToken()` — catch -> return `{ status: "needs_reauth" }`
    - Fetch `GET https://www.googleapis.com/calendar/v3/users/me/calendarList` with Bearer token
    - Map response items to CalendarInfo: `{ id: item.id, name: item.summary, color: item.backgroundColor, primary: item.primary ?? false }`
    - Handle 401 -> `{ status: "needs_reauth" }`, network error -> `{ status: "error", error: "..." }`

    Return `{ fetchTodaysEvents, fetchCalendarList }`.

    **Production export:**

    ```typescript
    import { Hono } from "hono";
    import { createCalendarService } from "../services/calendar-service.js";
    import type { CalendarServiceDeps } from "../services/calendar-service.js";

    export function createCalendarRouter(deps?: CalendarServiceDeps): Hono {
      const service = createCalendarService(deps);
      const router = new Hono();

      // GET /calendar/events — today's events for brief generation (per D-11)
      router.get("/calendar/events", async (c) => {
        const result = await service.fetchTodaysEvents();
        return c.json(result);
      });

      // GET /calendar/list — available calendars for selection UI (per D-08)
      router.get("/calendar/list", async (c) => {
        const result = await service.fetchCalendarList();
        return c.json(result);
      });

      return router;
    }

    export const calendar = createCalendarRouter();
    ```

| Boundary | Description |
|----------|-------------|
| Client -> /v1/calendar/* | API key-authenticated requests for calendar data |
| Server -> Google Calendar API | Server uses stored access token to fetch events |
| Server -> PostgreSQL | Read/update encrypted tokens |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-74-06 | Information Disclosure | /v1/calendar/events response | mitigate | Only return D-10 fields (title, start, end, location, calendarName, calendarColor). No attendees, descriptions, conference links. |
| T-74-07 | Elevation of Privilege | /v1/calendar/* routes | mitigate | Routes registered after bearerAuth middleware — require valid API key |
| T-74-08 | Information Disclosure | token refresh logs | mitigate | Log "token refreshed for provider=google" — never log the actual tokens |
| T-74-09 | Denial of Service | Google Calendar API rate limits | accept | Single-user system; rate limits unlikely to be hit. If needed, add caching in future phase. |
</threat_model>
