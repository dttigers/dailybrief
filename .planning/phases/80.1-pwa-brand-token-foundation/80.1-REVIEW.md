# Phase 80.1: Code Review Report

**Reviewed:** 2026-04-13T12:00:00Z
**Depth:** standard
**Files Reviewed:** 25
**Status:** issues_found

## Summary

Phase 80.1 migrated PWA components from hardcoded Tailwind color classes to brand tokens (`info`, `warning`, `success`, `teal`). The font-weight corrections (semibold/bold to medium) are complete -- no `font-semibold` or `font-bold` remains. The old `slate-`, `indigo-`, and `blue-` palette classes have been fully removed.

However, the critical finding is that the brand token definitions are missing. Components reference classes like `bg-info-50`, `text-warning-400`, `bg-success-50` but no `@theme` block, Tailwind config, or CSS custom properties define these tokens. These classes will silently produce no styles at runtime, making affected badges invisible or unstyled.

## Critical Issues

### CR-01: Brand token colors undefined -- all semantic badges render with no background/text color

**File:** `vigil-pwa/src/index.css:1-2`
**Issue:** The file contains only `@import "tailwindcss"` with no `@theme` block defining the custom color tokens (`info`, `warning`, `success`). There is no `tailwind.config.*` file either. Components across the codebase reference these undefined classes:
- `bg-info-50`, `text-info-400` (used in ThoughtRow, ThoughtAssignmentRow, WorkOrderRow, ProjectCard, InsightsPage)
- `bg-warning-50`, `text-warning-400` (used in ThoughtRow, ThoughtAssignmentRow)
- `bg-success-50`, `text-success-400` (used in ThoughtRow, ThoughtAssignmentRow)
- `bg-teal-50`, `text-teal-50` (used in ThoughtRow, ThoughtAssignmentRow, WorkOrderRow)

Note: `teal` is a default Tailwind v4 color, so `bg-teal-600`, `text-teal-400` etc. work fine. But `bg-teal-50` paired with `text-teal-600` as a badge background may appear near-white on a dark theme, which could be a design intent issue.

**Fix:** Add a `@theme` block to `index.css` defining the semantic color tokens:
```css
@import "tailwindcss";

@theme {
  --color-info-50: oklch(0.25 0.05 250);
  --color-info-400: oklch(0.7 0.15 250);
  --color-warning-50: oklch(0.25 0.05 85);
  --color-warning-400: oklch(0.75 0.15 85);
  --color-success-50: oklch(0.25 0.05 155);
  --color-success-400: oklch(0.7 0.15 155);
}
```
(Adjust exact values to match your brand spec. The key point is these tokens must be defined for the classes to produce any CSS output.)

## Warnings

### WR-01: Fallback badge color `bg-gray-50` renders near-white on dark background

**File:** `vigil-pwa/src/components/ThoughtRow.tsx:69-70`
**Issue:** The fallback style for uncategorized or unknown categories is `'bg-gray-50 text-gray-400'`. In Tailwind, `gray-50` is near-white (#f9fafb), which will create a bright white badge background against the dark gray-900 page. This same pattern appears in:
- `ThoughtAssignmentRow.tsx:24-25`
- `ProjectCard.tsx:7,29`

**Fix:** Use a dark-appropriate fallback like `bg-gray-400/30 text-gray-400` (matches the pattern used elsewhere in the codebase for inactive/default states):
```typescript
const categoryStyle = thought.category
  ? (CATEGORY_STYLES[thought.category] ?? 'bg-gray-400/30 text-gray-400')
  : 'bg-gray-400/30 text-gray-400'
```

### WR-02: Inconsistent category badge shape between ThoughtRow and ThoughtAssignmentRow

**File:** `vigil-pwa/src/components/ThoughtAssignmentRow.tsx:35`
**Issue:** Category badges in `ThoughtAssignmentRow` use `rounded` (small border-radius, square-ish) while the same category badges in `ThoughtRow` (line 137) use `rounded-full` (pill shape). The phase description states "category=pill" as the correct shape, so `ThoughtAssignmentRow` appears to be using the wrong shape.

**Fix:** Change `rounded` to `rounded-full` on line 35 of ThoughtAssignmentRow.tsx:
```tsx
<span className={`text-xs px-1.5 py-0.5 rounded-full font-medium ${categoryColor}`}>
```
