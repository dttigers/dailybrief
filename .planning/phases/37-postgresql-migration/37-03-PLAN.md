  <name>Task 1: Migrate tags.ts routes to Drizzle</name>
  <files>vigil-core/src/routes/tags.ts</files>
  <action>
  Replace all better-sqlite3 imports and queries with Drizzle ORM.

  **POST /thoughts/:id/tags (add tag):**
  With JSONB, use PostgreSQL's jsonb concatenation instead of read-modify-write:
  ```typescript
  // First check thought exists
  const [thought] = await db.select({ id: thoughts.id, tags: thoughts.tags })
    .from(thoughts).where(eq(thoughts.id, id));

  // Append tag if not already present
  const currentTags: string[] = (thought.tags as string[]) || [];
  if (!currentTags.includes(tag)) {
    currentTags.push(tag);
    await db.update(thoughts).set({
      tags: currentTags,
      modifiedAt: new Date(),
      syncStatus: 'pending',
    }).where(eq(thoughts.id, id));
  }
  ```
  Alternatively, use raw SQL for atomic JSONB append:
  ```typescript
  sql`CASE WHEN ${thoughts.tags} IS NULL THEN ${JSON.stringify([tag])}::jsonb
      WHEN NOT ${thoughts.tags} @> ${JSON.stringify([tag])}::jsonb
      THEN ${thoughts.tags} || ${JSON.stringify([tag])}::jsonb
      ELSE ${thoughts.tags} END`
  ```
  Use whichever approach is cleaner. The read-modify-write is simpler and fine for single-user.

  **DELETE /thoughts/:id/tags/:tag (remove tag):**
  Read current tags, filter out the target tag, update:
  ```typescript
  const currentTags: string[] = (thought.tags as string[]) || [];
  const newTags = currentTags.filter(t => t !== tag);
  await db.update(thoughts).set({
    tags: newTags.length > 0 ? newTags : null,
    modifiedAt: new Date(),
    syncStatus: 'pending',
  }).where(eq(thoughts.id, id));
  ```

  **GET /tags (list all unique tags):**
  Replace `json_each()` with PostgreSQL's `jsonb_array_elements_text()`:
  ```typescript
  const result = await db.execute(
    sql`SELECT DISTINCT jsonb_array_elements_text(tags) as tag
        FROM ${thoughts}
        WHERE tags IS NOT NULL AND sync_status != 'pendingDeletion'
        ORDER BY tag`
  );
  ```

  **PUT /thoughts/:id/favorite (toggle favorite):**
  ```typescript
  const [thought] = await db.select({ isFavorited: thoughts.isFavorited })
    .from(thoughts).where(eq(thoughts.id, id));

  const [updated] = await db.update(thoughts).set({
    isFavorited: !thought.isFavorited,
    modifiedAt: new Date(),
    syncStatus: 'pending',
  }).where(eq(thoughts.id, id)).returning();
  ```
  With native boolean, no more 0/1 conversion needed.

  **Remove all better-sqlite3 references from tags.ts.**

  <action>
  **links.ts — POST /thoughts/:id/links (create bidirectional link):**
  Use Drizzle's transaction for atomicity:
  ```typescript
  await db.transaction(async (tx) => {
    await tx.insert(thoughtLinks).values({
      sourceThoughtId: id,
      targetThoughtId: linkedId,
    }).onConflictDoNothing();
    await tx.insert(thoughtLinks).values({
      sourceThoughtId: linkedId,
      targetThoughtId: id,
    }).onConflictDoNothing();
  });
  ```
  Use `.onConflictDoNothing()` instead of SQLite's `INSERT OR IGNORE`. This requires the unique constraint on (sourceThoughtId, targetThoughtId) defined in the schema.

  **links.ts — DELETE /thoughts/:id/links/:linkedId (remove bidirectional link):**
  ```typescript
  await db.transaction(async (tx) => {
    await tx.delete(thoughtLinks).where(
      and(eq(thoughtLinks.sourceThoughtId, id), eq(thoughtLinks.targetThoughtId, linkedId))
    );
    await tx.delete(thoughtLinks).where(
      and(eq(thoughtLinks.sourceThoughtId, linkedId), eq(thoughtLinks.targetThoughtId, id))
    );
  });
  ```

  **links.ts — GET /thoughts/:id/links (get linked thoughts):**
  Replace the complex JOIN with Drizzle:
  ```typescript
  const linked = await db.select({ thought: thoughts })
    .from(thoughtLinks)
    .innerJoin(thoughts, or(
      and(eq(thoughtLinks.sourceThoughtId, id), eq(thoughtLinks.targetThoughtId, thoughts.id)),
      and(eq(thoughtLinks.targetThoughtId, id), eq(thoughtLinks.sourceThoughtId, thoughts.id))
    ))
    .where(ne(thoughts.syncStatus, 'pendingDeletion'));
  ```
  Or use a simpler approach: query thoughtLinks for the id, collect all linked IDs, then query thoughts by those IDs.

  **bulk.ts — POST /thoughts/bulk/delete (soft delete multiple):**
  ```typescript
  const result = await db.update(thoughts).set({
    syncStatus: 'pendingDeletion',
    modifiedAt: new Date(),
  }).where(inArray(thoughts.id, ids)).returning({ id: thoughts.id });
  ```
  Use `inArray()` instead of building dynamic `WHERE IN (?, ?, ?)` placeholders.

  **bulk.ts — POST /thoughts/bulk/recategorize:**
  ```typescript
  const result = await db.update(thoughts).set({
    category: newCategory,
    modifiedAt: new Date(),
    syncStatus: 'pending',
  }).where(inArray(thoughts.id, ids)).returning({ id: thoughts.id });
  ```

  **bulk.ts — POST /thoughts/bulk/therapy-classify:**
  Same pattern as recategorize but updating therapyClassification field.

  **bulk.ts — POST /thoughts/bulk/tag (add/remove tag):**
  For bulk tag operations, fetch all matching thoughts, modify tags in app code, update each:
  ```typescript
  const matchingThoughts = await db.select({ id: thoughts.id, tags: thoughts.tags })
    .from(thoughts).where(inArray(thoughts.id, ids));

  let modifiedCount = 0;
  for (const t of matchingThoughts) {
    const currentTags: string[] = (t.tags as string[]) || [];
    let newTags: string[];
    if (action === 'add' && !currentTags.includes(tag)) {
      newTags = [...currentTags, tag];
    } else if (action === 'remove') {
      newTags = currentTags.filter(x => x !== tag);
    } else continue;

    await db.update(thoughts).set({
      tags: newTags.length > 0 ? newTags : null,
      modifiedAt: new Date(),
      syncStatus: 'pending',
    }).where(eq(thoughts.id, t.id));
    modifiedCount++;
  }
  ```
  Wrap in a transaction for atomicity if needed.
