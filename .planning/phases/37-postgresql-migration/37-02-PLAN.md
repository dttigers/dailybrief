<task type="auto">
  <name>Task 1: Migrate GET /thoughts (with filters, FTS, pagination) and GET /thoughts/:id</name>
  <files>vigil-core/src/routes/thoughts.ts</files>
  <action>
  Replace all better-sqlite3 imports and queries in thoughts.ts with Drizzle ORM.

  **Import changes:**
  - Remove: `import { getDb } from '../db'` (or similar better-sqlite3 import)
  - Add: `import { db } from '../db/connection'`
  - Add: `import { thoughts } from '../db/schema'`
  - Add: `import { eq, and, or, sql, desc, count, ilike, inArray } from 'drizzle-orm'`

  **GET /thoughts (list with filters):**
  Build dynamic WHERE conditions array using Drizzle's `and()`:
  - category filter: `eq(thoughts.category, categoryParam)`
  - source filter: `eq(thoughts.source, sourceParam)`
  - taskStatus filter: `eq(thoughts.taskStatus, statusParam)`
  - therapyClassification filter: `eq(thoughts.therapyClassification, classParam)`
  - favoritesOnly: `eq(thoughts.isFavorited, true)`
  - date range: `gte(thoughts.createdAt, startDate)` and `lte(thoughts.createdAt, endDate)`
  - Soft delete exclusion: always add `not(eq(thoughts.syncStatus, 'pendingDeletion'))` or use `ne()`

  **Tag filter:** Tags are now JSONB. Use PostgreSQL JSONB containment:
  ```typescript
  sql`${thoughts.tags} @> ${JSON.stringify([tagParam])}::jsonb`
  ```
  This checks if the tags array contains the specified tag. Much more efficient than SQLite's json_each().

  **Full-text search (FTS5 → tsvector):**
  Replace `JOIN thoughts_fts ON t.id = fts.rowid WHERE thoughts_fts MATCH ?` with:
  ```typescript
  sql`${thoughts.searchVector} @@ plainto_tsquery('english', ${searchParam})`
  ```
  Use `plainto_tsquery` (not `to_tsquery`) — it handles plain text input without requiring special syntax.

  **Pagination:**
  ```typescript
  const results = await db.select()
    .from(thoughts)
    .where(and(...conditions))
    .orderBy(desc(thoughts.createdAt))
    .limit(limitParam)
    .offset(offsetParam);

  const [{ total }] = await db.select({ total: count() })
    .from(thoughts)
    .where(and(...conditions));
  ```

  **GET /thoughts/:id:**
  ```typescript
  const thought = await db.select().from(thoughts).where(eq(thoughts.id, id)).limit(1);
  ```

  **Response mapping:**
  The old code manually parsed `JSON.parse(row.tags)` and converted `isFavorited` from 0/1 to boolean. With JSONB and native boolean, this is no longer needed — Drizzle returns tags as string[] and isFavorited as boolean directly. However, ensure the response shape matches what clients expect (the ThoughtResponse interface). Map `createdAt`/`modifiedAt` timestamps to ISO strings if clients expect string dates.

<task type="auto">
  <name>Task 2: Migrate POST, PUT, DELETE /thoughts endpoints</name>
  <files>vigil-core/src/routes/thoughts.ts</files>
  <action>
  **POST /thoughts (create):**
  ```typescript
  const [newThought] = await db.insert(thoughts).values({
    content: body.content,
    source: body.source,
    category: body.category ?? null,
    tags: body.tags ?? null,
    cloudKitRecordID: crypto.randomUUID(),
  }).returning();
  ```
  Use `.returning()` to get the created row back (PostgreSQL supports RETURNING, unlike SQLite).
  Validate input same as before (content required, source must be valid enum, etc.).

  **PUT /thoughts/:id (update):**
  Build dynamic update object:
  ```typescript
  const updates: Partial<typeof thoughts.$inferInsert> = {};
  if (body.content !== undefined) updates.content = body.content;
  if (body.category !== undefined) updates.category = body.category;
  if (body.taskStatus !== undefined) updates.taskStatus = body.taskStatus;
  if (body.therapyClassification !== undefined) updates.therapyClassification = body.therapyClassification;
  if (body.tags !== undefined) updates.tags = body.tags;
  if (body.isFavorited !== undefined) updates.isFavorited = body.isFavorited;
  updates.modifiedAt = new Date();
  updates.syncStatus = 'pending';

  const [updated] = await db.update(thoughts)
    .set(updates)
    .where(eq(thoughts.id, id))
    .returning();
  ```

  **DELETE /thoughts/:id (soft delete):**
  ```typescript
  const [deleted] = await db.update(thoughts)
    .set({ syncStatus: 'pendingDeletion', modifiedAt: new Date() })
    .where(eq(thoughts.id, id))
    .returning();
  ```
  This is a soft delete — update syncStatus, don't actually delete the row.

  **Error handling:** Keep the same HTTP status codes (400 for validation, 404 for not found, 503 for db error). Wrap queries in try/catch.

  **Remove all references to the old getDb() / better-sqlite3 pattern.** The entire thoughts.ts file should only import from '../db/connection' and '../db/schema'.
