<!-- CS# extraction regex correction -->
RESEARCH Probe 6 + Pitfall: REQUIREMENTS.md `/^CS\d{7}$/` is wrong (real Polaris titles are compound like `CS1234567 - Printer not working`). Use extraction:

```javascript
function extractCaseNumber(title) {
  const match = title.match(/\bCS\d{7}\b/)
  return match ? match[0] : null
}
```

<!-- Manifest content_scripts load-order contract (Checker BLOCKER 5 resolution) -->
PATTERNS.md's manifest example shows ONLY `content-script.js` in the content_scripts[0].js array. **This plan's spec supersedes that example.** Both `popup-helpers.js` AND `content-script.js` MUST be present, with `popup-helpers.js` at index 0 and `content-script.js` at index 1.

Why this matters: `content-script.js` calls `extractCaseNumber` as a global. Chrome MV3 content_scripts inject in array order into the same isolated world, so a script at index 1 can reference globals defined by index 0. If `popup-helpers.js` is missing OR appears AFTER `content-script.js`, then `extractCaseNumber` is `undefined` at the MutationObserver callback site — SVCNOW-02 silently no-ops with NO thrown error (chrome.runtime.sendMessage is never called; no devtools console error fires either, because the ReferenceError is swallowed by the observer's microtask boundary). The drift detection appears to "work" until field-test time when no TITLE_DRIFT messages ever arrive.

Acceptance criterion: `JSON.parse(manifest).content_scripts[0].js` deepEquals exactly `["popup-helpers.js", "content-script.js"]` — index-position and array-length both pinned.
