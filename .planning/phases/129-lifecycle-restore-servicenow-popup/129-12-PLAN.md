<objective>
Close GAP-129-H (process gap, severity high): plan 129-02 shipped TypeScript build-breakers because the per-plan verify command exercised only the plan-scoped test files, not a full workspace `tsc` over every TypeScript file. The missing `navigateTo` import in `main.ts:restoreScreenFn` and the unused `TTL_MS` import in `launch-source-helpers.ts` only surfaced when the operator attempted to dev-sideload on 2026-05-16 (commits ca91f60, 57d7996 fix them).

The fix is procedural, not code: every PLAN.md producing TypeScript source changes must require a full workspace `npm run build` before SUMMARY.md is authored. This plan creates the convention document, the reusable task template, and the STATE.md lessons-learned entry.

The gap-closure plans 129-09 and 129-10 already adopt the gate in their verify commands; this plan formalizes it as the canonical rule.

Purpose: Prevent the same class of failure from recurring in any future phase. Make the rule discoverable via `.planning/conventions/` (a directory created by this plan as the home for process-level conventions).

Output: Convention document + task template + STATE.md update. No production source code changes.
</objective>

The reusable task template is a snippet of PLAN.md XML that planners drop in as the LAST task before SUMMARY authoring. The convention document defines the snippet's shape:
- task type: auto
- name: "Task N: [BLOCKING] full workspace build — gate before SUMMARY"
- files: none modified (verification-only)
- read_first: references the convention document
- action: describes the rule and rationale
- verify automated: runs the canonical workspace build command
- acceptance criteria: command exits 0; tsc clean across all sub-workspaces
- done: workspace build clean; safe to author SUMMARY

Exceptions: plans whose files_modified contains zero `.ts` / `.tsx` / `.js` / `.jsx` entries are exempt (e.g., doc-only plans like 129-11). All other plans (any TypeScript / JavaScript source touched, INCLUDING test files) MUST include the gate.
