  - `oauth2ClientId: String` — Azure AD app client ID (default: "")
  - `oauth2TenantId: String` — Azure AD tenant ID (default: "")
  - `oauth2RefreshToken: String` — stored after device code auth (default: "")

  Use `decodeIfPresent` with defaults for backward compatibility (existing configs without these fields must still load). Add CodingKeys for snake_case JSON mapping.

  **EmailService changes (EmailService.swift):**
  Pass `authType` and OAuth2 fields to IMAPClient. In the Python IMAP script:

  When authType is "oauth2":
  1. Exchange refresh_token for access_token via HTTP POST to `https://login.microsoftonline.com/{tenant_id}/oauth2/v2.0/token` with grant_type=refresh_token, client_id, refresh_token, scope=https://outlook.office365.com/.default
  2. Use `mail.authenticate('XOAUTH2', lambda x: f'user={email}\x01auth=Bearer {access_token}\x01\x01'.encode())` instead of `mail.login(email, password)`

  When authType is "app_password" (default): keep existing `mail.login()` behavior unchanged.

  Add error handling: if token exchange fails, log the HTTP status and error description. If XOAUTH2 auth fails, log "OAuth2 authentication failed — check Azure AD app permissions and refresh token."

  **Important:** Escape all Python string interpolation carefully in the Swift string literal. Use `\\(...)` for Python f-strings vs `\(...)` for Swift interpolation. The existing script already handles this pattern — follow the same escaping approach.
  </action>

<task type="auto">
  <name>Task 2: Add --email-auth CLI command for OAuth2 device code flow</name>
  <files>Sources/DailyBrief/DailyBrief.swift</files>
  <action>
  Add a new subcommand `EmailAuth` (invoked as `dailybrief email-auth`) to the ArgumentParser command group. This command performs Microsoft 365 OAuth2 device code flow:

  1. Load config to get `oauth2ClientId` and `oauth2TenantId` — error if empty with message "Set oauth2_client_id and oauth2_tenant_id in config first (see --setup)"
  2. POST to `https://login.microsoftonline.com/{tenant_id}/oauth2/v2.0/devicecode` with client_id and scope=`offline_access https://outlook.office365.com/IMAP.AccessAsUser.All`
  3. Parse response: `device_code`, `user_code`, `verification_uri`, `interval`, `expires_in`
  4. Print to console:
     ```
     To authenticate, visit: {verification_uri}
     Enter code: {user_code}
     Waiting for authentication...
     ```
  5. Poll `https://login.microsoftonline.com/{tenant_id}/oauth2/v2.0/token` with grant_type=urn:ietf:params:oauth:grant-type:device_code, device_code, client_id at the specified interval
  6. On success: extract `refresh_token` from response, update config.email.oauth2RefreshToken, save config via ConfigLoader.save(), print "Authentication successful! Refresh token saved."
  7. On "authorization_pending": continue polling
  8. On "expired_token" or other error: print error and exit

  Use Foundation URLSession for HTTP requests (async/await). Use JSONSerialization for parsing (avoid adding dependencies).

  Also update the --setup config template to include the new OAuth2 fields with comments:
  ```json
  "auth_type": "app_password",
  "oauth2_client_id": "",
  "oauth2_tenant_id": "",
  "oauth2_refresh_token": ""
  ```
  </action>
  <verify>swift build -c debug 2>&1 | tail -5 — compiles without errors. `dailybrief email-auth --help` shows usage.</verify>
