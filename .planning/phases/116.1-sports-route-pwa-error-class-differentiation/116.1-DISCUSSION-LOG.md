## Status code precision

### Q1: How should BDL upstream failures map to status codes?
| Option | Description | Selected |
|--------|-------------|----------|
| 502 + Retry-After proxy on 429 (Recommended) | Single 502, body verbatim per pasted scope, Retry-After header proxied when BDL sends 429 | ✓ |
| Single 502 for everything | All BDL failures → plain 502, no Retry-After | |
| Full split: 502/503/504 | 502 = BDL 5xx, 503 = BDL 429, 504 = timeout (with AbortController) | |

**User's choice:** 502 + Retry-After proxy on 429
**Notes:** Preserves pasted scope; surfaces real upstream backpressure when BDL gives it.

### Q2: Should fetchJSON in sports-service get an AbortController timeout?
| Option | Description | Selected |

### Q3: How should BDL 401/403 (the local env-gap symptom) classify?

### Q4: Response body shape — stick with pasted scope or extend?
