### Human Verification Required

#### 1. PWA countdown + disabled Retry — browser interaction

**Test:** Open PWA Settings, enable MLB league. Use browser DevTools (Network tab → intercept) or a local proxy to make `/v1/sports/teams/mlb` return a 502 response with body `{"error":"Upstream sports provider unavailable","retryAfter":30}` and header `Retry-After: 30`.
**Expected:** Red error block appears with text "Sports data temporarily unavailable." A "Try again in 0:29" (or similar) countdown appears next to the Retry button. The Retry button is disabled (visually grayed out, `disabled` attribute set) while countdown > 0. When countdown reaches 0 the button re-enables and the countdown disappears.
**Why human:** Live countdown timers, CSS opacity, and disabled-button interaction require visual + click testing in a real browser. Automated tests cover the logic via fake timers but cannot confirm the rendered visual state and actual button clickability.

#### 2. End-to-end local-env-gap reproduction

**Test:** Remove `BALLDONTLIE_API_KEY` from local `.env` (or set to empty string). Open PWA Settings and enable any league.
**Expected:** Server returns HTTP 502 with body `{"error":"Upstream sports provider unavailable"}` (no "balldontlie" substring). PWA shows "Sports data temporarily unavailable." copy. No stack trace, no BDL URL, no API key in any response or UI text.
**Why human:** The original bug trigger requires a real missing-key environment configuration — automated tests mock the fetch layer and cannot test actual Node.js process.env resolution.

#### 3. Brief PDF with upstream failure — visual placeholder

**Test:** With BDL unreachable (block `api.balldontlie.io` in `/etc/hosts` or use an invalid API key), trigger a brief generation for a user who has at least one league enabled.
**Expected:** Brief PDF renders a sports section showing placeholder text such as "NFL data temporarily unavailable." for each failed league, or "Sports data temporarily unavailable. Try again on tomorrow's brief." if all leagues failed. No league is silently omitted.
**Why human:** PDF rendering requires visual inspection of the generated document. The placeholder shape (`recentGame=null`, `upcomingGame=null`, `standings=[]`) feeds `drawSportSection` in pdf-service.ts which the automated tests do not exercise with real PDF output.
