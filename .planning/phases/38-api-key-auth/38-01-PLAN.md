<task type="auto">
  <name>Task 1: API keys schema + key generation script</name>
  <files>vigil-core/src/db/schema.ts, vigil-core/scripts/generate-key.ts</files>
  <action>
1. Add `api_keys` table to schema.ts:
   - `id` serial primary key
   - `name` text not null (human label like "g2-glasses", "mac-app")
   - `keyHash` text not null unique (SHA-256 hash of the actual key)
   - `keyPrefix` text not null (first 8 chars of key for identification in logs)
   - `createdAt` timestamp with timezone, default now
   - `lastUsedAt` timestamp with timezone, nullable
   - `isActive` boolean not null default true

   Store hash only — never store raw keys. Use `keyPrefix` for log-safe identification.

2. Run `drizzle-kit generate` then `drizzle-kit push` to apply schema.

3. Create `vigil-core/scripts/generate-key.ts`:
   - Generate a 32-byte random key with `crypto.randomBytes(32).toString('hex')`
   - Prefix with `vk_` for easy identification (format: `vk_{64 hex chars}`)
   - Hash with SHA-256, store hash + first 8 chars of key as prefix
   - Accept `--name` argument (required) for the key label
   - Print the raw key ONCE to stdout (it cannot be recovered after)
   - Use the same `db` import from `../src/db/connection.js`
   - Example usage: `npx tsx scripts/generate-key.ts --name "mac-app"`

4. Add npm script to package.json: `"db:generate-key": "tsx scripts/generate-key.ts"`
  </action>
  <verify>
- `npm run db:generate` succeeds (migration generated)
- `npm run db:push` succeeds (schema applied to database)
- `npx tsx scripts/generate-key.ts --name test-key` prints a key starting with `vk_`
- Database has api_keys row with hashed key (verify via drizzle-kit studio or raw query)
  </verify>
  <done>api_keys table exists in PostgreSQL, generate-key script creates and stores hashed keys</done>
</task>

<task type="auto">
  <name>Task 2: Auth middleware + protect all routes</name>
  <files>vigil-core/src/middleware/auth.ts, vigil-core/src/index.ts</files>
  <action>
1. Create `vigil-core/src/middleware/auth.ts`:
   - Export a Hono middleware function `bearerAuth`
   - Extract `Authorization: Bearer {token}` header
   - If missing or malformed: return 401 `{ error: "Missing or invalid Authorization header" }`
   - Hash the provided token with SHA-256
   - Look up hash in api_keys table where isActive = true
   - If no match: return 401 `{ error: "Invalid API key" }`
   - If match: update `lastUsedAt` to now (fire-and-forget, don't await — non-blocking)
   - Call `await next()` to proceed
   - Use timing-safe comparison is NOT needed here since we're comparing hashes looked up by exact match (the DB query itself is the comparison)

2. Update `vigil-core/src/index.ts`:
   - Import `bearerAuth` middleware
   - Apply middleware to all routes EXCEPT /v1/health
   - Strategy: Mount health route first without middleware, then apply middleware globally before other routes
   - Hono approach: use `app.use("/v1/*", bearerAuth)` after mounting health but the health route needs to be excluded. Better approach: mount health separately, then use `app.use()` with a path filter that skips /v1/health:
     ```
     app.route("/v1", health);
     app.use("/v1/*", async (c, next) => {
       if (c.req.path === "/v1/health") return next();
       return bearerAuth(c, next);
     });
     ```
     OR simpler: register health route first, then use middleware on remaining. In Hono, middleware applies to routes registered AFTER it. So:
     ```
     app.route("/v1", health);
     app.use("/v1/*", bearerAuth);
     // ... all other routes
     ```
     Use whichever Hono pattern correctly skips health. Test both approaches if needed.

3. Ensure error responses are JSON with appropriate status codes:
   - 401 for missing/invalid auth
   - Existing 4xx/5xx responses unchanged
  </action>
  <verify>
- `npm run build` succeeds (TypeScript compiles)
- `curl http://localhost:3001/v1/health` returns 200 (no auth required)
- `curl http://localhost:3001/v1/thoughts` returns 401 (auth required)
- `curl -H "Authorization: Bearer vk_WRONG" http://localhost:3001/v1/thoughts` returns 401
- `curl -H "Authorization: Bearer {valid-key}" http://localhost:3001/v1/thoughts` returns 200
  </verify>
  <done>All endpoints except /v1/health require valid Bearer token. Invalid/missing tokens return 401 JSON.</done>
