## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1` = 4px) |
| sm | 8px | Compact element spacing, tight list gaps (`py-2`), back-navigation icon+label gap (`gap-2`) |
| md | 16px | Default horizontal padding (`px-4`), section-level vertical padding (`py-6`) |
| lg | 24px | Section heading bottom margin (`mb-6`) |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Empty state centering vertical padding (`py-16`) |

Exceptions:
- iframe height: 600px fixed on desktop, 400px on mobile (max-w-sm breakpoint). Not a spacing token; a fixed layout constraint for PDF viewport.
- Touch targets for Generate and Download buttons: minimum 44px height (accessibility requirement for mobile).

## Interaction Contract

### Generate Flow
1. Page loads → check `briefs` list for today's date (YYYY-MM-DD).
2. **No today brief:** Show GenerateBriefButton. User clicks → `POST /v1/brief/generate`.
3. **During generation:** Replace button with GeneratingSpinner. Disable all brief card interactions.
4. **Success:** Fetch PDF binary via `GET /v1/brief/:date` → create `URL.createObjectURL(blob)` → render PdfPreviewFrame + DownloadButton.
5. **Error:** Show red error banner below generate section. Button reverts. Retry is manual (user clicks Generate again).

### PDF Preview Flow
- Today's brief: shown inline at page top after generation (or on page load if today's brief already exists).
- Past briefs: clicking a history card fetches PDF binary via `GET /v1/brief/:date` (same blob URL approach), replaces iframe src. Back button ("← Back to Briefs") returns to list. Detail view shows iframe at top, brief metadata (date, thought/task counts) below.
- Blob URLs: revoke via `URL.revokeObjectURL()` on React cleanup (`useEffect` return function).

### Download Flow
- DownloadButton renders as `<a href={blobUrl} download="vigil-brief-YYYY-MM-DD.pdf">`.
- Blob URL is valid while component is mounted. No additional state needed.

### Mobile Fallback
- If iframe PDF rendering is not supported (edge case on some mobile browsers), iframe will show blank. No programmatic detection — browser native behavior is acceptable for this phase.
- The DownloadButton remains functional regardless of iframe rendering support.

### vigilFetch Binary Variant
New `getBriefPdf(date: string): Promise<Blob>` in `client.ts`:
- Calls `vigilFetch` without `Content-Type: application/json` override (pass `headers: {}` to clear the JSON default).
- Returns `res.blob()` not `res.json()`.
- Used for both generate response and GET by date.

New `generateBrief(): Promise<Blob>` in `client.ts`:
- `POST /v1/brief/generate` with no request body.
- Override content-type: pass `headers: { 'Content-Type': '' }` or omit body entirely.
- Returns `res.blob()`.
