**client.ts — add two new exported functions** at the end of the "Brief History API" section (after `getBriefByDate`):

```typescript
export async function generateBrief(): Promise<Blob> {
  const res = await vigilFetch('/v1/brief/generate', {
    method: 'POST',
    headers: { 'Content-Type': '' },
  })
  if (!res.ok) {
    const text = await res.text().catch(() => '')
    throw new Error(`Brief generation failed: ${res.status}${text ? ` — ${text}` : ''}`)
  }
  return res.blob()
}

export async function getBriefPdf(date: string): Promise<Blob> {
  const res = await vigilFetch(`/v1/brief/${date}`, {
    headers: { 'Content-Type': '' },
  })
  if (!res.ok) throw new Error(`Failed to load brief PDF: ${res.status}`)
  return res.blob()
}
```

Critical details per D-09:
- Pass `headers: { 'Content-Type': '' }` to override vigilFetch's default `'Content-Type': 'application/json'`. The spread `...init?.headers` in vigilFetch means our empty string replaces the default.
- Use `res.blob()` NOT `res.json()` — the response is binary PDF.
- `generateBrief()` uses POST with no body. Do NOT pass a `body` field.
- `getBriefPdf()` calls `/v1/brief/${date}` (singular `brief`), NOT `/v1/briefs/${date}` (plural). The existing `getBriefByDate` uses the plural `/v1/briefs/` path for JSON metadata — these are different endpoints.

**handleGenerate function:**

```typescript
async function handleGenerate() {
  setGenerateState('generating')
  setGenerateError(null)
  try {
    const blob = await generateBrief()
    if (todayBlobUrl) URL.revokeObjectURL(todayBlobUrl)
    const url = URL.createObjectURL(blob)
    setTodayBlobUrl(url)
    setGenerateState('done')
  } catch (e: unknown) {
    setGenerateError(e instanceof Error ? e.message : 'Brief generation failed. Try again.')
    setGenerateState('error')
  }
}
```
